## Integration Findings

### Critical: CropLot Not Created for SYNCED HarvestEvents

**Root cause:** The staged-ops approve flow (`/api/admin/staged-ops/[id]/route.ts`, lines 181-218) creates a `HarvestEvent` with `dataSource: "SYNCED"` but does not create a corresponding `CropLot` record. The manual harvest route (`/api/field-enterprises/[id]/harvest/route.ts`, lines 66-137) does create CropLots via `generateLotNumber()`.

**Impact chain:**
1. SYNCED harvests have no lot number in the field history timeline (FIELD-06 partial)
2. SYNCED harvests show "—" for lot number in the PDF Harvest Log (RPT-03 partial)
3. SYNCED harvests are absent from the mass balance entirely (RPT-04 partial)

**Fix:** Add `generateLotNumber()` + `prisma.cropLot.create()` to the approve flow in `staged-ops/[id]/route.ts`, mirroring the pattern in `harvest/route.ts`.

### Non-Critical: Auth Guard Missing on Field Routes

`GET /api/fields` returns all fields across all farms when called without a `farmId` parameter. Both UI callers (`fields/page.tsx`, `reports/page.tsx`) call it with no filter. This is a **pre-existing pattern** from before the milestone work, not a regression. Adding `auth()` with session-based `farmId` extraction would fix multi-tenant isolation.
