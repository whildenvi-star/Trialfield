# Vision: Crop Year Accounting & QBO Integration (v5.0)

## Context

Farm-budget is currently a single-season budgeting tool. The user needs it to become a full management P&L system that:
- Tracks costs and revenue by **crop year** (harvest year), not calendar year
- Pulls actual expenses from **QuickBooks Online** to replace budget estimates
- Pulls revenue detail from **grain-tickets** (loads, buyers, settlements)
- Shows **both views**: crop year P&L (management) and cash flow (taxes/QBO)
- Serves as the **central hub** for the whole-farm financial picture

This is a new milestone — none of the current v4.0 phases cover this capability.

---

## Captured Decisions

### Core Model
- **Crop year = harvest year** — 2027 crop = everything harvested in 2027, regardless of when inputs were bought
- **Two-date system** — every transaction has a cash date (when paid) AND a crop year (which crop it belongs to)
- **No formal accrual accounting** — the two-date model handles prepaid/accrued without journal entries
- **Budget estimates → QBO actuals** — farm-budget already has planned costs; QBO actuals replace estimates as payments come in
- **Variance tracking** — show budgeted vs actual: "Fert: budgeted $45/ac, actual $48/ac (+$3/ac)"

### QBO Integration
- **QBO is the books** — all bills/checks/deposits flow through QBO; accountant uses it for taxes
- **Farm-specific chart of accounts** — Seed, Fertilizer, Chemical, Fuel, Repairs, Rent, etc. (already set up)
- **No farm/field breakdown in QBO** — all expenses in one bucket; farm-budget adds field-level assignment
- **Pull direction**: QBO → farm-budget (manual sync button, user-initiated)
- **Auto-process + exceptions** — system auto-assigns crop year/category/fields for confident matches; user only reviews exceptions
- **Review log** — "23 auto-assigned, 7 need review" summary; user can drill into auto-processed if curious
- **Exception UI** — show QBO transaction details + 2-3 best guesses for crop year/field assignment; user picks or overrides
- **Auto-learn from corrections** — after assigning "Helena Chemical" to "Fertilizer" 3 times, stop asking; auto-assign

### Allocation
- **Enterprise-based allocation** — pick enterprise (e.g., "2027 Org Corn"), all fields split by acres automatically
- **Split multi-enterprise transactions first** — break $30K into $18K wheat + $12K corn (line items), then allocate each piece
- **Acres is accurate enough** — no per-field override needed; acre-based split is good enough for management
- **Simple confirmation** — "Allocated $50K across 10 fields in 2027 Org Corn" — done

### Revenue
- **Grain-tickets has the detail** — loads, tickets, buyers, prices, settlements
- **QBO has the deposits** — they should reconcile with grain-tickets
- **Gov payments included** — ARC/PLC, crop insurance indemnities as revenue line items in crop year P&L

### Dashboard & Views
- **Farm-budget is the hub** — pulls from QBO (expenses), grain-tickets (revenue), fsa-acres (gov payments), farm-registry (fields)
- **Whole-farm dashboard** — all crop years side by side, total revenue/expenses/net income, drill down to enterprise → field
- **Rolling 3 years** — current, last, year before
- **Source-tagged data** — small tags (QBO, GT, FSA) show where each number came from, but no clutter
- **Color-coded confidence** — black/bold = actual (QBO confirmed), gray/italic = estimated (still budget); numbers "solidify" as actuals arrive

## Proposed Phase Structure

Rough breakdown (to be refined during `/gsd:new-milestone`):

- **Phase 24: Multi-crop-year data model** — Add crop year to transactions, rolling 3-year support in farm-budget
- **Phase 25: QBO OAuth2 + transaction pull** — Connect QBO, sync button, raw transaction import
- **Phase 26: Smart assignment engine** — Auto-process, exception queue, auto-learn, allocation by enterprise/acres
- **Phase 27: Whole-farm dashboard** — Cross-module data aggregation, crop year P&L, cash flow view, confidence indicators
- **Phase 28: Revenue integration** — Grain-tickets revenue pull, QBO deposit reconciliation, gov payment line items

## Prerequisites
- Finish v4.0 (Phases 21 + 23 remaining)
- QBO developer account and OAuth2 app registration
