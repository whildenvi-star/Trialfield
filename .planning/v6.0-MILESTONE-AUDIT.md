### Partial (2/27)

#### FSA-05: Validation Warnings with Clickable Links — PARTIAL

**Requirement:** User can see validation warnings (missing crop, date, unreported) with clickable links

**What works:**
- `GET /api/fsa/validation` returns structured warnings with 5 types (missing-crop, missing-date, unreported, no-insurance, missing-price)
- `CluWorkspace` fetches warnings on mount and maps them to `warningsByRecordId`
- `CluCard` renders warning badge (count) with tooltip showing warning messages

**What's missing:**
- No `onClick` handler on warning badges — they are tooltip-only
- `ValidationWarning.filter` field is computed server-side but never consumed in the UI
- No filter mechanism to scroll to or highlight affected CLU records when clicking a warning
- The "clickable links" portion of FSA-05 is not implemented

**Fix:** Add onClick to warning badge in CluCard that emits a filter event to CluWorkspace, filtering/scrolling to matching records.

#### FSA-06: Auto-Populate from Farm-Budget — PARTIAL

**Requirement:** User can auto-populate CLU crop assignments from farm-budget macro rollup with preview

**What works:**
- `GET /api/fsa/auto-populate-preview` is fully implemented
- Correctly proxies to farm-budget `localhost:3001/api/dashboard`
- Returns proposals for all 444 CLU records with `matchConfidence` (exact/suggested/none)
- Handles farm-budget offline with 502 response

**What's missing:**
- No button in CluWorkspace or any FSA component calls this endpoint
- Zero import sites for this route across all `src/components/fsa/*.tsx` files
- No proposal display UI, no confirm/apply action
- User literally cannot trigger auto-populate from the portal

**Fix:** Add "Auto-Populate from Budget" button to CluWorkspace header that fetches proposals, renders a review modal with match confidence, and triggers bulk-update on confirmation.
