### Integration Gaps (2 findings)

**IG-1: Portal CLU POST missing `registry_crop_id` in ALLOWED_FIELDS** (new finding)
- **File:** `glomalin-portal/src/app/api/fsa/clu-records/route.ts` lines 44-62
- **Impact:** New CLU records via portal POST have `registry_crop_id = null`. Yield-push matches on `registry_field_id + registry_crop_id`, so these records never receive grain-ticket yield updates.
- **Severity:** Medium — only portal-created new CLU records affected. fsa-acres Express path and PATCH endpoint both work correctly.
- **Fix:** Add `'registry_crop_id'` to the `ALLOWED_FIELDS` Set (one-line addition).
- **Affects:** CONS-10, CONS-11, PIPE-02

**IG-2: Organic-cert `crops.find` name collision** (carried forward)
- **File:** `organic-cert/src/app/(app)/field-enterprises/page.tsx` line 251
- **Impact:** `crops.find(c => c.name === v)` returns wrong variant for 13 organic/conventional name pairs — stores wrong `registryCropId`.
- **Severity:** Medium — affects organic-cert crop ID accuracy for organic crops.
- **Fix:** Match on crop ID from dropdown value instead of name string, or add organic flag to matching.
- **Affects:** CONS-11

### Documentation Gap

**ECOSYSTEM_TOKEN not in farm-budget .env.example** — Yield-push auth uses `ECOSYSTEM_TOKEN || EMBED_TOKEN` fallback. EMBED_TOKEN is documented; ECOSYSTEM_TOKEN is not. No runtime impact.
