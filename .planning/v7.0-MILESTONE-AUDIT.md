## E2E Flow Verification

### Flow 1: DNS -> Caddy -> PM2 -> Response
**Status: COMPLETE**
DNS A record -> Caddy HTTPS -> reverse_proxy localhost:PORT -> PM2 app responds. All config files chain correctly for all 8 apps.

### Flow 2: Native Module Access (FSA, Insurance, Claims)
**Status: COMPLETE**
User visits portal -> logs in -> clicks native module -> component renders from Supabase data. No cross-app dependency.

### Flow 3: Embedded Module Access (Grain Tickets, Budget, etc.)
**Status: BROKEN in production**
User visits portal -> clicks embedded module -> `getEmbedUrl()` returns `NEXT_PUBLIC_EMBED_URL_*` -> iframe renders with `http://localhost:PORT` -> browser cannot reach localhost on VPS.
**Fix:** Document `NEXT_PUBLIC_EMBED_URL_*` production values in DEPLOY.md section 6.
