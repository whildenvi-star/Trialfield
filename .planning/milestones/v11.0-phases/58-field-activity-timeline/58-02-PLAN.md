    **timeline-workspace.tsx** — 'use client' component:
    - Props: `{ fieldId: string, fieldName: string }`
    - State:
      - `entries: TimelineEntry[]` — merged from all resolved sources
      - `warnings: string[]` — names of sources that failed
      - `sourceLoading: Record<TimelineSource, boolean>` — per-source loading state, initialized to `{ budget: true, cert: true, fieldops: true, grain: true }` on fetch
      - `activeSources: Set<TimelineSource>` (all 4 active by default)
      - `year: number` (CURRENT_CROP_YEAR from config)
      - `expandedIds: Set<string>`

    - **Progressive per-source loading (critical — implements user decision):**
      On mount / fieldId change / year change:
      1. Reset entries to [], warnings to [], set all sourceLoading values to true.
      2. Fire 4 independent fetches in parallel (NOT Promise.allSettled — each resolves independently):
         ```
         const sources: TimelineSource[] = ['budget', 'cert', 'fieldops', 'grain'];
         sources.forEach(source => {
           fetch(`/api/timeline/${fieldId}/${source}?year=${year}`)
             .then(res => res.json())
             .then((data: SingleSourceResponse) => {
               if (data.error) {
                 setWarnings(prev => [...prev, `${source}: ${data.error}`]);
               } else {
                 setEntries(prev => sortByDate([...prev, ...data.entries]));
               }
               setSourceLoading(prev => ({ ...prev, [source]: false }));
             })
             .catch(() => {
               setWarnings(prev => [...prev, `${source}: network error`]);
               setSourceLoading(prev => ({ ...prev, [source]: false }));
             });
         });
         ```
      3. Use an AbortController per fetch cycle — abort all in-flight fetches when fieldId or year changes (cleanup in useEffect return).
      4. The `sortByDate` helper sorts entries by sortDate ascending, then by source priority (cert before budget for same date). This is called each time new entries merge in.

    - While ANY sourceLoading value is true, show skeleton shimmers below existing entries (3-4 placeholder cards with pulsing bg-glomalin-surface). As each source resolves, its entries appear immediately in the timeline — the user sees partial data progressively, never a blank screen waiting for all 4.
    - After all sources resolve (all sourceLoading false): build pairedMap for entries with pairedWith, create lookup so the card can receive its paired entry. (Pairing only computed after all sources loaded since budget+cert entries need to find each other.)
    - Render vertical timeline layout:
      - Left edge: date markers (day labels). Group entries by date.
      - Between date groups: if gap > 14 days (Claude's discretion on threshold), show a collapsed gap marker: "... N weeks ..." in muted text.
      - For each entry: render TimelineEntryCard with appropriate sourceColor from SOURCE_COLORS.
    - Warning banner: if warnings.length > 0, show amber banner at top listing unavailable sources.
    - Compute availableYears from a simple range: [CURRENT_CROP_YEAR - 2, CURRENT_CROP_YEAR - 1, CURRENT_CROP_YEAR].
    - Summary stats: total entries (filtered), count of status==='planned', most recent date in entries.
