**1. Create grain-tickets/public/settlement-summary.js** — a vanilla JS IIFE module (same pattern as settlements.js, farms.js):

Module structure:
- IIFE wrapping all code
- `var summaryInitialized = false;` guard for one-time init
- Listen for `tab-activate` event — when detail is 'settlements', call `initSettlementSummary()`
- `getCropYear()` helper (same harvest-season logic: months 1-5 = prior year)

`initSettlementSummary()`:
- If already initialized, just return (guard)
- Set `summaryInitialized = true`
- Find the container div `#settlement-summary-container` (added in index.html)
- Build the UI inside it:

UI layout (all built with DOM createElement, no template literals — match existing module patterns):

a) **Header row**: "Financial Summary" heading (h3) + crop year selector dropdown. Pre-populate dropdown with years from current crop year down to 2023. Default to current crop year. On change, call `loadSummary(selectedYear)`.

b) **Summary table container**: A div that will hold the rendered table.

c) **Totals row**: Below the table, show grand totals: total delivered bushels, total deductions, total net payment.

`loadSummary(cropYear)`:
- Show a "Loading..." message in the table container
- `fetch('/api/settlement-summary?cropYear=' + cropYear)`
- Parse JSON response
- Call `renderSummaryTable(data.summary, data.contractsAvailable)`

`renderSummaryTable(rows, contractsAvailable)`:
- If rows is empty, show "No settlement data for this crop year" message
- Sort rows by buyerName (asc), then crop (asc)
- Build an HTML table with columns:
  1. **Buyer** — buyerName
  2. **Crop** — crop name
  3. **Delivered BU** — deliveredBushels formatted with commas, 1 decimal
  4. **Avg Price/BU** — avgPricePerBushel formatted as $X.XXXX
  5. **Contract Price** — contractPricePerBushel formatted as $X.XXXX, or "—" if null
  6. **Variance** — priceVariance formatted as +$X.XXXX or -$X.XXXX:
     - Positive: green text (use CSS class `variance-positive`, color: var(--green, #7A9E7E))
     - Negative: red text (use CSS class `variance-negative`, color: #c44)
     - Null: "—" in muted color
  7. **Deductions** — totalDeductions formatted as $X,XXX.XX
  8. **Net Payment** — netPayment formatted as $X,XXX.XX

- If `contractsAvailable` is false, show a small note below the table: "Contract prices unavailable — portal unreachable" in muted text.

- After the table, render a **totals row** (outside table, as a summary div):
  - Total Delivered BU (sum), Total Deductions (sum), Total Net Payment (sum)

Formatting helpers:
- `fmtBu(n)` — returns number with commas and 1 decimal (e.g., "12,345.6")
- `fmtDollars(n)` — returns "$1,234.56" with commas
- `fmtPrice(n)` — returns "$5.4500" (4 decimals)
- `fmtVariance(n)` — returns "+$0.1200" or "-$0.1200" with sign prefix

Table styling: Use inline styles or add a `<style>` block at the top of the container matching the existing grain-tickets dark theme:
- Table: width 100%, border-collapse, font-size 0.85rem
- Header: background var(--surface, #0e0c0b), text var(--text, #e8d8c0), font-weight 600, text-align right for numbers
- Cells: padding 6px 10px, border-bottom 1px solid var(--border, #2a2218)
- Buyer/Crop cells: text-align left
- Number cells: text-align right, font-variant-numeric: tabular-nums
- Totals: bold, border-top 2px solid var(--accent, #C8860A)
