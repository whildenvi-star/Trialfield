The protected layout is a server component (async function). Since localStorage is client-only, we need a client wrapper for the banner toggle state. Approach:

**Step 1: Create a client-side banner wrapper in layout.tsx**

Extract the banner section into a new client component `BannerSection` defined in the same file (or a new file `glomalin-portal/src/components/layout/banner-section.tsx`). This component:
- Uses `useState` initialized from `localStorage.getItem('glomalin-banner-disabled')` — if value is `'true'`, banner starts hidden.
- Renders the desktop/mobile ASCIIBannerStrip instances conditionally (only when NOT disabled).
- Exposes a `bannerDisabled` state and `toggleBanner` callback.
- Passes `bannerDisabled` and `onBannerToggle` to the Header via props or context.

Since layout.tsx is a server component and Header is client, the cleanest approach is:
1. Create `glomalin-portal/src/components/layout/banner-section.tsx` as a 'use client' component that wraps both the Header and the banner strips.
2. Move the Header + banner rendering into this client component.
3. The server layout passes user data to `BannerSection` which passes it to Header.

**BannerSection component (`glomalin-portal/src/components/layout/banner-section.tsx`):**
```
'use client'
- Props: { user: { email, fullName, role } }
- State: bannerDisabled (boolean, init from localStorage key 'glomalin-banner-disabled' === 'true')
- On toggle: flip state, write localStorage.setItem('glomalin-banner-disabled', newValue ? 'true' : 'false')
- Renders: <Header user={user} bannerDisabled={bannerDisabled} onBannerToggle={toggleFn} />
- Conditionally renders: desktop ASCIIBannerStrip (hidden md:block, height=72) and mobile ASCIIBannerStrip (block md:hidden, height=48, nodeCount=6) — only when !bannerDisabled
```
