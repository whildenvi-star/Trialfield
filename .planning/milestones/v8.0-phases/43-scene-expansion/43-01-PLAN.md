- Export `type SceneType = 'mycelium' | 'drone' | 'seasonal'`
- Export `SCENE_LIST: SceneType[] = ['mycelium', 'drone', 'seasonal']` (order for cycling)
- Export interface `SceneRenderer`: a function signature `(cols: number, rows: number, time: number) => Float32Array` that returns a brightness grid identical in format to what generateMycelium returns
- Export `nextScene(current: SceneType): SceneType` helper that cycles to next scene in SCENE_LIST

Create scene-drone.ts:
- Import `noise2D`, `fbm` from `./ascii-noise` (reuse existing noise utilities)
- Export `generateDroneLandscape(cols: number, rows: number, time: number): Float32Array`
- The landscape is a top-down drone view scrolling slowly downward (time-based Y offset)
- Layer 1 (ground): Use fbm with 4 octaves to generate rolling terrain height. Map terrain height to brightness (higher = brighter, like sunlit hills). Shift terrain slowly with `time * 0.03` for gentle drift
- Layer 2 (crop rows): At regular Y intervals (every 3-4 rows), add horizontal brightness bands that undulate slightly with noise. Crop rows should be subtle (brightness 0.1-0.2) and only visible on lower terrain
- Layer 3 (clouds): Use fbm with 2 octaves at a different scale and faster time offset (`time * 0.08`) to create cloud shadows. Clouds reduce brightness where they overlap (multiply by 0.3-0.7). Clouds should be large, blobby, and slowly drifting
- Layer 4 (depth fog): Vertical gradient — top rows (far away) are dimmer (multiply by 0.3), bottom rows (close) are at full brightness. Linear interpolation from row 0 to last row
- Overall brightness should be comparable to mycelium (0.0-0.8 range) so the same CHAR_RAMP and charColor palette works
- Use different seeds than mycelium to ensure visually distinct noise patterns (seed offsets of 100+)

  <action>
Add multi-scene support to ASCIIBannerStrip:

1. Add `scene` prop to ASCIIBannerStripProps: `scene?: SceneType` (default 'mycelium')

2. Import SceneType from scene-types.ts and generateDroneLandscape from scene-drone.ts

3. Track `activeScene` and `prevScene` state for crossfade:
   - `activeScene` ref: the currently rendering scene
   - `prevScene` ref: the scene being faded out (null when no transition)
   - `transitionProgress` ref: 0..1 over 200ms
   - `transitionStart` ref: timestamp when transition began

4. When `scene` prop changes:
   - Set `prevScene` = current `activeScene`
   - Set `activeScene` = new scene
   - Set `transitionStart` = Date.now()
   - `transitionProgress` starts at 0

5. In the render function, modify the grid generation:
   - If no transition in progress: generate grid for activeScene only
   - If transition in progress (prevScene !== null):
     - Generate grid for prevScene
     - Generate grid for activeScene
     - Blend: `blended[i] = prevGrid[i] * (1 - progress) + activeGrid[i] * progress`
     - When progress reaches 1.0, set prevScene = null

6. Grid generation dispatch: Based on scene type, call appropriate generator:
   - 'mycelium': existing generateMycelium (with nodes, pulses, edges, edgeStates)
   - 'drone': generateDroneLandscape(cols, rows, time)
   - 'seasonal': for now, fall through to mycelium (implemented in Plan 02)

7. Transition timing: 200ms crossfade. In the render loop, compute progress as `Math.min(1, (Date.now() - transitionStart) / 200)`.

8. Keep ALL existing mycelium state management (nodes, edges, pulses, edgeStates) intact. Only initialize mycelium state when scene is 'mycelium' or during a transition FROM mycelium. When scene is NOT mycelium and no transition, skip mycelium tick functions to save CPU.

9. Also export `onNodeClick` prop: `onNodeClick?: () => void`. When canvas receives a click event, check if the click coordinates hit a bright node (brightness > 0.65 at click position). If so, call onNodeClick(). This is the easter egg hook — Plan 02 will wire it to scene cycling.
   - Convert click clientX/clientY to canvas-relative coordinates
   - Check the brightness grid at that cell position
   - Only fire if brightness > 0.65 (a visibly bright node)
  </action>
