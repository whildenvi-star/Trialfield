**1. Create GET /api/fields/[id]/history/route.ts:**

Create the directory `src/app/api/fields/[id]/history/` and add `route.ts` with a GET handler that:
- Accepts optional `offset` query param (default 0) for shifting the 3-year window
- Computes `years = [currentYear - offset, currentYear - offset - 1, currentYear - offset - 2]`
- Uses `prisma.field.findUnique` with `include.enterprises` filtered by `cropYear: { in: years }`, ordered by `cropYear: "desc"`
- Each enterprise includes: `fieldOperations` (with equipment, operator, ordered by operationDate asc), `materialUsages` (with material, ordered by applicationDate asc), `harvestEvents` (with equipment, cropLots, ordered by harvestDate asc), `fertilityEvents` (ordered by applicationDate asc)
- Returns 404 if field not found
- Returns `{ field, years }` on success
- Use `auth()` from `@/lib/auth` for session check. Use the session's `farmId` in the where clause to ensure tenant isolation: `where: { id, farmId: session.user.farmId }`

Follow the existing API route pattern from `src/app/api/fields/route.ts` for auth boilerplate and NextResponse usage.

**2. Extend GET /api/fields/route.ts:**

In the existing fields list endpoint, extend the Prisma query to include per-field activity stats:
- Add `include.enterprises` with `orderBy: { cropYear: "desc" }`, `take: 3` (3 most recent years)
- Inside enterprises include: `_count: { select: { fieldOperations: true, harvestEvents: true, materialUsages: true, fertilityEvents: true } }`
- Also include `fieldOperations: { orderBy: { operationDate: "desc" }, take: 1, select: { operationDate: true } }` and `harvestEvents: { orderBy: { harvestDate: "desc" }, take: 1, select: { harvestDate: true } }`
- After the query, compute `lastActivityDate` (max of latest operation/harvest dates across enterprises) and `totalRecords` (sum of all _count values across enterprises) for each field
- Return these computed fields alongside the existing field data

Do NOT use raw SQL or fetch all records. Use Prisma `_count` for efficient counting.
