# Milestone v1.0: Mobile PWA

**Status:** ✅ SHIPPED 2026-06-05
**Phases:** 1–5
**Total Plans:** 15

## Overview

Farm team members can open portal.whughesfarms.com on a phone and navigate, work offline, submit field observations with photos, see a role-gated dashboard, and (on OWL Orin) run corn-specialized weed detection with stem avoidance. The portal went from desktop-only to a fully functional mobile PWA with a hardened offline sync layer.

## Milestone Summary

**Key Decisions:**

- MobileHeader purely presentational (no pathname read inside component) — layout derives pageTitle from pathname for clean separation
- CSS-only dual shell (md:hidden / hidden md:block) — no UA detection, no JS runtime branching
- useSyncStatus uses CustomEvent pub/sub — decoupled from drawer/banner without prop drilling
- QueueDetailSheet implemented as native fixed bottom sheet — @radix-ui/react-dialog not in project dependencies
- IDB conflicts store uses number 0/1 instead of boolean — IDB indexes on boolean false are browser-inconsistent
- Queue-first: IDB write before upload attempt — guarantees no data loss even if network dies mid-submission
- Admin role short-circuits module_access query — all MODULES.map(m=>m.id) without DB filter
- FP16 (half=True) over INT8 for TensorRT export — ~26ms/frame on Orin Nano, avoids mAP drop for crop-protection safety
- is_in_buffer uses <= boundary — weed at exact radius distance is suppressed (crop protection priority)

**Issues Resolved:**

- SSH deploy blocker (01-03) — resolved before phase 1 close
- --sidebar-w guard and embed-breadcrumb layout regressions — fixed in 03-03 pre-deploy cleanup
- Safari Private Mode IDB crash — direct-upload fallback prevents ObservationForm crash

**Issues Deferred:**

- sync-macro endpoint unguarded (medium) — carried forward
- Sidebar budget-summary link visible to CREW (low) — carried forward
- Acres budget math wrong (v2.0 bug, unrelated to v1.0 Mobile PWA) — deferred, see STATE.md

**Technical Debt Incurred:**

- Phase 4 field observations: no production deploy/verification plan — sync verified locally only
- Phase 5 OWL Orin: training pipeline requires GPU host; Orin-only runtime scripts kept separate
- Resolution in ConflictDrawer marks resolved:1 locally only — no server reconciliation (field observations rarely conflict; safety net not merge engine)
