**4. Add deliveries CRUD with status auto-transition** — custom routes (NOT crudRoutes factory) because delivery saves must recalculate order status:

- `GET /api/deliveries` — return `store.deliveries` (optionally filter by `?orderId=`)
- `POST /api/deliveries` — create delivery, then call `recalcOrderStatus()` on linked order, save
- `PUT /api/deliveries/:id` — update delivery, then call `recalcOrderStatus()` on linked order, save
- `DELETE /api/deliveries/:id` — remove delivery, then call `recalcOrderStatus()` on linked order, save

Add a helper function `recalcOrderStatus(order)`:
```javascript
function recalcOrderStatus(order) {
  if (!order) return;
  var orderDeliveries = store.deliveries.filter(d => d.orderId === order.id);
  if (orderDeliveries.length === 0) { order.status = 'ordered'; return; }
  var delivered = {};
  orderDeliveries.forEach(d => (d.items || []).forEach(item => {
    delivered[item.productName] = (delivered[item.productName] || 0) + (item.deliveredQty || 0);
  }));
  var allComplete = (order.items || []).every(item =>
    (delivered[item.productName] || 0) >= (item.orderedQty || 0)
  );
  order.status = allComplete ? 'complete' : 'partial';
}
```
