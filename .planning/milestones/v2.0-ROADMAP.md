# Milestone v2.0: Projected vs Actual Farm Budget

**Status:** SHIPPED 2026-03-22
**Phases:** 5-8 (including 6.1 inserted)
**Total Plans:** 10

## Overview

Enable the office manager to record actuals against the farm manager's projected crop plan, while keeping financial performance data private to admin.

## Phases

### Phase 5: Privacy Foundation
**Goal**: Financial performance data is invisible to OFFICE and CREW through every access vector — API response, browser DevTools, and UI — before any role-filtered feature is built
**Depends on**: Nothing (first v2.0 phase)
**Requirements**: PRIV-01, PRIV-02, PRIV-03, PRIV-04
**Plans**: 2/2 plans complete

Plans:
- [x] 05-01-PLAN.md — Auth foundation + RBAC permissions + API field stripping
- [x] 05-02-PLAN.md — UI role-conditional rendering + end-to-end verification

**Success Criteria:**
  1. An OFFICE-authenticated request to the budget-summary API returns cost and yield data but contains no revenue, margin, sale price, or profit fields — confirmed via curl or DevTools Network tab
  2. A request with no session cookie to any budget API route receives a 401 error, not ADMIN-level data
  3. Sandy (OFFICE) can log in and view the enterprise Budget tab without seeing rental rates, overhead, labor, sale prices, or profit/acre anywhere in the page — including page source
  4. New `budget:read` and `budget:financial` permissions exist in rbac.ts and are enforced on the budget-summary route

- [x] 06-02-PLAN.md — Budget-summary dual computation + actuals PATCH/POST API routes

## Milestone Summary

**Decimal Phases:**
- Phase 6.1: Phase 6 Defect Fixes (inserted after Phase 6 for auth guard, seed formula, and category alignment fixes)

**Key Decisions:**
- Actuals as separate layer, not modifying projected — admin's crop plan is source of truth
- Financial data hidden at API + UI level — spread-conditional stripping, silent omission UI pattern
- No approval gate for actuals entry — Sandy's entries record immediately
- All enterprises (not just organic) — farm budget covers entire operation
- Spread-conditional field stripping — financial fields absent from JSON keys, not set to null
- Per-acre display with total-cost save — UI shows $/acre, API stores total cost
- EnterpriseType derived from organicStatus — registry is source of truth, not budget category
- Background sync never blocks initial render — mount with existing data, update after sync
- Weighted-average grand total across all rows — no simple sum across subtotals

**Issues Resolved:**
- ADMIN fallback in getAuthContext() bypassed all RBAC — removed unconditionally
- Budget-summary mixed PLANNED/CONFIRMED operations — split into projected (all) and actual (CONFIRMED only)
- Sync upsert match key collision for same-field different-type enterprises — @@unique extended to include enterpriseType
- Seed variance showed $/unit vs $/acre mismatch — fixed formula to actualPricePerUnit * rate
- Unguarded POST /api/import-plan/confirm route — auth guard added

**Issues Deferred (Tech Debt):**
- POST /api/fields/sync-macro has no auth guard — additive/idempotent operation, medium severity
- Sidebar budget-summary link visible to CREW — no data leak but CREW lands on 403 page
- Seed ActualCell displays raw actualPricePerUnit with /ac sub-label — cosmetic mismatch, variance math correct
- SUMMARY.md frontmatter missing requirements-completed field across plan summaries
