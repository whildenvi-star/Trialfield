**Denied toast component** — `components/denied-toast.tsx`:
- Client component (`'use client'`) that reads `useSearchParams()` for a `denied` query parameter
- If `denied` param exists, look up the module label from MODULES array (import from `@/lib/modules`): `MODULES.find(m => m.id === denied)?.label`
- Show a toast: fixed position bottom-right, `bg-soil-surface border border-soil-border rounded-lg shadow-xl p-4`, with text: "You don't have access to {Module Label}. Contact your administrator for access." (per CONTEXT.md decision — must name the specific module)
- Auto-dismiss after 5 seconds using useEffect + setTimeout
- Show/hide with useState, animate with opacity transition
- After dismissal, use `router.replace('/dashboard', { scroll: false })` to clean up the `?denied=` query param from the URL (use `useRouter` from `next/navigation`)

**Protected layout** — `(protected)/layout.tsx`:
- Server component that fetches the current user session
- Call `createClient()` from `@/lib/supabase/server`, then `supabase.auth.getUser()` to get the user
- Query profile: `supabase.from('profiles').select('full_name, role').eq('id', user.id).single()`
- Render `<Header user={{ email: user.email, fullName: profile.full_name, role: profile.role }} />`
- Render `<DeniedToast />` (wrapped in `<Suspense>` since it uses useSearchParams)
- Render `{children}` below the header in a `<main>` container with padding
- If no user (shouldn't happen — middleware catches this, but defensive): redirect to /login

**Dashboard page** — `(protected)/dashboard/page.tsx`:
- Simple server component placeholder
- Heading: "Dashboard" in soil-text
- Subtext: "Welcome to Glomalin" in soil-muted
- This is a placeholder — Phase 26 will build the real dashboard with module cards
