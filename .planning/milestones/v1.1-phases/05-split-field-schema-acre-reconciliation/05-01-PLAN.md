  <name>Task 1: Evolve FieldEnterprise schema, push to DB, apply partial index</name>
  <files>organic-cert/prisma/schema.prisma</files>
  <action>
Edit the FieldEnterprise model in `organic-cert/prisma/schema.prisma`:

1. **Add new fields** (insert after `crop String` line, before `variety`):
   - `label         String?       // "North 40", "Corn 2" — null for single enterprises`
   - `isFallow      Boolean       @default(false)`
   - `fallowCostAmount   Float?   // optional overhead cost for idle land`
   - `fallowCostCategory String?  // "Overhead", "Land Rent", "Insurance", etc.`

2. **Replace the existing unique constraint:**
   - Remove: `@@unique([fieldId, cropYear, crop])`
   - Add: `@@unique([fieldId, cropYear, crop, label])`

3. **Run schema push and client regeneration** (single command):
   ```bash
   cd organic-cert && npx prisma db push && npx prisma generate
   ```
   This syncs the database schema and regenerates `src/generated/prisma/`. The project uses `db push` (no migrations directory).

4. **Apply the partial unique index** to enforce null-label uniqueness (PostgreSQL treats NULL as distinct in unique constraints, so the composite `@@unique` alone would allow multiple `[fieldId, cropYear, crop, null]` rows):
   ```bash
   cd organic-cert && npx prisma db execute --stdin <<'SQL'
   CREATE UNIQUE INDEX IF NOT EXISTS "FieldEnterprise_no_label_unique"
     ON "FieldEnterprise"("fieldId", "cropYear", "crop")
     WHERE "label" IS NULL;
   SQL
   ```
   If `prisma db execute` is not available in this Prisma version, use `psql` with the DATABASE_URL from `.env` directly.

5. **Verify** existing data is unaffected: all existing FieldEnterprise records have `label = null`, `isFallow = false` by default. No data migration needed.

**Design decisions per user context:**
- Uniqueness is `[fieldId, cropYear, crop, label]` where label defaults to null for single-crop fields (per locked decision)
- `isFallow Boolean @default(false)` chosen over an enum (per Claude's discretion in research — simpler for a binary distinction)
- Fallow enterprises store their acreage in `plantedAcres` (per research open question recommendation — keeps acre math consistent: sum of all enterprise plantedAcres = total allocated)
