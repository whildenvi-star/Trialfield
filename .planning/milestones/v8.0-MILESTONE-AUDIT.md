### E2E Flows

#### Flow 1: Page Load with Preference Restoration

`layout.tsx (auth)` -> `BannerSection (read localStorage)` -> `ASCIIBannerStrip (render stored scene)` -> **COMPLETE**

- Auth guard checks Supabase session
- BannerSection reads `glomalin-banner-disabled` and `glomalin-scene` from localStorage
- If disabled, banner not rendered; otherwise, stored scene preference determines which renderer fires
- Banner fades in over 400ms

#### Flow 2: Scene Cycling via Easter Egg

`User clicks bright node` -> `handleCanvasClick (brightness > 0.65)` -> `onNodeClick` -> `banner-section handleNodeClick` -> `nextScene()` -> `localStorage write` -> `ASCIIBannerStrip crossfade (200ms)` -> **COMPLETE**

- Canvas click handler checks stored brightness grid for clicked cell
- If brightness > 0.65, triggers onNodeClick callback
- banner-section cycles to next scene in order: mycelium -> drone -> seasonal -> mycelium
- New preference saved to localStorage
- ASCIIBannerStrip detects scene prop change, starts 200ms dual-grid crossfade blend

### Component Hierarchy

```
(protected)/layout.tsx (server, auth guard)
  └─ BannerSection (client, localStorage orchestrator)
       ├─ Header (banner toggle [ON]/[OFF])
```
