**Status:** SHIPPED 2026-03-04

## Overview

Rewire organic-cert from a manual data-entry app into a live compilation engine that pulls field plans, inputs, seed, rotations, and harvest data from farm-budget, farm-registry, and grain-tickets — then compiles a complete NOP inspection packet with zero double-entry and total transparency. Plus procurement pipeline redesign for farm-budget.

## Milestone Summary

**Key Decisions:**
- Leech pattern: organic-cert reads from ecosystem, never writes back
- Zero new npm packages — all capability already installed
- Preview before commit: every compile shows diff before DB write
- Ecosystem clients: AbortController timeout + Promise.allSettled
- Field identity mismatch: explicit mapping step, not silent string comparison
- NOP compliance rules on resolved materials only
- Rotation snapshot is non-deferrable (calendar-gated hard deadline)
- Forecast endpoint is server-side (avoids duplicating Calc logic client-side)
- Deliveries use custom routes (not crudRoutes) for auto status transition

**Integration Risks Resolved:**
- Prisma partial unique index via manual migration + resolve (established pattern)
- Harvest crop name normalization between grain-tickets and organic-cert
- farm-budget single-season design bridged by rotation snapshots

**Tech Debt:**
- CNH FieldOps staging API still on mock mode (no production credentials)
