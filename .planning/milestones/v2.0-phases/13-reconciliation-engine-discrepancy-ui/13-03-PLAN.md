  <name>Task 1: Fix settlements list endpoint to honor buyerId and cropYear query params</name>
  <files>grain-tickets/server.js</files>
  <action>
    Modify the `GET /api/settlements` handler (server.js lines 1259-1274) to support optional `buyerId` and `cropYear` query parameters. When provided, add them to the Prisma `where` clause so the endpoint returns only settlements for the specified buyer and crop year. When not provided, the endpoint continues to return all settlements (backward compatible).

    Current code (lines 1259-1274):
    ```
    app.get('/api/settlements', async (req, res) => {
      try {
        res.set('Cache-Control', 'no-store');
        const settlements = await prisma.settlement.findMany({
          orderBy: { importedAt: 'desc' },
          include: {
            buyer: true,
            _count: { select: { lines: true } }
          }
        });
        res.json(settlements);
      } catch (e) { ... }
    });
    ```

    Change to:
    ```
    app.get('/api/settlements', async (req, res) => {
      try {
        res.set('Cache-Control', 'no-store');
        const where = {};
        if (req.query.buyerId) {
          const buyerId = parseInt(req.query.buyerId, 10);
          if (!isNaN(buyerId)) where.buyerId = buyerId;
        }
        if (req.query.cropYear) {
          const cropYear = parseInt(req.query.cropYear, 10);
          if (!isNaN(cropYear)) where.cropYear = cropYear;
        }
        const settlements = await prisma.settlement.findMany({
          where,
          orderBy: { importedAt: 'desc' },
          include: {
            buyer: true,
            _count: { select: { lines: true } }
          }
        });
        res.json(settlements);
      } catch (e) { ... }
    });
    ```

    This is the only change in server.js. Do NOT modify the reconciliation summary endpoint — its `farmCount` field name is correct and the client will be updated to match it in Task 2.
