**Add auth + permission gate at the top of the GET handler (PRIV-01, PRIV-03):**
1. Import `getAuthContext` from `@/lib/auth` and `hasPermission` from `@/lib/rbac` and `Role` type from `@prisma/client`
2. At the start of the GET function, call `const user = await getAuthContext()`
3. If `!user`, return `NextResponse.json({ error: "Unauthorized" }, { status: 401 })`
4. Cast role: `const role = user.role as Role`
5. If `!hasPermission(role, "budget:read")`, return `NextResponse.json({ error: "Forbidden" }, { status: 403 })`
6. Compute `const canSeeFinancial = hasPermission(role, "budget:financial")`

**Strip financial fields from response for non-ADMIN (PRIV-01):**
In the final `NextResponse.json()` return, use spread-conditional to omit financial fields when `!canSeeFinancial`:
- `revenueProjection` -- omit entirely (use `...(canSeeFinancial && revenueProjection ? { revenueProjection } : {})`)
- `fallowCost` -- strip the `category` field for non-ADMIN: when `canSeeFinancial`, return full `{ amount, category }`; when not, return only `{ amount: enterprise.fallowCostAmount }` (the category can reveal overhead/rent/insurance which are hidden per user decision)
- Any fields containing: land rent, overhead, drying costs, interest, crop insurance, sale prices, revenue, projected revenue, gross margin, profit/acre -- must be conditionally included only when `canSeeFinancial` is true

Fields that ALWAYS stay visible (per locked decision): seedCosts, materialCosts, operationCosts, totalSeedCost, totalMaterialCost, totalOperationCost, totalCostOfProduction, costPerAcre, acres, and their line-item detail (variety, brand, rate, pricePerUnit, totalCost, etc.).

**CRITICAL: Use spread-conditional pattern, NOT null values.** Financial fields must be completely absent from JSON keys when stripped -- `...(condition ? { key: value } : {})` not `key: condition ? value : null`. Per user decision: "no trace in DevTools Network tab."

Keep all existing computation logic unchanged -- only add the auth gate at the top and modify the response construction at the bottom.
