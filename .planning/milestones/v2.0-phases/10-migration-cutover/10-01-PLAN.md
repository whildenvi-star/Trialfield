Create `grain-tickets/migrate-json.js` as a standalone Node.js CommonJS script. This script is run ONCE during the cutover window with the server stopped.

**Script structure:**

1. **Setup and CLI parsing:**
   - `require('dotenv/config')` at top (same pattern as lib/db.js)
   - Import: `fs`, `path`, `{ PrismaClient }` from `@prisma/client`, `Calc` from `./public/calc.js`
   - Create a NEW PrismaClient instance (not the singleton — this is a short-lived script)
   - Parse `--dry-run` flag from `process.argv`
   - Define `DATA_FILE = path.join(__dirname, 'data', 'data.json')`

2. **Data loading:**
   - `JSON.parse(fs.readFileSync(DATA_FILE, 'utf8'))` into `store`
   - Extract `{ tickets, farms, cropConfig }` from store
   - Console log source counts: `Source: 527 tickets, 63 farms, 37 crop configs`

3. **Dry-run gate:**
   - If `--dry-run`, log what would be migrated (counts, sample HBT extraction on first 5 tickets, sample truck extraction), then `prisma.$disconnect()` and return

4. **HBT bin number extraction function:**
   ```javascript
   function extractHbtBinNo(notes) {
     if (!notes) return null;
     const hbtMatch = notes.match(/HBT\s*#\s*(\d+)/i);
     if (hbtMatch) return hbtMatch[1];
     const blueMatch = notes.match(/blue\s+ticket\s+(\d+)/i);
     if (blueMatch) return blueMatch[1];
     return null;
   }
   ```

5. **Truck ID extraction function:**
   ```javascript
   function extractTruckId(notes) {
     if (!notes) return null;
     const m = notes.match(/(?:Trk|Truck)\s*#?\s*(\w+)/i);
     if (m) return m[1];
     return null;
   }
   ```

6. **cropYear derivation function:**
   ```javascript
   function getCropYear(dateStr) {
     if (!dateStr) return 2025;
     const year = parseInt(dateStr.slice(0, 4), 10);
     return isNaN(year) ? 2025 : year;
   }
   ```

7. **Migration steps (all inside an async run() function):**

   **Step 1 — Migrate CropConfigs:**
   - Convert `Object.entries(cropConfig)` to rows: `{ cropYear: 2025, cropName, testWeight: cfg.testWeight || 56, moistureShrink: cfg.moistureShrink || 0, discount: cfg.discount || 0 }`
   - `prisma.cropConfig.createMany({ data: cropRows })`
   - Verify: `result.count === cropRows.length`
   - Log: `CropConfigs: ${result.count} migrated`

   **Step 2 — Migrate Farms:**
   - Check for duplicate farm names (warn if found, keep first occurrence)
   - Map each farm to: `{ legacyId: f.id, name: (f.farm || '').trim(), crop: (f.crop || '').trim() || null, acres: parseFloat(f.acres) || 0, unit: f.unit || 'BU', type: f.type || 'Conventional', guarantee: parseFloat(f.guarantee) || 0, coverage: parseFloat(f.coverage) || 0, claimThreshold: parseFloat(f.claimThreshold) || 0, discount: parseFloat(f.discount) || 0, testWeight: parseFloat(f.testWeight) || 56, driver: f.driver || null, truck: parseFloat(f.truck) || 0 }`
   - `prisma.farm.createMany({ data: farmRows })`
   - Verify count matches
   - Log: `Farms: ${result.count} migrated`

   **Step 3 — Migrate Tickets:**
   - Map each ticket to:
     ```javascript
     {
       legacyId: t.id,
       date: new Date(t.date + 'T12:00:00.000Z'),  // noon UTC — safe for any timezone
       cropYear: getCropYear(t.date),
       farm: (t.farm || '').trim(),
       netWeight: parseFloat(t.netWeight) || 0,
       moisture: parseFloat(t.moisture) || 0,
       fm: parseFloat(t.fm) || 0,
       crop: (t.crop || '').trim(),
       ticketNo: (t.ticketNo || '').trim() || null,
       hbtBinNo: extractHbtBinNo(t.notes),
       truckId: extractTruckId(t.notes),
       notes: (t.notes || '').trim() || null
     }
     ```
   - Use `new Date(dateStr + 'T12:00:00.000Z')` for dates (noon UTC avoids timezone shift — per research Pitfall 1)
   - `prisma.ticket.createMany({ data: ticketRows })`
   - Verify count matches
   - Log: `Tickets: ${result.count} migrated`
   - Log extraction stats: `HBT bins extracted: ${count}, Truck IDs extracted: ${count}`

   **Step 4 — Verify calc.js parity on 10 random tickets:**
   - Build cropConfig object from DB: `prisma.cropConfig.findMany({ where: { cropYear: 2025 } })` -> convert rows to `{ cropName: { discount, testWeight, moistureShrink } }` object
   - Pick 10 unique random indices from source tickets array
   - For each: look up DB ticket by legacyId via `prisma.ticket.findFirst({ where: { legacyId } })`
   - Convert DB ticket to calc.js shape: `{ netWeight, moisture, fm, crop }`
   - Run `Calc.computeTicket()` against JSON source ticket with JSON cropConfig, and against DB-shaped ticket with DB-derived cropConfig
   - Compare results via `JSON.stringify()` — must be identical
   - If ANY mismatch: log error with full details and `process.exit(1)` — do NOT proceed to archive
   - If all match: log `calc.js parity: 10/10 tickets match`

   **Step 5 — Archive data.json and delete .bak files:**
   - `fs.renameSync(DATA_FILE, DATA_FILE + '.archive')` — atomic rename
   - Delete `data.json.bak.1` through `data.json.bak.5` if they exist via `fs.unlinkSync`
   - Log each deletion

   **Step 6 — Print summary:**
   - Total counts: tickets, farms, crop configs, HBT extractions, truck extractions
   - Any warnings (duplicate farm names, data anomalies like 0 weight or missing dates)
   - `prisma.$disconnect()`

8. **Error handling:**
   - Wrap entire run() in try/catch
   - On error: log full error, `prisma.$disconnect()`, `process.exit(1)`
   - The script is fail-fast: any row count mismatch or calc parity failure aborts immediately

**Important notes:**
- Do NOT import from lib/db.js — this script creates its own PrismaClient (it runs outside the server process)
- Do NOT use `skipDuplicates` on createMany — we want to fail on true duplicates
- Data anomalies (0 weight, missing dates, empty farm names): warn in console but migrate everything — bad data in JSON stays as-is in DB per user decision
- Notes field keeps original text intact — extraction copies data, doesn't remove it
