Then create `prisma/schema.prisma` with exactly these contents:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model Ticket {
  id          Int      @id @default(autoincrement())
  legacyId    String?  // Preserves "t_000001" string IDs from JSON for Phase 10 migration matching
  date        DateTime
  cropYear    Int      // 2025, 2026 — derived from date during Phase 10 migration
  farm        String   // Farm name (string, matched to Farm.name)
  netWeight   Float    // Pounds — raw scale weight
  moisture    Float    // Moisture percentage
  fm          Float    @default(0) // Foreign matter percentage
  crop        String   // Crop name (matched to CropConfig.cropName)
  ticketNo    String?  // NOT unique — 14 known duplicates in existing data
  hbtBinNo    String?  // Hughes Blue Ticket bin number — extracted from notes in Phase 10
  truckId     String?  // Truck identifier — extracted from notes in Phase 10
  destination String?  // Free-text destination — replaced by buyerId FK in Phase 11
  notes       String?  // Original notes text preserved after field extraction
  buyerId     Int?     // FK to Buyer — null until Phase 11 wires buyer selection
  buyer       Buyer?   @relation(fields: [buyerId], references: [id])
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  settlementLines SettlementLine[] // Matched settlement lines — populated in Phase 13

  @@index([ticketNo])       // Non-unique: 14 duplicate ticket numbers exist in data
  @@index([farm, cropYear]) // Farm summary queries
  @@index([crop, cropYear]) // Crop summary queries
  @@index([buyerId])        // Buyer filter queries (Phase 11)
}

model Farm {
  id             Int       @id @default(autoincrement())
  legacyId       String?   // Preserves "f_001" string IDs from JSON
  registryId     String?   // farm-registry field ID for future sync
  name           String    @unique // "Airport", "Schultz", etc.
  crop           String?   // Primary crop assignment (from farms array)
  acres          Float     @default(0) // Field acres for this farm-crop combo
  reportingAcres Float     @default(0) // Synced from farm-registry
  organicAcres   Float     @default(0) // Synced from farm-registry
  unit           String    @default("BU") // BU, CWT, etc.
  type           String    @default("Conventional") // Conventional, Organic, etc.
  guarantee      Float     @default(0) // Crop insurance guarantee
  coverage       Float     @default(0) // Coverage percentage
  claimThreshold Float     @default(0) // Claim threshold
  discount       Float     @default(0) // Per-farm discount override
  testWeight     Float     @default(56) // Per-farm test weight override
  driver         String?   // Default driver for this farm
  truck          Float     @default(0) // Default truck number
  notes          String?
  syncedAt       DateTime? // Last sync from farm-registry API
  createdAt      DateTime  @default(now())
  updatedAt      DateTime  @updatedAt
}

model CropConfig {
  id             Int      @id @default(autoincrement())
  cropYear       Int      // 2025, 2026, etc. — allows per-year config changes
  cropName       String   // "Hybrid Rye", "Organic Yellow Corn", etc.
  testWeight     Float    @default(56) // Pounds per bushel
  moistureShrink Float    @default(0)  // Moisture shrink threshold percentage
  discount       Float    @default(0)  // Discount percentage
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  @@unique([cropYear, cropName])
  @@index([cropYear])
}

model Buyer {
  id          Int      @id @default(autoincrement())
  name        String   @unique // "Meristem Malt", "ADM", etc.
  shortCode   String?  // "MRM", "ADM", "COOP" — for compact display
  type        String?  // "elevator", "maltster", "co-op"
  notes       String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  tickets     Ticket[]
  settlements Settlement[]
  columnMaps  BuyerColumnMap[]
}

model BuyerColumnMap {
  id        Int      @id @default(autoincrement())
  buyerId   Int
  buyer     Buyer    @relation(fields: [buyerId], references: [id], onDelete: Cascade)
  fieldName String   // "ticketNo", "netWeight", "moisture", etc.
  csvColumn String   // Column name or index from buyer's settlement CSV/Excel
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([buyerId, fieldName])
}

model Settlement {
  id          Int      @id @default(autoincrement())
  buyerId     Int
  buyer       Buyer    @relation(fields: [buyerId], references: [id])
  cropYear    Int
  importedAt  DateTime @default(now())
  sourceFile  String?  // Original uploaded filename
  notes       String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  lines SettlementLine[]

  @@index([buyerId, cropYear])
}

model SettlementLine {
  id           Int        @id @default(autoincrement())
  settlementId Int
  settlement   Settlement @relation(fields: [settlementId], references: [id], onDelete: Cascade)
  ticketNo     String?    // Ticket number from buyer's file — may not exactly match farm ticket
  date         DateTime?
  netWeight    Float?     // Pounds — buyer's reported weight
  moisture     Float?     // Buyer's reported moisture
  netBushels   Float?     // Buyer's computed bushels (their shrink method)
  price        Decimal?   @db.Decimal(10, 4) // $/bushel — Decimal for monetary precision
  deductions   Decimal?   @db.Decimal(10, 2) // Total deductions in dollars
  netPayment   Decimal?   @db.Decimal(10, 2) // Net payment in dollars
  ticketId     Int?       // FK to Ticket — set during Phase 13 reconciliation matching
  ticket       Ticket?    @relation(fields: [ticketId], references: [id])
  matchStatus  String     @default("unmatched") // "unmatched" | "matched" | "disputed" | "manual"
  notes        String?
  createdAt    DateTime   @default(now())
  updatedAt    DateTime   @updatedAt

  @@index([settlementId])
  @@index([ticketNo])
  @@index([ticketId])
}
```

**Key schema decisions explained:**
1. `ticketNo` uses `@@index` not `@unique` — 14 known duplicates would block Phase 10 migration
2. `hbtBinNo` and `truckId` are first-class columns — chain-of-custody fields extracted from notes in Phase 10
3. `CropConfig.cropYear` enables per-season config changes (test weights, shrink rates evolve)
4. `Decimal` for monetary fields on SettlementLine — prevents IEEE 754 rounding errors on financial sums
5. `Float` for weights and moisture — matches existing calc.js arithmetic, no precision change needed
6. `legacyId` on Ticket and Farm — preserves string IDs for Phase 10 migration cross-referencing
7. `Farm` model mirrors the full farms array shape from data.json (crop, acres, guarantee, coverage, etc.) — not just a name lookup
8. `BuyerColumnMap` uses `onDelete: Cascade` — if a buyer is deleted, their column mappings go too
9. `SettlementLine` uses `onDelete: Cascade` from Settlement — deleting a settlement deletes its lines
10. `destination` field on Ticket captures current free-text destination until Phase 11 replaces it with buyerId FK

Create `grain-tickets/lib/db.js` with exactly these contents:

```javascript
// PrismaClient singleton for grain-tickets
// Prevents "too many connections" from multiple instantiations during dev hot-reload (nodemon)
// Pattern: organic-cert/src/lib/prisma.ts adapted to CommonJS

require('dotenv/config');
const { PrismaClient } = require('@prisma/client');

let prisma;

if (process.env.NODE_ENV === 'production') {
  prisma = new PrismaClient();
} else {
  if (!global.__prisma) {
    global.__prisma = new PrismaClient();
  }
  prisma = global.__prisma;
}

module.exports = prisma;
```

**Key design decisions:**
- `require('dotenv/config')` at the top — ensures DATABASE_URL is loaded before PrismaClient reads it, regardless of which file requires db.js first
- `global.__prisma` (double underscore prefix) — avoids collision with any other global named `prisma`
- No explicit `$connect()` call — PrismaClient connects lazily on first query (Prisma best practice)
- CommonJS `module.exports` — matches the rest of grain-tickets (vanilla JS, no TypeScript)
- Production path creates a fresh instance (no global guard needed in single-process production)
