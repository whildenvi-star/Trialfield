Create `src/app/api/budget-summary/route.ts` with a GET handler that:

**Auth & RBAC (exact pattern from existing budget-summary route):**
- Call `getAuthContext()` — return 401 if null

- Derive `canSeeFinancial = hasPermission(role, "budget:financial")`

**Crop year resolution (no schema change):**
```typescript
const cropYearParam = new URL(request.url).searchParams.get("cropYear");
let cropYear: number;
if (cropYearParam) {
  cropYear = parseInt(cropYearParam, 10);
} else {
  const maxYear = await prisma.fieldEnterprise.aggregate({
    _max: { cropYear: true },
    where: { field: { farmId } },
  });
  cropYear = maxYear._max.cropYear ?? new Date().getFullYear();
}
```

**Fetch all enterprises for the crop year:**
```typescript
const enterprises = await prisma.fieldEnterprise.findMany({
  where: { cropYear, field: { farmId } },
  include: {
    field: true,
    seedUsages: { include: { seedLot: true } },
    materialUsages: { include: { material: true }, orderBy: { applicationDate: "asc" } },
    fieldOperations: { orderBy: { operationDate: "asc" } },
  },
  orderBy: [{ enterpriseType: "asc" }, { crop: "asc" }, { label: "asc" }],
});
```

**Per-enterprise budget computation — replicate the proven arithmetic from the existing per-enterprise `budget-summary/route.ts`:**

For each enterprise, compute:
1. **Seed costs** — same formula: `unitsPerAcre = rate / seedsPerUnit; totalCost = unitsPerAcre * purchasePrice * acres`. Actuals use `actualPricePerUnit`.
2. **Material costs** — separate projected (`dataSource !== "ACTUAL"`) and unplanned (`dataSource === "ACTUAL" && totalCost === null`). Compute `totalCost` per the same formula.
3. **Operation costs** — same formula: `costPerAcre * acresWorked`.
4. **Projected totals** — sum all seed + material + operation costs (ALL operations regardless of passStatus).
5. **Actual totals** — seed: sum where `actualPricePerUnit !== null`; material: sum where `actualTotalCost !== null` + unplanned; operations: CONFIRMED only. `allActualsNull` guard applies.
6. **Variance** — `projected - actual` (null if actual is null). Computed server-side only.

**Category bucketing for Macro Rollup columns — use same keyword matching as BudgetTab.tsx:**
- Fertilizer: `category.toLowerCase().includes("fertilizer") || category.toLowerCase().includes("fert")`
- Chemical: `category.toLowerCase().includes("chemical") || category.toLowerCase().includes("chem") || category.toLowerCase().includes("pesticide") || category.toLowerCase().includes("herbicide") || category.toLowerCase().includes("insecticide")`
- Other materials (Custom Application, etc.) fall into a catch-all that gets added to the relevant column or a separate column

For each enterprise, produce per-acre values by dividing total costs by `plantedAcres`:
```typescript
const acres = enterprise.plantedAcres || 0;
// ... compute totalSeedCost, then:
const projectedSeedPerAcre = acres > 0 ? round2(totalSeedCost / acres) : 0;
```

Same for Fertilizer, Chemical, Operations. Actual per-acre values are null if the underlying actual total is null.

**Build the row object:**
```typescript
interface FarmBudgetRow {
  enterpriseId: string;
  crop: string;
  variety: string | null;
  label: string | null;
  fieldName: string;
  acres: number;
  enterpriseType: "ORGANIC" | "CONVENTIONAL";
  projectedSeedPerAcre: number;
  projectedFertilizerPerAcre: number;
  projectedChemicalPerAcre: number;
  projectedOperationsPerAcre: number;
  projectedTotalPerAcre: number;
  actualSeedPerAcre: number | null;
  actualFertilizerPerAcre: number | null;
  actualChemicalPerAcre: number | null;
  actualOperationsPerAcre: number | null;
  actualTotalPerAcre: number | null;
  varianceTotalPerAcre: number | null;
  // Financial — only if canSeeFinancial
  projectedRevPerAcre?: number;
  projectedMarginPerAcre?: number;
}
```

**Grouping:**
- Split rows into `organic` (enterpriseType === "ORGANIC") and `conventional` (enterpriseType === "CONVENTIONAL")

**Section subtotals — weighted average by acreage (NOT simple average):**
Create a helper function `computeSubtotal(rows: FarmBudgetRow[])` that:
```typescript
const totalAcres = rows.reduce((s, r) => s + r.acres, 0);
// For each per-acre column:
const projectedSeedPerAcre = totalAcres > 0
  ? round2(rows.reduce((s, r) => s + r.projectedSeedPerAcre * r.acres, 0) / totalAcres)
  : 0;
// Same for fertilizer, chemical, operations, total
// For actual columns: only include rows where actual is not null
const rowsWithActuals = rows.filter(r => r.actualTotalPerAcre !== null);
const actualAcres = rowsWithActuals.reduce((s, r) => s + r.acres, 0);
const actualTotalPerAcre = actualAcres > 0
  ? round2(rowsWithActuals.reduce((s, r) => s + (r.actualTotalPerAcre ?? 0) * r.acres, 0) / actualAcres)
  : null;
// Variance for subtotals: if actualTotalPerAcre is null, variance is null
```

**Grand total** — same weighted average logic across ALL rows (organic + conventional combined).

**Financial fields — spread-conditional pattern:**
Only include `projectedRevPerAcre` and `projectedMarginPerAcre` on rows and subtotals when `canSeeFinancial` is true. Use the existing spread-conditional pattern:
```typescript
...(canSeeFinancial && revenueProjection ? {
  projectedRevPerAcre: round2(projectedGrossRevenue / acres),
  projectedMarginPerAcre: round2(projectedGrossMargin / acres),
} : {}),
```

Revenue projection per enterprise uses `targetYieldPerAcre * targetPricePerUnit * acres` (same as existing route).

**Response shape:**
```typescript
{
  cropYear: number,
  organic: FarmBudgetRow[],
  conventional: FarmBudgetRow[],
  organicSubtotal: SubtotalRow,
  conventionalSubtotal: SubtotalRow,
  grandTotal: SubtotalRow,
}
```

Use `round2` helper: `const round2 = (n: number) => Math.round(n * 100) / 100;`

**Error handling:** Wrap in try/catch, return 500 with sanitized error message on failure.

**Important: Do NOT use the existing per-enterprise budget-summary route as a sub-call.** Replicate the arithmetic inline to avoid N+1 API calls. The existing route is the reference for correct arithmetic — match it exactly.
