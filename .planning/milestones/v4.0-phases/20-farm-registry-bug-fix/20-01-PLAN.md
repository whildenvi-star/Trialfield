**In farm-registry/public/app.js:**

Modify the save handler (the `btn-save` click handler, currently lines 340-448):

1. **Add loading state on save button.** At the start of the save handler (after all validation passes, right before the `api()` call on line 438):
   - Get the save button: `var btn = document.getElementById('btn-save');`
   - Store original text: `var originalText = btn.textContent;`
   - Set loading state: `btn.textContent = 'Saving...'; btn.disabled = true;`

2. **Replace the `api()` helper call with a raw `fetch()` call** so we can check `response.ok` before parsing JSON. The current `api()` helper (line 6-9) always calls `.json()` regardless of status code — it does not distinguish success from error. Replace:
   ```js
   api(method, url, body).then(function (saved) { ... });
   ```
   With:
   ```js
   fetch(url, {
     method: method,
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify(body)
   })
   .then(function (res) {
     btn.textContent = originalText;
     btn.disabled = false;
     if (!res.ok) return res.json().then(function (err) { throw err; });
     return res.json();
   })
   .then(function (saved) {
     // existing success logic: show green toast, set selectedId, loadFields()
     var toast = document.getElementById('save-toast');
     toast.textContent = isNew ? 'Field created' : 'Saved';
     toast.className = 'save-toast';
     toast.classList.remove('hidden');
     setTimeout(function () { toast.classList.add('hidden'); }, 2000);
     isNew = false;
     selectedId = saved.id;
     document.getElementById('btn-delete').style.display = '';
     loadFields();
   })
   .catch(function (err) {
     btn.textContent = originalText;
     btn.disabled = false;
     var toast = document.getElementById('save-toast');
     var msg = 'Save failed';
     if (err && err.errors && err.errors.length) {
       msg = err.errors.map(function (e) { return e.message; }).join('. ');
     }
     toast.textContent = msg;
     toast.className = 'save-toast save-toast-error';
     toast.classList.remove('hidden');
     setTimeout(function () { toast.classList.add('hidden'); }, 4000);
     // Do NOT call loadFields() — retain user's form edits so they can fix and retry
   });
   ```

Key behaviors per user decisions:
- Red toast on error (same element, different class) — 4 second display (longer than success for readability)
- Retain user's edits in the form on failure (no loadFields call in catch)
- Loading state: button text "Saving..." and disabled during request
- Button restored to original text on both success and failure
- Error messages joined from server's `errors[].message` array
