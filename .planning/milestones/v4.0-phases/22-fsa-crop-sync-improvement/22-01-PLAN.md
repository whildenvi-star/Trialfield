<objective>
Add an enterprise-level crop sync preview to the FSA acres app that fetches macro rollup data from farm-budget's /api/dashboard endpoint and displays a side-by-side comparison of FSA CLU acres vs farm-budget enterprise acres by crop — filtering out non-crop and reported CLUs.

Purpose: Currently the sync preview only does field-level matching. This adds a macro/enterprise view so the user can see at a glance whether their FSA crop acre totals line up with their farm-budget plan before drilling into individual field assignments.

Output: New server endpoint + frontend preview panel in the FSA Data tab
</objective>

Add a new GET endpoint `/api/sync-crops/enterprise-preview` in fsa-acres/server.js, placed immediately BEFORE the existing `/api/sync-crops/preview` route (around line 567). This endpoint:

1. Fetches farm-budget dashboard data using `cachedFetch('http://localhost:3001/api/dashboard')`. If null, return 502 with `{ error: 'Farm budget unavailable — is port 3001 running?' }`.

2. Builds budget-side crop totals from `dash.enterpriseSummaries[].cropRows[]`:
   - For each enterpriseSummary, iterate its `cropRows` array
   - Each cropRow has `{ crop, acres }` — use `normName(cr.crop)` as the grouping key
   - Sum `acres` across all enterprises for each normalized crop name
   - Track enterprise details: `{ name: es.enterprise.name, category: es.enterprise.category, acres: cr.acres }`
   - Store both `displayName` (original crop name from first encounter) and `budgetAcres` (sum)

3. Builds FSA-side filtered crop totals from `store.cluRecords`:
   - Apply ALL of these filters (exclude if any match):
     - `r.reported === true` (FSA-04)
     - `r.landClass` is in `['Grass/GLS', 'CRP', 'Hay/Forage', 'Idle', 'NC']`
     - `r.use === 'forage'`
     - Crop name (lowercased, trimmed) is in `['', 'nc', 'gls', 'crp', 'idle', 'mixed forage / hay', 'alfalfa', 'grass', 'intermediate wheatgrass']` (FSA-03)
   - For passing CLUs, group by `normName(r.crop)`, sum `r.fsaAcres`, count CLU records
   - Store `displayName` and `fsaAcres` and `cluCount`

4. Merges into comparison rows:
   - Collect all unique normalized keys from both budget and FSA sides
   - For each key, produce: `{ crop (display name), budgetAcres, fsaAcres, diff (fsa - budget), cluCount, enterprises[] }`
   - Use `Math.round(n * 100) / 100` for rounding (round2 is not exported from Calc)
   - Sort rows by `budgetAcres` descending

5. Returns `{ rows: [...], budgetGrandTotal: number, fsaGrandTotal: number }`:
   - `budgetGrandTotal` from `dash.grandTotals.acres` if available
   - `fsaGrandTotal` is the sum of all filtered FSA crop acres

Use the established error handling pattern: try/catch with 502 status on failure.

IMPORTANT: Do NOT use `cachedFetch` without a URL — it requires the full URL string. Do NOT modify the existing `/api/sync-crops/preview` endpoint. The `normName()` function already exists at line 534 of server.js — reuse it, do not duplicate it.
