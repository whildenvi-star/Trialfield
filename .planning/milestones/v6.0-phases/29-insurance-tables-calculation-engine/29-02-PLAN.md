  <action>
    **1. Create `glomalin-portal/src/app/api/insurance/aph-lookup/route.ts`** — GET handler for INS-05:
    - Auth check (same pattern as /api/fsa/clu-records)
    - Accept query params: `?crop=` and `?farmName=`
    - Query clu_records from Supabase: `.select('farm_name, farm_number, aph, fsa_acres').ilike('crop', '%' + crop + '%').eq('crop_year', 2026)`
    - Further filter results in TypeScript using `normName()` from `@/lib/insurance/calc` for farm_name substring matching (both directions: policy farmName contains CLU farm_name OR CLU farm_name contains policy farmName)
    - Call `computeAphFromClus()` on filtered results
    - Return: `{ avgAph, count, totalRecords, farmName, crop }` with HTTP 200
    - When count=0 but totalRecords>0: the response signals "CLU records found but no APH values set" (the UI in Phase 30 interprets this)
    - When totalRecords=0: the response signals "No matching CLU records found"

    **2. Create `glomalin-portal/src/app/api/insurance/yield-sync/route.ts`** — POST handler for INS-06:
    - Auth check
    - Accept JSON body: `{ policyId: string }`
    - Fetch the policy from Supabase: `.from('insurance_policies').select('*').eq('id', policyId).single()`
    - If not found, return 404
    - Cross-app fetch to grain-tickets: `fetch('http://localhost:3000/api/farms', { signal: AbortSignal.timeout(5000), next: { revalidate: 0 } as any })` — wrap in try/catch
    - On fetch failure (connection refused, timeout): return HTTP 200 with `{ error: 'Grain ticket service unavailable — is port 3000 running?', matched: false, policy: null }`
    - On success: parse response as `GrainFarm[]` (the /api/farms endpoint returns this shape)
    - Call `findBestGrainMatch()` from calc.ts with the policy and grain farm data
    - If score >= 2: update insurance_policies row — set `actual = match.yieldPerAcre`, `actual_synced_from_grain = true`
    - After updating actual: call `computeClaimAlert()` with the updated policy data, then update `claim_alert` column
    - Return: `{ matched: true, score, match: { farm, crop, yieldPerAcre, ticketCount }, policy: updatedPolicy }` or `{ matched: false, score, bestMatch: null, policy: currentPolicy }`
  </action>
