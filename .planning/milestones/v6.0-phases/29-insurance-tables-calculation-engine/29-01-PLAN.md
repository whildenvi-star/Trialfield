      ```sql
      ALTER TABLE insurance_policies
        ADD COLUMN IF NOT EXISTS aph_computed              numeric(10,2),
        ADD COLUMN IF NOT EXISTS aph_clu_count             integer,
        ADD COLUMN IF NOT EXISTS actual_synced_from_grain  boolean NOT NULL DEFAULT false,

      ```
    - Adds authenticated_write RLS policy if not exists (use DO $$ block checking pg_policies):
      ```sql
      DO $$
      BEGIN
        IF NOT EXISTS (
          SELECT 1 FROM pg_policies WHERE tablename = 'insurance_policies' AND policyname = 'authenticated_write'
        ) THEN
          CREATE POLICY "authenticated_write" ON insurance_policies
            FOR ALL USING (auth.role() = 'authenticated');
        END IF;
      END$$;
      ```
    - Prints the SQL to console (same pattern as migrate-fsa.ts) so user can also run it manually in Supabase SQL editor

    **2. Create `glomalin-portal/src/lib/insurance/calc.ts`** — pure TypeScript calc engine. Port from fsa-acres/public/insurance.js reference implementations. Exports:

    - `normName(n: string | null): string` — lowercase, strip non-alphanumeric except spaces, collapse whitespace, trim (11 lines from insurance.js)
    - `computeAphFromClus(clus: { aph: number | null; fsa_acres: number | null }[]): { avgAph: number; count: number; totalRecords: number }` — simple average of non-zero aph values. Returns count=0 when no CLUs have aph>0. Distinguishes "no matching CLUs" (totalRecords=0) from "CLUs found but no APH values" (totalRecords>0, count=0)
    - `computeClaimAlert(policy: { guarantee: number; actual: number; coverage_level: number }): 'none' | 'potential'` — returns 'potential' when actual > 0 AND guarantee > 0 AND actual < (guarantee * coverage_level / 100). If guarantee=0 or actual=0, returns 'none' (avoids false alerts from corrupt data like ins_482)
    - `findBestGrainMatch(policy: { farm_name: string | null; crop: string | null }, gtFarms: GrainFarm[]): { match: GrainFarm | null; score: number }` — port from insurance.js lines 514-535. Score: 3=farm+crop, 2=crop only, 1=farm only, 0=none. Only score>=2 should be applied by callers.
    - `GrainFarm` interface: `{ farm: string; crop: string; acres: number; totalBU: number; yieldPerAcre: number; ticketCount: number }`

    Use `Array.from()` for any Set/Map iteration (TypeScript pitfall from Phase 27-02). Keep functions pure — no Supabase calls, no side effects. Use `round2` helper: `(n: number) => Math.round(n * 100) / 100`.
