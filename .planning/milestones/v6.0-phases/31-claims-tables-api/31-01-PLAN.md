**migrate-31.ts** — Follow exact pattern from `scripts/migrate-29.ts` (inline .env.local parser, service_role client, print SQL to console, attempt exec_sql RPC, verify columns):

SQL to generate (all idempotent):
1. Create `claim_stage` enum via DO/EXCEPTION block (not `IF NOT EXISTS`):
   ```sql
   DO $$ BEGIN
     CREATE TYPE claim_stage AS ENUM ('notice_of_loss','filed','adjuster_assigned','under_review','settled','closed');
   EXCEPTION WHEN duplicate_object THEN NULL;
   END$$;
   ```

2. `CREATE TABLE IF NOT EXISTS claims` with columns per research Pattern 1:
   - id (uuid PK default gen_random_uuid())
   - policy_id (uuid NOT NULL REFERENCES insurance_policies(id) ON DELETE CASCADE) — NOT unique (multi-claim per policy)
   - clu_record_id (uuid REFERENCES clu_records(id) ON DELETE SET NULL) — optional link for Phase 33
   - stage (claim_stage NOT NULL DEFAULT 'notice_of_loss')
   - stage_entered_at (timestamptz NOT NULL DEFAULT now())
   - crop (text), coverage_type (text), coverage_level (integer), effective_guarantee (numeric(10,2))
   - date_of_loss (date), cause_of_loss (text), description (text)
   - Financial: estimated_loss_bu (numeric(10,2)), appraised_value (numeric(12,2)), indemnity_amount (numeric(12,2)), deductible_amount (numeric(12,2))
   - Deadline: deadline_at (timestamptz), deadline_overridden (boolean NOT NULL DEFAULT false)
   - Adjuster: adjuster_name (text), adjuster_phone (text)
   - notes (text), created_at (timestamptz NOT NULL DEFAULT now()), updated_at (timestamptz NOT NULL DEFAULT now())

3. `CREATE TABLE IF NOT EXISTS claim_documents` per research Pattern 1:
   - id (uuid PK), claim_id (uuid NOT NULL REFERENCES claims(id) ON DELETE CASCADE)
   - storage_path (text NOT NULL), filename (text NOT NULL), file_size (integer), mime_type (text)
   - category (text) — 'notice_of_loss' | 'adjuster_report' | 'photos' | 'settlement' | 'other'
   - uploaded_by (uuid REFERENCES profiles(id) ON DELETE SET NULL)
   - created_at (timestamptz NOT NULL DEFAULT now())

4. `CREATE TABLE IF NOT EXISTS claim_timeline` per research Pattern 1:
   - id (uuid PK), claim_id (uuid NOT NULL REFERENCES claims(id) ON DELETE CASCADE)
   - event_type (text NOT NULL), event_data (jsonb), note (text)
   - actor_id (uuid REFERENCES profiles(id) ON DELETE SET NULL)
   - created_at (timestamptz NOT NULL DEFAULT now())
   - NO updated_at — immutable append-only

5. RLS — enable on all 3 tables + authenticated_all policy (same pattern as insurance_policies authenticated_write):
   ```sql
   ALTER TABLE claims ENABLE ROW LEVEL SECURITY;
   CREATE POLICY "authenticated_all" ON claims FOR ALL USING (auth.role() = 'authenticated');
   ```
   Same for claim_documents and claim_timeline. Wrap each in DO/IF NOT EXISTS block.

6. Storage bucket creation via JS client: `supabase.storage.createBucket('claim-documents', { public: false })` wrapped in try/catch ignoring duplicate. Print manual fallback instructions.

7. Storage objects RLS — 3 policies scoped to `bucket_id = 'claim-documents'`:
   - authenticated_upload (INSERT), authenticated_select (SELECT), authenticated_delete (DELETE)
   - Each wrapped in DO/IF NOT EXISTS block checking pg_policies on storage.objects

8. `updated_at` trigger on claims table:
   ```sql
   CREATE TRIGGER claims_updated_at BEFORE UPDATE ON claims FOR EACH ROW EXECUTE FUNCTION set_updated_at();
   ```
   Wrap in DO/IF NOT EXISTS.

9. Indexes: `CREATE INDEX IF NOT EXISTS claims_policy_id_idx ON claims(policy_id)`, `CREATE INDEX IF NOT EXISTS claims_stage_idx ON claims(stage)`, `CREATE INDEX IF NOT EXISTS claim_documents_claim_id_idx ON claim_documents(claim_id)`, `CREATE INDEX IF NOT EXISTS claim_timeline_claim_id_idx ON claim_timeline(claim_id)`.

**POST /api/claims (create-from-policy — CLM-07):**
- Auth check: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser()` — 401 if not authenticated
- Body: `{ policy_id: string, date_of_loss: string, description: string }`
- Fetch policy: `supabase.from('insurance_policies').select('crop, plan_type, coverage_level, guarantee, farm_name').eq('id', policy_id).single()` — 404 if not found
- Compute `effective_guarantee = (policy.guarantee ?? 0) * ((policy.coverage_level ?? 75) / 100)`
- Insert claim with: policy_id, stage='notice_of_loss', stage_entered_at=now, crop=policy.crop, coverage_type=policy.plan_type, coverage_level=policy.coverage_level, effective_guarantee, date_of_loss, description, deadline_at=addDays(new Date(date_of_loss), INITIAL_DEADLINE_DAYS)
- Insert timeline event: event_type='created', event_data={policy_id, stage:'notice_of_loss'}, actor_id=user.id
- Return 201 with `{ claim }`

**GET /api/claims:**
- Auth check
- Query params: `?year=` (optional, filter by extract(year from date_of_loss) or created_at year)
- Select claims ordered by created_at desc, join with insurance_policies for policy display info if useful
- Return `{ claims, count }`

**GET /api/claims/[id]:**
- Auth check, `const { id } = await params` (Next.js 15 Promise params)
- Fetch single claim by id — 404 if not found
- Return `{ claim }`

**PATCH /api/claims/[id]:**
- Auth check, await params
- Fetch current claim first (per Phase 29-02 decision pattern: fetch before recompute)
- Accept partial body: stage, financial fields, adjuster fields, deadline_at, deadline_overridden, notes, cause_of_loss, etc.
- **Stage transition logic:** If `patch.stage && patch.stage !== currentClaim.stage`:
  - Set `stage_entered_at = new Date().toISOString()`
  - If `!patch.deadline_overridden`, recalculate deadline from STAGE_DEADLINE_DAYS map
  - Insert timeline event: event_type='stage_change', event_data={from_stage, to_stage}, actor_id
- **Financial update:** If any financial field changed, insert timeline event: event_type='financial_update', event_data with changed fields
- **Deadline override:** If deadline_at changed and deadline_overridden=true, insert timeline event: event_type='deadline_change'
- **Adjuster assignment:** If adjuster_name set from null/empty, insert timeline event: event_type='adjuster_assigned'
- Update claim, return `{ claim }`

**DELETE /api/claims/[id]:**
- Auth check, await params
- Delete claim (CASCADE deletes documents + timeline)
- Return 200 with `{ deleted: true }`

**GET /api/claims/[id]/timeline:**
- Auth check, await params
- Select from claim_timeline where claim_id=id, order by created_at asc
- Return `{ events, count }`

**POST /api/claims/[id]/timeline:**
- Auth check, await params
- Body: `{ note: string }` — manual note addition
- Insert: event_type='note', note=body.note, actor_id=user.id
- Return 201 with `{ event }`
