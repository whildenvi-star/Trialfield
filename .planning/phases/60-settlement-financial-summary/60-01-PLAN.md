  <action>
Add a new GET /api/settlement-summary endpoint in grain-tickets/server.js (place it after the existing /api/settlement-prices endpoint around line 953).

The endpoint must:

1. **Accept query params**: `?cropYear=YYYY` (default: current harvest year using same logic as getCropYear — if month 1-5, use prior year).

2. **Query settlement lines**: Use Prisma to fetch all SettlementLines where:
   - `settlement.cropYear` equals the requested crop year
   - `matchStatus` is 'matched' or 'manual' (same filter as existing settlement-prices)
   - `ticketId` is not null (line is linked to a farm ticket)
   Include relations: `ticket` (select: `crop`, `buyerId`), `settlement` (select: `buyer` with `name` and `id`).
   Also select: `netBushels`, `price`, `deductions`, `netPayment`.

3. **Aggregate per buyer+crop**: Group the lines by `settlement.buyer.name` + `ticket.crop`. For each group compute:
   - `buyerName`: string
   - `buyerId`: number (from settlement.buyer.id)
   - `crop`: string
   - `deliveredBushels`: sum of netBushels (parseFloat, skip nulls)
   - `avgPricePerBushel`: weighted average price = totalNetPaymentBeforeDeductions / deliveredBushels. If deliveredBushels is 0, use simple average of price values.
   - `totalDeductions`: sum of deductions (parseFloat, skip nulls)
   - `netPayment`: sum of netPayment (parseFloat, skip nulls)
   - `lineCount`: number of settlement lines in this group

4. **Fetch contract prices from portal**: Use `fetch` to call the portal contracts API:
   - URL: `${portalUrl}/api/marketing/contracts?year=${cropYear}` where `portalUrl` comes from `process.env.PORTAL_ORIGIN || 'http://localhost:3010'` (reuse the existing portalUrl pattern already in server.js at line ~544).
   - Wrap in try/catch with AbortSignal.timeout(5000) — if portal is down, proceed with no contract data (graceful degradation).
   - Parse the response JSON: `{ contracts: GrainContract[] }`.

5. **Join contract prices to summary rows**: For each summary row, find matching contracts from the portal response where `contract.buyer` (case-insensitive) matches `buyerName` AND `contract.crop` (case-insensitive) matches `crop`. If multiple contracts match, use the weighted average price (sum of bushels*price / sum of bushels) from contracts that have a `price_per_bushel`. Set:
   - `contractPricePerBushel`: number | null — the weighted avg contract price
   - `contractedBushels`: number | null — total contracted bushels for this buyer+crop
   - `priceVariance`: number | null — `avgPricePerBushel - contractPricePerBushel` (positive = settlement paid more than contract, negative = paid less). Null if no contract match.

6. **Return response**: `{ summary: SummaryRow[], cropYear: number, contractsAvailable: boolean }` where `contractsAvailable` indicates whether the portal fetch succeeded.

Round all monetary values to 2 decimal places and price values to 4 decimal places in the response.

**Pattern reference**: Follow the existing /api/settlement-prices endpoint style (same Prisma query pattern, same error handling with try/catch + console.error + 500 response).
