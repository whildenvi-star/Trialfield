<feature>
  <name>Reconciliation Matching Engine and API</name>
  <files>grain-tickets/server.js</files>
  <behavior>
    **normalizeTicketNo(ticketNo):**
    - Input: "H066666" → Output: "66666"
    - Input: "066666" → Output: "66666"
    - Input: "66666" → Output: "66666"
    - Input: "h066666" → Output: "66666"
    - Input: null → Output: null
    - Input: "0" → Output: null
    - Input: "" → Output: null
    - Input: "H000" → Output: null (all zeros after H strip)

    **runMatch(settlementId):**
    - Fetches settlement with buyerId + cropYear
    - Loads farm tickets scoped to that buyerId + cropYear (NOT global)
    - Builds normalized ticket number lookup map
    - Iterates settlement lines, comparing normalized numbers
    - Skips lines with matchStatus 'manual' or 'disputed' (preserve user flags)
    - Updates matching lines: ticketId = farmTicket.id, matchStatus = 'matched'
    - Updates non-matching lines: ticketId = null, matchStatus = 'unmatched'
    - Returns { matched, unmatched } counts

    **Auto-match integration:**
    - Called at end of POST /api/settlements/:id/commit after createMany succeeds
    - Response includes matched + unmatched counts alongside linesCreated

    **POST /api/settlements/:id/rematch:**
    - Calls runMatch(settlementId), returns { matched, unmatched }

    **GET /api/tickets enrichment:**
    - Prisma query adds: include: { settlementLines: { select: { matchStatus: true } } }
    - dbTicketToJson adds _reconciliation: { status, lineCount }
    - Status derivation priority: disputed > manual > matched > unreconciled

    **GET /api/tickets/:id enrichment:**
    - Same include and _reconciliation enrichment as list endpoint

    **GET /api/reconciliation/summary?buyerId=X&cropYear=Y:**
    - Aggregates farm ticket netWeight per crop for given buyer+cropYear
    - Aggregates settlement line netWeight per crop (via matched ticket's crop)
    - Returns rows: { crop, farmLbs, farmCount, buyerLbs, buyerBushels, varianceLbs, variancePct }
    - varianceLbs = farmLbs - buyerLbs (positive = farm weighed more = potential underpayment)
    - variancePct = (varianceLbs / farmLbs) * 100, rounded to 2 decimal places

    **GET /api/reconciliation/unmatched?buyerId=X&cropYear=Y:**
    - farmOnly: tickets with buyerId+cropYear that have no matched/manual settlement line
    - settlementOnly: settlement lines with matchStatus 'unmatched' for buyer+cropYear
    - Each farmOnly ticket includes hint: "No settlement for [buyer]" if zero settlements exist, else "Ticket# not in settlement"
    - Returns { farmOnly: [...], settlementOnly: [...] }

    **POST /api/reconciliation/manual-link:**
    - Body: { ticketId, settlementLineId }
    - Updates settlement line: ticketId = given, matchStatus = 'manual'
    - Returns updated settlement line

    **PATCH /api/settlement-lines/:lineId/dispute:**
    - Only allowed when current matchStatus is 'matched', 'manual', or 'disputed'
    - Sets matchStatus = 'disputed', writes notes from request body
    - Returns updated settlement line
  </behavior>

       - Returns { matched, unmatched }
