  <action>
**POST /api/claims/[id]/upload-url** — Generates a signed upload URL for client-side file upload:
- Auth check: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser()` — 401 if not authenticated
- `const { id } = await params` (Next.js 15 Promise params pattern)
- Body: `{ filename: string, mimeType: string }`
- Validate file type: allow `application/pdf`, `image/jpeg`, `image/png`, `image/webp`, `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` (XLSX), `text/csv`. Return 400 for unsupported types.
- Verify claim exists: `supabase.from('claims').select('id').eq('id', id).single()` — 404 if not found
- Build storage path: `claims/${id}/${Date.now()}-${sanitizedFilename}` where sanitizedFilename replaces spaces with hyphens and strips non-alphanumeric characters (except dots and hyphens)
- Generate signed URL: `supabase.storage.from('claim-documents').createSignedUploadUrl(path)` — 500 if error
- Return 200 with `{ path, token: data.token, signedUrl: data.signedUrl }`

**Note on file size:** Supabase Storage enforces its own max file size (default 50MB per object). No need to check size server-side — Storage will reject oversized files. Phase 32 UI can add a client-side 25MB check for better UX.

**POST /api/claims/[id]/documents** — Saves document metadata after client upload completes:
- Auth check, await params
- Body: `{ storagePath: string, filename: string, fileSize: number, mimeType: string, category?: string }`
- Validate category if provided: must be one of 'notice_of_loss', 'adjuster_report', 'photos', 'settlement', 'other'. Default to 'other' if omitted.
- Insert into claim_documents: claim_id=id, storage_path=storagePath, filename, file_size=fileSize, mime_type=mimeType, category, uploaded_by=user.id
- Insert timeline event: event_type='doc_upload', event_data={filename, category, storage_path: storagePath}, actor_id=user.id, claim_id=id
- Return 201 with `{ document }`

**GET /api/claims/[id]/documents** — Lists documents with signed download URLs:
- Auth check, await params
- Fetch all claim_documents where claim_id=id, ordered by created_at desc
- For each document, generate a signed download URL: `supabase.storage.from('claim-documents').createSignedUrl(doc.storage_path, 3600)` — 1 hour expiry
- Map documents to include `signedUrl` alongside existing fields
- Return `{ documents, count }`

**DELETE /api/claims/[id]/documents** — NOT implemented in this plan. Phase 32 can add a DELETE endpoint for individual documents if needed. Document deletion cascades with claim deletion via FK ON DELETE CASCADE.

All route handlers: import `createClient` from `@/lib/supabase/server`, use `NextResponse.json()`, standard auth pattern.
