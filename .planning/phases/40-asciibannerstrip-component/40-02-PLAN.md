**In ascii-noise.ts — enhance MyceliumNode and generateMycelium:**

1. **Node lifecycle model:** Add fields to MyceliumNode:
   - `birthTime: number` — when this node appeared (in animation time units)
   - `lifespan: number` — how long it lives (8-15 seconds, randomized)
   - `phase: 'growing' | 'active' | 'fading'` — derived from age vs lifespan

   Add a `tickNodes(nodes, time, cols, rows)` function that:
   - Checks each node's age (`time - birthTime`)
   - First 2s = growing (brightness ramps up), middle = active (full brightness), last 2s = fading (brightness ramps down)
   - When a node finishes fading, respawn it at a new random position with a new birthTime = time
   - This creates the "nodes bloom, persist, fade, new ones appear elsewhere" lifecycle

2. **Tendril growth model:** Replace the current static-edge drawing with growing tendrils:
   - Each edge has a `growthProgress: number` (0..1) that slowly advances over 8-12 seconds
   - Draw the tendril from node A only up to `growthProgress` along the path to node B
   - After fully grown, hold for a few seconds, then retract (growthProgress decreases back to 0)
   - On retraction completion, reassign the edge to connect to different nearby nodes
   - Add noise-based jitter perpendicular to the tendril path (slight displacement, not smooth curves) for the "digital/glitchy" feel described in context
   - Occasional forks: when growthProgress > 0.5, draw 2-3 smaller branch paths that fork off at 20-40 degree angles and fade out after a short distance

3. **Sparseness:** Reduce background fbm texture opacity to near-zero (0.03-0.05 max) so tendrils stand out against mostly-black negative space. The aesthetic is "lots of dark negative space" per context.

**In ASCIIBannerStrip.tsx — rendering enhancements:**

4. **White highlights:** Update the `charColor` function (or override in the render loop):
   - When brightness > 0.85, use `'#ffffff'` (white) instead of cyan — these are the "momentary white peaks" for the brightest nodes
   - This creates sparkle/flash effect at node centers

5. **Clock-based tab resume:** Change the animation loop to use wall-clock time:
   - Instead of using the RAF timestamp directly, compute `animationTime = Date.now() * 0.001 + timeOffset`
   - This means when the user switches away and comes back, the animation has "progressed" to where it would be — feels like it kept running in the background
   - Keep the RAF 50fps throttle for frame pacing, but the time input to render() should be clock-based

6. **Performance:** The tendril growth calculations must stay fast:
   - Pre-allocate growth state arrays alongside edges (not per-frame allocation)
   - The existing Float32Array grid approach is good — keep it
   - Tendril fork drawing: max 3 forks per edge, each max 8 steps — bounded work

**Visual targets from locked decisions:**
- 8-12 visible nodes at any time (some growing, some fading = actual visible count varies)
- Moderate speed: 8-12 second growth/retraction cycle
- Organic & sparse — few branching tendrils with lots of dark negative space
- Jittery straight paths (not smooth curves)
- High contrast — dim tendrils barely visible, nodes glow bright
