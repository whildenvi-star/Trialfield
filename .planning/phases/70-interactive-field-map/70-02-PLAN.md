<task type="auto">
  <name>Task 2: POST /api/maps/import — GeoJSON/KML intake, field matching, upsert, summary</name>

  <action>
Create `glomalin-portal/src/app/api/maps/import/route.ts`.

**Auth:** Admin-only. Check user session and profile role. Return 403 if not admin.

**Input:** `multipart/form-data` with a single file field named `file`. Accept `.geojson`, `.json`, `.kml` file extensions.

**GeoJSON parsing:**
- For `.geojson`/`.json`: `JSON.parse(text)` — expect either a FeatureCollection or a single Feature
- For `.kml`: Parse XML using the built-in `DOMParser` (available in Node 18+ via global or use a simple regex/string extraction). KML feature extraction pattern: look for `<Placemark>` elements, extract `<name>` as the field name and `<coordinates>` to build a GeoJSON Polygon geometry. KML support is best-effort — if parsing fails, return `{ error: "KML parse failed", detail: "..." }` with 422.

**Field matching:**
1. Fetch registry fields: `fetchRegistryService('/api/fields')` — import the existing proxy helper from `@/app/api/mobile/_lib/proxy`
2. Build a name→field lookup map (lowercase trim both sides)
3. For each incoming GeoJSON feature, attempt name match via `feature.properties.Name` or `feature.properties.name` or `feature.properties.FIELD_NAME` (SMS exports vary). Try all three property keys.

**Centroid computation:**
For each matched feature, compute polygon centroid from the geometry coordinates:
- For a Polygon: average all coordinate pairs in the outer ring (index 0)
- For a MultiPolygon: average all outer ring coordinates across all polygons
- Formula: `avg_lat = sum(coords.map(c => c[1])) / count`, `avg_lng = sum(coords.map(c => c[0])) / count`

**Upsert to Supabase:**
Use service role client (not session client) for upserts — bypasses RLS. Check if `process.env.SUPABASE_SERVICE_ROLE_KEY` is available; if not, fall back to session client.

Use `supabase.from('field_boundaries').upsert(rows, { onConflict: 'registry_field_id' })`.

Each upserted row: `{ registry_field_id: field.id, name: field.name, geojson: feature.geometry, centroid_lat, centroid_lng }`.

**Farm center update:**
After all upserts succeed, compute the farm center as the average of all centroid_lat/centroid_lng values across all matched features. Store in `farm_map_config` with key `'farm_center'` and value `{ lat, lng, zoom: 13 }`.

**Summary response:**
```typescript
{
  matched: number,      // features matched to registry fields
  updated: number,      // rows upserted (same as matched if all succeed)
  unmatched: string[],  // feature names that had no registry match
  noGeometry: string[], // registry fields with no incoming boundary
  farmCenter: { lat: number, lng: number } | null
}
```

`noGeometry` = registry field names that were NOT in the incoming file (already have or never had a boundary).

Return 200 with the summary. Return 400 if file is missing or unreadable.

Do NOT use any external GeoJSON/KML parsing library — use built-in JSON.parse for GeoJSON and basic string/DOM operations for KML. No new npm dependencies.
