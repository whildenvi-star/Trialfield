**sw.js — extend existing service worker (currently v6, network-first for GET, skips API):**

1. Bump CACHE_NAME to `grain-tickets-v7`

2. Add IndexedDB helpers inside sw.js (raw IDB API — same pattern as Phase 47 portal sw.ts, no library available in SW context):
   - `openTicketDB()` — opens `grain-tickets-offline` DB (version 1) with two object stores:
     - `ticket-queue` (keyPath: `id`, autoIncrement: false) — pending ticket entries
     - `ref-cache` (keyPath: `key`) — cached farms/crops/buyers lists for offline form validation
   - `getAllPending()` — reads all from `ticket-queue` store
   - `deletePending(id)` — removes one entry by id
   - `updatePending(id, data)` — updates retryCount/status fields

3. Add `sync` event listener for tag `'ticket-sync'`:
   - Open IDB, get all pending tickets
   - For each pending ticket (FIFO by `createdAt`):
     - POST to `/api/tickets` with ticket body
     - On 201: delete from queue, increment synced count
     - On 409 (duplicate): mark ticket status as `'conflict'`, store server response (existing ticket details) in `conflictData` field — do NOT delete from queue
     - On 4xx: mark as `'failed'` with error message
     - On network error / 5xx: increment `retryCount`, mark `'failed'` if retryCount >= 3
   - After processing: post message to all clients `{ type: 'ticket-sync-complete', synced, conflicts, failed }`

4. Add fetch handler for API responses (extend existing fetch listener):
   - For GET requests to `/api/farms`, `/api/registry/crops`, `/api/buyers` — cache responses in Cache API (stale-while-revalidate)
   - These cached responses serve the form dropdowns when offline

**app.js — add offline banner and sync event wiring:**

1. Add offline detection after existing tab navigation code:
   - Create offline banner element: thin bar at top, text "You're offline — changes will sync when connected", amber/yellow background
   - `window.addEventListener('online', ...)` — hide banner, dispatch custom event `'app-online'`
   - `window.addEventListener('offline', ...)` — show banner
   - On page load: check `navigator.onLine` and show/hide accordingly

2. Add service worker message listener:
   - On `'ticket-sync-complete'` message: show auto-dismissing toast "Synced N tickets" (4s), dispatch custom event `'tickets-synced'` so ticket list can refresh

3. Add helper `window.ticketQueue` object exposing:
   - `add(ticketData)` — opens IDB, generates UUID id, stores `{ id, body: ticketData, status: 'pending', createdAt: Date.now(), retryCount: 0 }`
   - `getAll()` — returns all entries from `ticket-queue` store
   - `getPending()` — filters to status === 'pending'
   - `getConflicts()` — filters to status === 'conflict'
   - `delete(id)` — removes entry
   - `update(id, fields)` — partial update (for editing pending tickets)
   - `requestSync()` — calls `navigator.serviceWorker.ready.then(reg => reg.sync.register('ticket-sync'))` with fallback manual sync if Background Sync API unavailable
