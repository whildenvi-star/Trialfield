  <files>glomalin-portal/src/components/insurance/aph-panel.tsx</files>
  <action>
    Create a 'use client' component `AphPanel` with props: `{ policyId: string; coverageLevel: number; onGuaranteeChange?: (guarantee: number) => void }`.

    **Data fetching:** On mount (and when policyId changes), fetch GET /api/insurance/aph?policyId={policyId}. Store records, computedAph, guarantee, includedCount, excludedCount in state.

    **Year table:** Display records sorted by crop_year DESC in a table:
    - Columns: Year | Actual Yield (bu/ac) | Source | Disaster Year | Actions
    - Source column shows a badge: 'manual' = muted gray badge, 'grain-tickets' = green "GT" badge (same style as the GT badge in the policy table), 'import' = blue badge
    - Disaster Year column: toggle checkbox. When toggled, call PATCH /api/insurance/aph/[id] with { is_disaster_year: !current }. On success, refetch APH data. Disaster year rows shown with strikethrough on the yield value and muted text color.
    - Actions column: small red "x" delete button. On click, confirm("Remove {year} APH record?") then DELETE /api/insurance/aph/[id], refetch on success.

    **Computed APH display:** Above the table, show a summary row:
    - "Computed APH: {computedAph} bu/ac" in large accent text
    - "(from {includedCount} of {totalCount} years, {excludedCount} excluded)" in muted text
    - "Guarantee at {coverageLevel}%: {guarantee} bu/ac" in secondary text
    - Show the formula: "APH = avg({included yields joined by +}) = {computedAph}"

    **Add year form:** Below the table, an inline row with:
    - Number input for crop_year (default: current year - 1, constrained to reasonable range 2010-2026)
    - Number input for actual_yield (bu/ac, step 0.1)
    - Select for source: manual (default), grain-tickets, import
    - "Add Year" button that calls POST /api/insurance/aph with { policy_id: policyId, crop_year, actual_yield, source }
    - On 409 (duplicate year), show inline error text "Year already exists"
    - On success, refetch APH data and clear the form

    **Guarantee callback:** When computedAph or guarantee changes after any CRUD operation, call onGuaranteeChange(guarantee) so the parent can update the policy table display.

    **Styling:** Follow glomalin design tokens throughout. Use font-mono, glomalin-accent for headings, glomalin-surface for table background, glomalin-border for borders. Match the existing insurance table styling from insurance-workspace.tsx.
