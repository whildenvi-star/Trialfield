  <action>
Add a `propagateField(field)` async function and a `propagationLog` in-memory array (capped at 100 entries) near the top of server.js, after the existing constants.

**Environment variables** (read at startup, with defaults):
- `FARM_BUDGET_URL` — default `http://localhost:3001`
- `GRAIN_TICKETS_URL` — default `http://localhost:3007`
- `PORTAL_URL` — default `http://localhost:3010`
- `EMBED_TOKEN` — already exists in the codebase, reuse it for auth on farm-budget and grain-tickets API calls

**propagateField(field) function:**
1. Build an array of 3 targets:
   - `{ name: 'farm-budget', url: FARM_BUDGET_URL + '/api/fields' + (EMBED_TOKEN ? '?token=' + encodeURIComponent(EMBED_TOKEN) : ''), body: { name: field.name, acres: field.reportingAcres || 0, registryFieldId: field.id, registryFieldName: field.name } }`
   - `{ name: 'grain-tickets', url: GRAIN_TICKETS_URL + '/api/farms' + (EMBED_TOKEN ? '?token=' + encodeURIComponent(EMBED_TOKEN) : ''), body: { farm: field.name, acres: field.reportingAcres || 0, registryId: field.id } }`
   - `{ name: 'portal', url: PORTAL_URL + '/api/fsa/webhook/field-created', body: { field_name: field.name, fsa_acres: field.reportingAcres || 0, registry_field_id: field.id } }` (portal uses its own auth — the webhook endpoint will handle this separately in Plan 02)
2. For each target, use `Promise.allSettled` to fire all 3 in parallel:
   - `fetch(target.url, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(target.body), signal: AbortSignal.timeout(5000) })`
   - On success (2xx): record `{ target: name, status: 'ok', timestamp: new Date().toISOString() }`
   - On failure: log to console.error, record `{ target: name, status: 'failed', error: message, timestamp }`, then **retry once** after 3 seconds using setTimeout:
     - The retry uses the same fetch call
     - On retry success: update the log entry status to 'ok (retry)'
     - On retry failure: update to 'failed (retry exhausted)', log to console.error
3. Return the results array (one entry per target with status).
4. Push each result into `propagationLog` (shift oldest if length > 100).

**Modify POST /api/fields handler (line ~245):**
- After `res.status(201).json(field)`, call `propagateField(field).catch(err => console.error('Propagation error:', err))` — fire-and-forget, do NOT await before responding.
- The 201 response is unchanged — propagation is fully async.

**Add GET /api/propagation-log endpoint:**
- Returns `{ log: propagationLog }` — useful for debugging and for the UI to show "Propagation pending" indicators.
- Protected by existing EMBED_TOKEN auth gate (already applied to all /api routes).

Important: Do NOT modify the existing POST handler's response format. The field creation must succeed independently of propagation. The propagation is best-effort with one retry.
