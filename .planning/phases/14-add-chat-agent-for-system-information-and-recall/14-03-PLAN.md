  <action>
Add a new "Glomalin Notes" section to admin.html following the existing pattern used by the "Grain Bins" and "Buyers" sections. Place it AFTER the existing "Column Mapping" section.

**Section structure:**

1. **Section header:** `<h2>Glomalin Notes</h2>` with a brief description: "Manage facts that Glomalin remembers across conversations."

2. **Filter bar:**
   - Category dropdown: All, Farm, Crop, Buyer, General
   - Active/Inactive toggle: Show Active (default), Show All
   - "Add Note" button (primary style)

3. **Notes table:**
   - Columns: Content, Category, Source, Active, Created, Actions
   - Each row displays one AgentNote record
   - "Source" shows "agent" (learned in chat) or "admin" (created here) with different badge styling
   - "Active" column: toggle switch (calls PUT /api/agent/notes/:id with active: true/false)
   - "Actions" column: Edit and Delete buttons
   - Empty state: "No notes yet. Glomalin will learn facts during conversations, or you can add them here."

4. **Add/Edit form (inline or modal):**
   - Content textarea (required, multi-line)
   - Category select: farm, crop, buyer, general
   - Save button (POST for new, PUT for edit)
   - Cancel button (hides form)

**JavaScript behavior (inline in admin.html, following existing patterns):**

- `loadNotes()` — fetch GET /api/agent/notes (with category and active query params from filter state), render table
- `createNote(content, category)` — fetch POST /api/agent/notes, reload table
- `updateNote(id, data)` — fetch PUT /api/agent/notes/:id, reload table
- `deleteNote(id)` — confirm dialog, then fetch DELETE /api/agent/notes/:id, reload table
- `toggleNoteActive(id, active)` — fetch PUT /api/agent/notes/:id with {active}, reload table
- Category filter: on change, re-call loadNotes() with updated query params
- Active filter: on change, re-call loadNotes() with active param (or omit for "all")
- Double-click content cell to inline edit (following the pattern used in Grain Bins section)

**Styling:** Use existing admin.html table styles. The section should look consistent with Grain Bins and Buyers sections. No new CSS file needed — admin.html uses inline styles or extends existing patterns.

**On page load:** Call `loadNotes()` to populate the table.

**Important:** Do NOT modify any existing sections in admin.html (Grain Bins, Buyers, Column Mapping). Only ADD the new section below them.
