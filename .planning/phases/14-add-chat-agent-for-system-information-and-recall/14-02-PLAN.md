**Create grain-tickets/public/glomalin.js** — a self-contained IIFE with these capabilities:

**Guard:** At the top, check `if (!window.GLOMALIN_ENABLED) return;` — if false, the entire IIFE exits and no DOM is created.

**State management:**
- `conversationHistory = []` — array of `{role, content}` objects sent with each request. Persists across popup open/close (same page load).
- `sessionKey` — random UUID generated once per page load (used for conversation logging).
- `isOpen = false`, `isStreaming = false` — UI state flags.

**DOM construction (all done in JS, no HTML template):**

1. **Floating button** (`#glomalin-btn`):
   - Fixed position, bottom-right corner (bottom: 24px, right: 24px)
   - Contains a tractor emoji or SVG tractor icon. Use the Unicode tractor character or an inline SVG of a simple tractor silhouette. The user specified "farm-themed tractor icon."
   - Green status dot (`.glomalin-dot`) — a small 8px green circle positioned absolute top-right of the button, visible when agent is enabled.
   - Click toggles popup visibility.

2. **Popup container** (`#glomalin-popup`):
   - Fixed position, starts at bottom: 80px, right: 24px
   - Default size: width 400px, height 500px
   - CSS `resize: both; overflow: hidden;` — native resize handle
   - Min-width: 320px, min-height: 300px, max-width: 90vw, max-height: 85vh
   - Border-radius, box-shadow, dark background matching grain-tickets theme (use CSS variables from style.css: --bg, --surface, --text, --primary, --border)
   - z-index: 10000 (above everything)
   - `display: none` initially

3. **Header bar** (`#glomalin-header`):
   - Title "Glomalin" on the left
   - "Clear" button and "X" close button on the right
   - Draggable: mousedown on header starts drag. Track offsetX/offsetY on mousedown, update popup left/top on mousemove, stop on mouseup. Prevent drag when clicking buttons.
   - Cursor: grab (default), grabbing (while dragging)

4. **Messages area** (`#glomalin-messages`):
   - Scrollable container (overflow-y: auto) filling available space between header and input
   - Each message is a div with class `.glomalin-msg.user` or `.glomalin-msg.assistant`
   - User messages: right-aligned, accent background
   - Assistant messages: left-aligned, surface background
   - Auto-scroll to bottom on new message

5. **Input area** (`#glomalin-input-row`):
   - Textarea (`#glomalin-input`) with placeholder "Ask about your grain data..."
   - Send button (`#glomalin-send`)
   - Enter key sends (without shift). Shift+Enter for newline.
   - Disable textarea and send button while streaming.

**ASCII tractor loading animation:**
When waiting for a response (isStreaming = true), show a loading indicator inside a new assistant message bubble. The indicator is a pre-formatted ASCII art tractor with animated exhaust. Use a simple multi-line ASCII art tractor:
```
    ~~  ~~
   ~~~~~~~~
  ___||____||___
 /  _____  ___ \
|__|     |_|  |_|
  (o)     (o)
```
Animate the exhaust (`~~`) by cycling through 3 frames using setInterval (every 300ms): `~~  ~~`, ` ~~~ ~~`, `~~  ~~~`. Replace the loading bubble with the actual response once streaming starts.

**SSE streaming reader:**
- `sendMessage()` function:
  1. Get input text, trim. If empty, return.
  2. Add user message bubble to DOM.
  3. Push `{ role: 'user', content: text }` to conversationHistory.
  4. Clear input, disable it, set isStreaming = true.
  5. Show tractor loading animation in a new assistant bubble.
  6. `fetch('/api/agent/chat', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({ message: text, history: conversationHistory.slice(-20), sessionKey }) })` — send last 20 messages max (sliding window per research pitfall 3).
  7. Get `response.body.getReader()`, create TextDecoder.
  8. Buffer-based SSE parsing: accumulate chunks, split on `\n\n`, keep incomplete tail in buffer.
  9. For each SSE event:
     - `type: 'text_delta'`: Replace loading animation with streaming text. Append delta content to the assistant bubble's text. Call renderMessageContent() to process markdown/charts/csv on each delta (but use a throttled approach — update rendered HTML every 100ms, not every delta).
     - `type: 'tool_call'`: Show a small italic status line in the message area: "Querying {tool}..." (dimmed text).
     - `type: 'done'`: Finalize the message. Run full renderMessageContent() on fullText. Push `{ role: 'assistant', content: fullText }` to conversationHistory. Re-enable input. Set isStreaming = false.
     - `type: 'warning'`: Show a small warning banner at top of messages: "Approaching daily limit ({remaining} messages left)".
     - `type: 'error'`: Show error message in the assistant bubble. Re-enable input.
  10. On fetch error: show error in assistant bubble, re-enable input.

**Markdown renderer** (`renderMessageContent(text)`):
A minimal hand-rolled renderer. DO NOT use a library. Handle these patterns only:
- **Bold:** `**text**` -> `<strong>text</strong>`
- **Italic:** `*text*` -> `<em>text</em>` (careful not to conflict with bold)
- **Code inline:** `` `code` `` -> `<code>code</code>`
- **Code blocks:** ` ```lang\n...\n``` ` -> `<pre><code class="lang">...</code></pre>` EXCEPT for `chartjs` and `csv` blocks (special handling below)
- **Headers:** `### text` -> `<h4>text</h4>` (use h4-h6 to not conflict with page headers)
- **Lists:** `- item` or `* item` -> `<ul><li>item</li></ul>` (group consecutive list items)
- **Tables:** `| col | col |` rows with `|---|---|` separator -> `<table>` with `<thead>` and `<tbody>`
- **Links:** `[text](url)` -> `<a href="url">text</a>`
- **Deep links:** Links matching `#ticket-(\d+)` pattern get an onclick handler that navigates to the ticket in the app (switch to Ticket Log tab, search for ticket ID, scroll to it).
- **Line breaks:** `\n` -> `<br>` (except inside code blocks and tables)

**Chart rendering** (from ` ```chartjs ` blocks):
- Detect fenced code blocks with language `chartjs`
- Parse the JSON content as a Chart.js config object
- Replace the code block with a `<canvas>` element (unique ID)
- Use `setTimeout(() => { if (window.Chart) new Chart(canvas, config); }, 50)` to instantiate after DOM insertion
- Canvas max-height: 250px, responsive
- On parse error, fall back to showing the raw JSON in a `<pre>` block

**CSV export** (from ` ```csv ` blocks):
- Detect fenced code blocks with language `csv`
- Replace with a styled "Export CSV" button
- On click, create a Blob with the CSV content, trigger download via temporary `<a>` element with `download="glomalin-export.csv"`
- Also show a small preview table of the first 3 rows below the button

**Clear chat:**
- Reset conversationHistory to []
- Clear all messages from DOM
- Generate a new sessionKey

**Popup toggle:**
- Click button: toggle popup display (none/flex)
- Close button: hide popup (don't destroy — preserves conversation)
- Escape key: close popup if open

**Daily cap integration:**
- On widget init, fetch GET /api/agent/status to check remaining messages
- If remaining === 0, show a permanent message in the chat area: "Daily message limit reached. Try again tomorrow!"
- Disable input if at cap

**Step 1: Download Chart.js 4.x UMD bundle locally.**

```bash
cd grain-tickets/public && curl -sL "https://cdn.jsdelivr.net/npm/chart.js@4/dist/chart.umd.min.js" -o chart.min.js
```

If curl fails (no internet), create a minimal placeholder file with a comment explaining to download Chart.js manually. The widget has a guard (`if (window.Chart)`) that gracefully skips chart rendering if Chart.js is not loaded.

**Step 2: Update grain-tickets/public/index.html.**

Add these 3 lines BEFORE the existing `<script src="calc.js"></script>` line (near the bottom, before other app scripts):

```html
  <link rel="stylesheet" href="glomalin.css">
  <script src="chart.min.js" defer></script>
  <script src="glomalin.js" defer></script>
```

The `defer` attribute ensures Chart.js loads before glomalin.js executes, and both load after the HTML is parsed. The GLOMALIN_ENABLED flag is injected by the server-side `/` route handler (from Plan 01 Task 2) into the `<head>`, so it's available before glomalin.js runs.

**Important:** Do NOT add a CDN `<script>` tag for Chart.js. Use the local chart.min.js file per research recommendation (farm office may have unreliable internet).

**Step 3: Verify the script loading order is correct:**
1. `<head>` contains server-injected `<script>window.GLOMALIN_ENABLED=true;</script>` (from Plan 01)
2. `<link rel="stylesheet" href="glomalin.css">` loads styles
3. `<script src="chart.min.js" defer>` loads Chart.js
4. `<script src="glomalin.js" defer>` loads widget (after Chart.js due to DOM order with defer)
5. Existing app scripts (calc.js, app.js, etc.) are unaffected
