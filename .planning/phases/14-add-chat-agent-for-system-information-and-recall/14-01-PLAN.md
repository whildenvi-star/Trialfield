**Step 1: Add 3 new models to grain-tickets/prisma/schema.prisma** (append after SettlementLine model):

```prisma
model AgentConversation {
  id         Int      @id @default(autoincrement())
  sessionKey String   // UUID generated per page load — groups messages in one session
  role       String   // "user" | "assistant"
  content    String   // full message text
  toolCalls  String?  // JSON stringified tool calls made (for audit)
  createdAt  DateTime @default(now())

  @@index([sessionKey])
  @@index([createdAt])
}

model AgentNote {
  id        Int      @id @default(autoincrement())
  content   String   // The fact to remember
  category  String   @default("general") // "farm" | "crop" | "buyer" | "general"
  source    String   @default("agent")   // "agent" | "admin"
  active    Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@index([active])
}

model AgentDailyUsage {
  id        Int      @id @default(autoincrement())
  date      String   // YYYY-MM-DD — one row per calendar day
  count     Int      @default(0)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([date])
}
```

**grain-tickets/lib/agent/tools.js** — Tool definitions + Prisma implementations:
- `getToolDefinitions()` returns an array of tool definition objects with name, description, input_schema
- Tools to define:
  - `query_tickets` — filters: farm (contains, insensitive), crop (contains, insensitive), cropYear (int), dateFrom/dateTo (DateTime range), ticketNo (contains), buyerName (contains, insensitive), limit (default 50, max 100). Returns array of ticket records with computed fields (grossBU, netBU via Calc.js). MUST exclude any settlement/reconciliation fields from the response.
  - `get_farm_summary` — filters: cropYear, farmName. Groups by farm+crop, returns totalWeight, totalBU, avgMoisture, ticketCount, acres, yieldPerAcre. Uses Prisma groupBy or raw aggregation.
  - `get_crop_stats` — filter: cropYear. Aggregates by crop: totalWeight, totalBU, avgMoisture, ticketCount.
  - `get_buyers` — no filters. Returns buyer list with name, shortCode, type (from Prisma Buyer table).
  - `get_grain_bins` — no filters. Returns grain bin list with name, capacity, ticket count.
  - `add_ticket_note` — params: ticketId (required int), note (required string). Updates ticket.notes field. WRITE tool — executeTool must guard against calling without prior confirmation in conversation history.
  - `flag_ticket` — params: ticketId (required int), flagged (required boolean), reason (optional string). NOT a real settlement dispute flag — this adds a note prefix "[FLAGGED]" or removes it. WRITE tool.
  - `remember_note` — params: content (required string), category (optional, default "general"). Creates an AgentNote record. WRITE tool.
  - `recall_notes` — params: category (optional string filter). Returns active AgentNote records.
- `executeTool(name, input)` dispatches to the correct Prisma query function, returns the result object.
- Import prisma from `../db`, import Calc from `../../public/calc.js`.
- For write tools (`add_ticket_note`, `flag_ticket`, `remember_note`), the function should succeed but include a `_writeAction: true` property in the result so the system can audit.

**grain-tickets/lib/agent/system-prompt.js** — System prompt builder:
- `buildSystemPrompt()` async function that:
  1. Reads all active AgentNote records from the database
  2. Builds a system prompt array with a single object: `{ type: 'text', text: promptString, cache_control: { type: 'ephemeral' } }`
  3. The prompt text must include:
     - Character: "You are Glomalin" — Gen Z, super happy, chill farmhand vibe. Mix farm terms with Gen Z energy (e.g., "that rye yield is absolutely bussin fr fr"). Fun personality but data accuracy is NEVER compromised.
     - Critical rules: ONLY answer grain data questions. NO settlement/financial/reconciliation data access. Before ANY write action, describe what you'll do and ask "should I go ahead with that?" — wait for explicit confirmation.
     - Show your work: briefly mention what data was queried and how calculations were done.
     - Default to current crop year (dynamically computed) for queries unless user specifies otherwise.
     - Deep link format: when referencing a specific ticket, use markdown link `[Ticket #XXXXX](#ticket-ID)` (widget will intercept and navigate).
     - Chart instruction: when presenting data that benefits from visualization (comparisons, distributions, trends), output a fenced code block with language `chartjs` containing a valid Chart.js config JSON. Keep chart types simple: bar, line, pie, doughnut. Always include labels and at least one dataset.
     - CSV export instruction: when presenting tabular data, also output a fenced code block with language `csv` containing the data in CSV format. The widget will render an "Export CSV" button.
     - Known facts section: inject all active AgentNote records as bullet points.
     - Data access inventory: tickets (date, farm, netWeight, moisture, fm, crop, ticketNo, hbtBinNo, truckId, destination, notes, buyerId, cropYear), farms (name, crop, acres, type, yieldPerAcre, guarantee, coverage), cropConfig (cropYear, cropName, testWeight, moistureShrink, discount), buyers (name, shortCode, type), grainBins (name, capacity).
     - Cannot see: settlements, reconciliation, matchStatus, financial data, prices, payments.
  4. Exports the function.

**grain-tickets/lib/agent/daily-cap.js** — Daily cap:
- `checkAndIncrementCap()` async function:
  1. Reads `CHAT_DAILY_CAP` from env (default 50)
  2. Uses prisma.agentDailyUsage.upsert with today's date (YYYY-MM-DD)
  3. Returns `{ allowed: boolean, count: number, remaining: number, nearLimit: boolean }` where nearLimit = count >= cap * 0.8

**grain-tickets/lib/agent/chat.js** — Agentic loop + SSE handler:
- `handleChat(req, res)` async function:
  1. Extract `{ message, history, sessionKey }` from req.body
  2. Set SSE headers: Content-Type text/event-stream, Cache-Control no-cache, Connection keep-alive. Call `res.flushHeaders()`.
  3. Check daily cap via checkAndIncrementCap(). If not allowed, write SSE event `{ type: 'error', error: 'Daily message limit reached', count, remaining: 0 }` and end.
  4. If nearLimit, write SSE event `{ type: 'warning', message: 'Approaching daily limit', remaining }`.
  5. Build messages array: `[...history, { role: 'user', content: message }]`
  6. Enter agentic loop (max 10 iterations to prevent runaway):
     a. Call `anthropic.messages.stream()` with model `claude-haiku-4-5-20251001`, max_tokens 4096, system from buildSystemPrompt(), tools from getToolDefinitions(), messages.
     b. Attach `stream.on('text', delta => res.write(SSE event { type: 'text_delta', content: delta }))`.
     c. Await `stream.finalMessage()`.
     d. If stop_reason === 'end_turn': extract final text, break loop.
     e. If stop_reason === 'tool_use':
        - Push `{ role: 'assistant', content: response.content }` to messages.
        - For each tool_use block in response.content, call executeTool(block.name, block.input).
        - Collect all results into toolResults array: `[{ type: 'tool_result', tool_use_id: block.id, content: JSON.stringify(result) }]`.
        - Send SSE event `{ type: 'tool_call', tool: block.name }` for each tool (so UI can show "Querying tickets...").
        - Push `{ role: 'user', content: toolResults }` to messages.
        - Continue loop.
     f. If stop_reason === 'max_tokens': break with what we have.
  7. Write SSE event `{ type: 'done', fullText: finalText }`.
  8. Log conversation: create AgentConversation records for the user message and assistant response (with toolCalls JSON if any).
  9. End response.
  10. Wrap entire handler in try/catch. On error, write SSE event `{ type: 'error', error: 'Something went wrong' }` and end. Log the full error server-side.
- Import: `require('@anthropic-ai/sdk')`, `require('./tools')`, `require('./system-prompt')`, `require('./daily-cap')`, `require('../db')`.
- Create Anthropic client at module level: `const anthropic = new (require('@anthropic-ai/sdk').default)()`.

**Step 1: Add kill-switch middleware and route registration to grain-tickets/server.js.**

Near the top of server.js (after the existing `app.use('/api', ...)` cache-control middleware), add:

```javascript
// --- Agent kill switch middleware ---
app.use('/api/agent', (req, res, next) => {
  if (process.env.CHAT_AGENT_ENABLED !== 'true') {
    return res.status(503).json({ error: 'Chat agent is disabled' });
  }
  next();
});
```

**Step 2: Add route imports and registrations.** After the kill-switch middleware, add these routes:

```javascript
const { handleChat } = require('./lib/agent/chat');
const { checkAndIncrementCap } = require('./lib/agent/daily-cap');

// POST /api/agent/chat — SSE streaming chat endpoint
app.post('/api/agent/chat', handleChat);

// GET /api/agent/status — returns agent enabled status + daily usage
app.get('/api/agent/status', async (req, res) => {
  try {
    const cap = await checkAndIncrementCap();
    // Decrement because checkAndIncrement already incremented — status is a read-only check
    // Actually, don't call checkAndIncrement for status. Use a separate read.
    const today = new Date().toISOString().split('T')[0];
    const usage = await prisma.agentDailyUsage.findUnique({ where: { date: today } });
    const dailyCap = parseInt(process.env.CHAT_DAILY_CAP || '50', 10);
    const count = usage ? usage.count : 0;
    res.json({
      enabled: true, // if we got here, middleware passed
      dailyCap,
      messagesUsed: count,
      remaining: Math.max(0, dailyCap - count),
      nearLimit: count >= dailyCap * 0.8
    });
  } catch (err) {
    console.error('Agent status error:', err);
    res.status(500).json({ error: 'Failed to check agent status' });
  }
});

// --- Agent Notes CRUD ---
// GET /api/agent/notes — list all notes (optionally filter by category, active)
app.get('/api/agent/notes', async (req, res) => {
  try {
    const where = {};
    if (req.query.category) where.category = req.query.category;
    if (req.query.active !== undefined) where.active = req.query.active === 'true';
    else where.active = true; // default to active only
    const notes = await prisma.agentNote.findMany({ where, orderBy: { createdAt: 'desc' } });
    res.json(notes);
  } catch (err) {
    console.error('Agent notes list error:', err);
    res.status(500).json({ error: 'Failed to list notes' });
  }
});

// POST /api/agent/notes — create a note (from admin page)
app.post('/api/agent/notes', async (req, res) => {
  try {
    const { content, category } = req.body;
    if (!content || !content.trim()) return res.status(400).json({ error: 'Content is required' });
    const note = await prisma.agentNote.create({
      data: {
        content: content.trim(),
        category: category || 'general',
        source: 'admin'
      }
    });
    res.status(201).json(note);
  } catch (err) {
    console.error('Agent note create error:', err);
    res.status(500).json({ error: 'Failed to create note' });
  }
});

// PUT /api/agent/notes/:id — update a note
app.put('/api/agent/notes/:id', async (req, res) => {
  try {
    const id = parseInt(req.params.id, 10);
    const { content, category, active } = req.body;
    const data = {};
    if (content !== undefined) data.content = content.trim();
    if (category !== undefined) data.category = category;
    if (active !== undefined) data.active = active;
    const note = await prisma.agentNote.update({ where: { id }, data });
    res.json(note);
  } catch (err) {
    console.error('Agent note update error:', err);
    res.status(500).json({ error: 'Failed to update note' });
  }
});

// DELETE /api/agent/notes/:id — delete a note permanently
app.delete('/api/agent/notes/:id', async (req, res) => {
  try {
    const id = parseInt(req.params.id, 10);
    await prisma.agentNote.delete({ where: { id } });
    res.json({ deleted: true });
  } catch (err) {
    console.error('Agent note delete error:', err);
    res.status(500).json({ error: 'Failed to delete note' });
  }
});

// GET /api/agent/conversations — list recent conversations (for audit)
app.get('/api/agent/conversations', async (req, res) => {
  try {
    const limit = Math.min(parseInt(req.query.limit || '50', 10), 200);
    const conversations = await prisma.agentConversation.findMany({
      orderBy: { createdAt: 'desc' },
      take: limit
    });
    res.json(conversations);
  } catch (err) {
    console.error('Agent conversations error:', err);
    res.status(500).json({ error: 'Failed to list conversations' });
  }
});
```
