  <action>
Add a GET /health route to each of the 6 Express server.js files. Place it early in the route definitions (before static file serving) so it responds fast and is not affected by middleware.

Each endpoint must return:
- HTTP 200 status
- Content-Type: application/json
- JSON body: `{ "status": "ok", "app": "{app-name}", "uptime": process.uptime() }`

App names to use (matching PM2 ecosystem names):
- grain-tickets/server.js -> "grain-tickets"
- farm-budget/server.js -> "farm-budget"
- fsa-acres/server.js -> "fsa-acres"
- meristem-malt/server.js -> "meristem-malt"
- farm-registry/server.js -> "farm-registry"
- seed-inventory/server.js -> "seed-inventory"

The route should bypass CORS middleware (no credentials needed for health checks). Place `app.get('/health', ...)` BEFORE any CORS or auth middleware if possible. If CORS is already applied globally via `app.use(cors(...))`, that is fine -- the health endpoint will still work, it just means health checks from non-allowed origins get blocked, which is acceptable since health checks run from localhost or the same server.

Do NOT add any dependencies. This is plain Express: `app.get('/health', (req, res) => res.json({...}))`.
  </action>

  <action>
Create scripts/health-check.sh -- a bash script that curls all 6 Express app health endpoints and reports status.

Script requirements:
- Shebang: `#!/usr/bin/env bash`
- Define an associative array or list mapping app names to ports:

  - farm-budget: 3001
  - fsa-acres: 3002

  - seed-inventory: 3006

- Use ANSI color codes: green="\033[0;32m", red="\033[0;31m", reset="\033[0m"
- Mark the file executable (chmod +x)

Also include the 2 Next.js apps as a bonus check (glomalin-portal on 3000, organic-cert on 3004) but just check if the port responds (they may not have /health, just check TCP with curl to /). Label these separately as "Next.js apps" in the output.

Do NOT use PM2 commands in the script -- pure curl checks so it works with or without PM2.
