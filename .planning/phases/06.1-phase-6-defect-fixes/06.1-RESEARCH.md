**Researched:** 2026-03-21

## Summary

Phase 6.1 addresses three high-severity defects uncovered by the v2.0 milestone audit. All three are surgical, isolated fixes with no schema migrations and no new dependencies. The codebase has been read directly, so the analysis is based on the actual source files rather than training-data inference.

**Defect 1 (ACT-02):** `POST /api/import-plan/confirm` has zero auth code — no `getAuthContext()`, no `hasPermission()`, no `logAudit` userId. The fix is a three-line import + guard block that mirrors the pattern already in every other Phase 6 write route. This is the simplest of the three fixes.

**Defect 2 (VIEW-01, VIEW-02):** The seed section of `BudgetTab.tsx` displays `sc.actualPricePerUnit` (price per seed unit, e.g. $/bag) directly in the $/ac Actual column. The projected column shows `totalCost / acres`. These units are incompatible, making variance meaningless. The fix is client-side: compute `actualPricePerUnit * sc.rate / sc.acres` — or more precisely use the SeedCostRow's existing `rate` and `acres` fields — to derive actual seed cost per acre. The API stores `actualPricePerUnit` correctly; this is a display-only bug.

**Defect 3 (VIEW-01 category flow):** `BUDGET_CATEGORIES` in `BudgetTab.tsx` contains `"Equipment"`, which does not appear in `ALLOWED_CATEGORIES` in `unplanned-cost/route.ts`. The API accepts: Seed, Fertilizer, Chemical, Custom Application, Crop Insurance, Land Rent, Overhead, Labor, Fuel, Repair, Other. The UI must be updated to match the API exactly — remove "Equipment", add the missing 8 categories.

**Primary recommendation:** Three independent single-file fixes; plan as one wave with no ordering dependency between fix 2 and fix 3, but fix 1 (API route) is highest severity and should be listed first.

## Architecture Patterns

### Auth Guard Pattern (for Defect 1)

Every Phase 6 write route follows this exact pattern. `POST /api/import-plan/confirm` is the only route that does not.

**Reference implementation:** `src/app/api/field-enterprises/[id]/applications/[recordId]/actual/route.ts` lines 1–20

```typescript
// Source: actual codebase — applications/actual/route.ts
import { getAuthContext } from "@/lib/auth";
import { hasPermission } from "@/lib/rbac";
import { Role } from "@prisma/client";

export async function POST(request: Request) {
  try {
    const user = await getAuthContext();
    if (!user) {
      return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
    }
    const role = user.role as Role;
    if (!hasPermission(role, "budget:write")) {
      return NextResponse.json({ error: "Forbidden" }, { status: 403 });
    }
    // ... rest of handler
```

The `logAudit` call at the bottom of `confirm/route.ts` currently passes `userId: undefined` and `userName: undefined` because no auth context was available. After adding the guard, these should be populated from `user.id` and `user.email` (or equivalent fields from `getAuthContext()`).

**What `getAuthContext()` returns:** Check `@/lib/auth` to confirm the shape. Based on the pattern in other routes (`user.role as Role`), it returns an object with at minimum `{ role: string }`. The audit logger in the existing route uses `userId` and `userName` — confirm these field names from the auth context shape before coding.

### Seed Cost Per Acre Computation (for Defect 2)

The `SeedCostRow` interface already provides all needed fields:

```typescript
// Source: BudgetTab.tsx interface SeedCostRow
interface SeedCostRow {
  id: string;
  variety: string;
  brand: string | null;
  rate: number;       // e.g. 5 (bags per acre, or units per acre)
  rateUnit: string;   // e.g. "bags", "units", "lbs"
  acres: number;      // enterprise acres
  pricePerUnit: number | null;    // projected price per unit
  totalCost: number;              // projected total cost (= pricePerUnit * rate * acres)
  actualPricePerUnit: number | null;  // actual price per unit — THIS is what's stored
}
```

**Current (broken) logic (lines 598–601):**
```typescript
const actualPerAcre =
  sc.actualPricePerUnit !== null
    ? sc.actualPricePerUnit   // BUG: displays price/unit, not cost/acre
    : null;
```

**Correct computation:**
```typescript
const actualPerAcre =
  sc.actualPricePerUnit !== null && sc.acres > 0
    ? (sc.actualPricePerUnit * sc.rate) / sc.acres   // actual cost/acre
    : null;
```

Wait — let me think through the math more carefully:
- `actualPricePerUnit` = dollars per seed unit (e.g. $120/bag)
- `rate` = seed units applied per acre (e.g. 5 bags/acre) — confirmed from `rateUnit` shows as "{rate} {rateUnit}/ac" in the UI
- `actualCostPerAcre` = `actualPricePerUnit * rate` ($/bag × bags/ac = $/ac)
- The `acres` field is NOT needed in the per-acre formula since rate is already per-acre

**Revised correct computation:**
```typescript
const actualPerAcre =
  sc.actualPricePerUnit !== null
    ? sc.actualPricePerUnit * sc.rate   // $/unit × units/acre = $/acre
    : null;
```

This also ensures the `VarianceCell` compares `projectedPerAcre` (= `totalCost / acres`) against `actualPerAcre` (= `actualPricePerUnit * rate`) — both in $/ac units.

**Verify against projected:** `totalCost` = `pricePerUnit * rate * acres`, so `projectedPerAcre` = `totalCost / acres` = `pricePerUnit * rate`. The formula `actualPricePerUnit * rate` is unit-consistent with `pricePerUnit * rate`. Confirmed correct.

**Display label change:** The sub-label below the ActualCell currently shows `{sc.rateUnit}` (the seed unit name). After the fix, since the cell displays $/ac, the label should be updated to `/ac` to match the rest of the budget table columns.

### Category List Alignment (for Defect 3)

**API source of truth** (`unplanned-cost/route.ts` lines 8–20):
```typescript
const ALLOWED_CATEGORIES = [
  "Seed", "Fertilizer", "Chemical", "Custom Application",
  "Crop Insurance", "Land Rent", "Overhead", "Labor",
  "Fuel", "Repair", "Other",
] as const;
```

**UI current definition** (`BudgetTab.tsx` lines 147–154):
```typescript
const BUDGET_CATEGORIES = [
  "Seed", "Fertilizer", "Chemical", "Custom Application",
  "Equipment",   // ← NOT in API ALLOWED_CATEGORIES — causes silent 400 failure
  "Other",       // ← present in both
  // Missing: "Crop Insurance", "Land Rent", "Overhead", "Labor", "Fuel", "Repair"
] as const;
```

**Required UI fix:** Replace `BUDGET_CATEGORIES` in `BudgetTab.tsx` with exactly the API's 11 categories in the same order. Remove "Equipment". Add "Crop Insurance", "Land Rent", "Overhead", "Labor", "Fuel", "Repair".

**Anti-pattern to avoid:** Do not add client-side validation that shows an error for "Equipment" — remove it entirely. Do not add a silent filter on the save handler — fix the source list.

### Anti-Patterns to Avoid

- **Don't filter categories client-side after selection:** Fix the dropdown list itself.
- **Don't compute variance in the UI from incompatible units:** The seed fix is a display formula change only; do not change what `actualPricePerUnit` stores or how the API computes the seed total.
- **Don't add a userId lookup via a separate DB call in the auth guard:** `getAuthContext()` already returns user info — use it directly.
- **Don't change the audit log signature:** The existing `logAudit` call just needs `userId` and `userName` populated from the auth context.

## Common Pitfalls

### Pitfall 1: Auth Context Field Names
**What goes wrong:** Assuming `user.userId` when the field is `user.id`, or `user.name` when it's `user.email` — the `logAudit` call has `userId` and `userName` parameters.
**Why it happens:** The auth context shape is defined in `@/lib/auth` and hasn't been inspected here.
**How to avoid:** Read `src/lib/auth.ts` before writing the guard code to confirm the field names. The PATCH routes in Phase 6 only use `user.role` so this hasn't been an issue there — but the confirm route's `logAudit` call needs populated fields.
**Warning signs:** TypeScript error on `user.userId` or `user.userName` after adding the guard.

### Pitfall 2: Seed Rate Units Assumption
**What goes wrong:** Assuming `rate` is in a universal unit. If some seed products use "seeds" per acre (e.g. 140,000 seeds/ac), `actualPricePerUnit` might be priced per 1,000 seeds or per bag — the formula would be wrong.
**Why it happens:** `rateUnit` field varies by product.
**How to avoid:** The projected column already uses `totalCost / acres` which correctly handles any unit combination (since it's derived from actual purchase/invoice data). The formula `actualPricePerUnit * rate` mirrors exactly how `totalCost` is constructed (`pricePerUnit * rate * acres / acres = pricePerUnit * rate`). This is safe as long as `actualPricePerUnit` and `pricePerUnit` are denominated in the same unit — which they are (both per `rateUnit`).
**Warning signs:** If the projected and actual columns still show wildly different magnitudes after the fix, the rateUnit interpretation differs for that specific seed product.

### Pitfall 3: BUDGET_CATEGORIES TypeScript `as const`
**What goes wrong:** After replacing the array, TypeScript type inference breaks if any code pattern-matches against the old union type.
**Why it happens:** `as const` creates a readonly tuple — if other code derives `typeof BUDGET_CATEGORIES[number]` the type changes.
**How to avoid:** Search `BudgetTab.tsx` for any type derivation from `BUDGET_CATEGORIES` before replacing. The dropdown rendering on line 952 does `{BUDGET_CATEGORIES.map(...)}` — safe, just iteration.

### Pitfall 4: Confirm Route logAudit with Missing User
**What goes wrong:** After adding the guard, the audit call still logs `userId: undefined` because the developer forgets to pass the now-available user object.
**Why it happens:** The `logAudit` call is several lines below the guard.
**How to avoid:** Update both the guard (top of handler) AND the logAudit call (bottom of handler) in the same edit.

## Open Questions

1. **`getAuthContext()` return shape for `logAudit`**
   - What we know: Returns `null` or an object with at least `{ role: string }` (confirmed from usage in every Phase 6 route)
   - What's unclear: The exact field names for user ID and display name — `userId`? `id`? `email`? `userName`?
   - Recommendation: Read `src/lib/auth.ts` as the first action in the plan task. The fix itself is correct regardless; this only affects the audit log quality improvement.

2. **Seed rate unit consistency across all seed products**
   - What we know: `rate` and `rateUnit` are populated for every SeedCostRow; `totalCost` is computed as `pricePerUnit * rate * acres` at sync time
   - What's unclear: Whether any seed product uses a non-standard rate convention (e.g. cost per 1000 seeds vs per unit)
   - Recommendation: The formula is mathematically correct for any unit as long as `actualPricePerUnit` and `pricePerUnit` use the same denominator. Verify with one real data example during human verification.
