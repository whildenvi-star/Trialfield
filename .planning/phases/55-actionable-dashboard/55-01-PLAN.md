  <action>
Create GET /api/dashboard/action-items route. Auth-guarded (same pattern as /api/dashboard/summary).

The route fetches from 5 data sources using Promise.allSettled:

**Supabase queries (3):**
1. CLU records: `clu_records` where `crop_year = CURRENT_CROP_YEAR`. Compute action items:
   - Unreported CLUs: count where `reported = false` or null. Severity: warning if count > 0. Summary: "{N} CLU records missing acreage report". Link: `/app/fsa-578?filter=unreported`
2. Insurance policies: `insurance_policies` where `policy_year = CURRENT_CROP_YEAR` and `claim_alert = 'potential'`. Severity: warning. Summary: "{N} policies with potential claim alerts". Link: `/app/insurance?filter=claim-alerts`
3. Claims: `claims` where `stage != 'closed'`. Compute:
   - Open claims: count all. Severity: info. Summary: "{N} open claims in pipeline". Link: `/app/claims`
   - Overdue claims (if `stage` is 'filed' or 'notice-of-loss' and age > 30 days based on `created_at`): severity warning. Summary: "{N} claims overdue (>30 days)". Link: `/app/claims?filter=overdue`. Only include this item if count > 0.

**Express queries (2) — wrapped in try/catch, offline returns empty items with offline: true:**
4. Grain-tickets settlements: Call `fetchGrainService('/api/settlements?cropYear=' + CURRENT_CROP_YEAR)`. From the response, count settlements where `_count.lines === 0` (unreconciled — no matched lines). Severity: info if count > 0. Summary: "{N} settlements with unmatched loads". Link: `/app/grain-tickets?tab=settlements`
5. Farm-budget delivery shortfalls: Call `fetchBudgetService('/api/forecast')` and `fetchBudgetService('/api/deliveries')` via Promise.allSettled. Compare forecast totalQty vs delivered qty per product. If delivered < 80% of forecast for any product, that's a shortfall. Aggregate: severity info. Summary: "{N} input products below 80% delivery target". Link: `/app/farm-budget?tab=deliveries`

**Response assembly:**
- Group items by module using MODULE_SOURCES keys
- Only include groups that have items OR are offline (offline groups get `offline: true`, empty items array)
- Skip groups with zero items and online status (don't show empty modules)
- Calculate totalCount across all groups
- Return `{ groups, totalCount, fetchedAt: Date.now() }` as JSON
- Set `Cache-Control: no-cache` header (same as summary route)

**Key implementation details:**
- Use `Promise.allSettled` for ALL 5 fetches (Supabase queries can also be wrapped for consistency, though Supabase rarely fails)
- For Express failures (rejected promise or non-ok response or AbortSignal.timeout): set that group's `offline: true`
- Import CURRENT_CROP_YEAR from `@/lib/config`
- Import types from `@/lib/action-items`
- Import fetchBudgetService and fetchGrainService from proxy
- Each ActionItem needs a unique `id` string (e.g., `fsa-unreported`, `ins-claim-alerts`, `claims-open`, `claims-overdue`, `gt-unmatched`, `budget-delivery-shortfall`)
