    Extend the existing budget-summary GET route to return projected and actual data side by side. This resolves the STATE.md blocker: "Budget-summary computation mixes PLANNED and CONFIRMED operations."

    **Projected computation (existing, keep unchanged):**
    - `totalSeedCost`: sum of all seed costs from SeedLot.purchasePrice via SeedUsage
    - `totalMaterialCost`: sum of all MaterialUsage.totalCost (projected from sync)
    - `totalOperationCost`: sum of ALL FieldOperation costs (PLANNED + CONFIRMED) -- this is the full budget plan
    - `totalCostOfProduction`: sum of the above
    - `costPerAcre`: totalCostOfProduction / acres

    **Actual computation (NEW):**
    - `actualSeedCost`: sum of SeedUsage.actualPricePerUnit where set (null if none entered)
    - `actualMaterialCost`: sum of MaterialUsage.actualTotalCost where set (null if none entered)
    - `actualOperationCost`: sum of CONFIRMED FieldOperations only, using actualTotalCost if set, falling back to the operation's totalCost if no actual override
    - `actualYieldPerAcre`: from FieldEnterprise.actualYieldPerAcre (null if not entered)
    - `actualTotalCost`: sum of actualSeedCost + actualMaterialCost + actualOperationCost (null if ALL are null)
    - `actualCostPerAcre`: actualTotalCost / acres (null if actualTotalCost is null)

    **Variance computation (NEW, server-side):**
    - `varianceSeedCost`: totalSeedCost - actualSeedCost (null if actualSeedCost is null)
    - `varianceMaterialCost`: totalMaterialCost - actualMaterialCost (null if actual is null)
    - `varianceOperationCost`: totalOperationCost - actualOperationCost (null if actual is null)
    - `varianceTotalCost`: totalCostOfProduction - actualTotalCost (null if actual is null)
    - `varianceCostPerAcre`: costPerAcre - actualCostPerAcre (null if actual is null)

    **Unplanned costs (NEW):**
    - Query MaterialUsage records with `dataSource: ACTUAL` that have no corresponding projected record (i.e., `totalCost` is null but `actualTotalCost` is set)
    - Return as `unplannedCosts: { id, category, actualTotalCost }[]`
    - Include unplanned costs in actualTotalCost aggregation

    **Per-line-item actuals (NEW):**
    Include `actualTotalCost` and `actualUnitCost` in each material cost object returned. Include `actualPricePerUnit` in each seed cost object. Include `passStatus` in each operation cost object. This lets the UI show per-row actual values and badges.

    **Financial field stripping:** The existing `canSeeFinancial` guard from Phase 5 MUST remain intact. New actual fields are cost data (visible to OFFICE via budget:read), NOT financial data.

    **CRITICAL anti-pattern:** Do NOT compute variance client-side. All variance math happens in this route.
