**1. Add conflict detection to replayOperation in sync-engine.ts:**

Before replaying a 'confirm-pass' operation, check if the pass is already confirmed:
- For confirm-pass: the POST /api/mobile/passes/confirm endpoint will return a 409 Conflict if the FieldOperation already has status CONFIRMED (the organic-cert API rejects duplicate confirmations). Treat 409 as a "skip" — not an error.
- If the server returns 200 but the response body indicates the pass was already confirmed (e.g., `{ alreadyConfirmed: true }`), also treat as skip.
- Add the skipped item to the SyncResult.skipped array with reason "Already confirmed — skipped".

Add a `getLastSyncTimestamp()` export that reads from a 'last-sync' key in the sync-config store. After a successful processQueue run (at least one item synced), write the current ISO timestamp.

Add a `getQueueSummary()` export:
```ts
async function getQueueSummary(): Promise<{
  pending: QueuedOperation[]
  failed: QueuedOperation[]
  total: number
  lastSync: string | null
}>
```
This reads all operations from the queue, partitions by status, and reads lastSync from sync-config.
