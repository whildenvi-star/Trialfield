  <action>
**In grain-tickets/public/calc.js:**
Add a `USDA_TEST_WEIGHTS` constant object mapping canonical crop names to USDA standard test weights (lb/bu). This is the reference for bushel conversion when per-farm CropConfig is not available. Standard values: wheat=60, corn=56, soybeans=60, oats=32, barley=48, rye=56, sorghum=56, sunflower=28, flax=56, peas=60. Export it alongside existing exports.

**In grain-tickets/server.js:**
1. Create `async function computeYieldSummaries(cropYear)` that:
   - Queries all tickets for the given cropYear (default: current crop year from getCropYear)
   - Queries all farms to get Farm.registryId (the canonical registry_field_id) and Farm.acres
   - Groups tickets by Farm.registryId + Ticket.registryCropId (BOTH must be non-null to be included)
   - For each group: sums netWeight (net pounds after deductions — this IS the weight basis per user decision), computes totalNetBU using existing Calc.computeTicket(), counts tickets
   - Computes yieldPerAcre = totalNetBU / Farm.acres (using the farm's acres from the Farm table)
   - Returns array of: { registryFieldId, registryCropId, farmName, cropName, totalNetLbs, totalNetBU, yieldPerAcre, acres, ticketCount }
   - Tickets whose farm has no Farm.registryId are EXCLUDED — log a one-time warning: `console.warn('Yield pipeline: N tickets excluded — farm "${farmName}" has no registryId')`
   - Tickets with no registryCropId are also excluded with similar warning

2. Create `GET /api/yield-summaries` endpoint:
   - Optional query param: `?cropYear=2026` (defaults to current)
   - Calls computeYieldSummaries(cropYear)
   - Returns `{ summaries: [...], cropYear, excludedTickets: { noFieldId: N, noCropId: N } }`
   - CORS already configured (existing corsOptions allows portal origin)

3. After each ticket save (POST /api/tickets), update (PUT /api/tickets/:id), and delete (DELETE /api/tickets/:id), call a new `async function pushYieldUpdates(cropYear)` that:
   - Calls computeYieldSummaries(cropYear) to get fresh summaries
   - Pushes to portal insurance and farm-budget (implementation in Plan 02 — for now, just log: `console.log('Yield summaries recomputed:', summaries.length, 'field/crop combos')`)
   - Wrap in try/catch so push failures never block the ticket response
   - Call AFTER the response is sent (use `res.json(...); pushYieldUpdates(cropYear).catch(...)` pattern — don't await, fire-and-forget after response)

Note: The acre denominator for yieldPerAcre uses Farm.acres from grain-tickets. Plan 02 will handle the insurance-specific acre denominator (planted_acres from insurance_policies) when pushing to the portal.
