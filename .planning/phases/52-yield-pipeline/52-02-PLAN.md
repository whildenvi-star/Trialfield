**In glomalin-portal — Create POST /api/insurance/yield-push/route.ts:**
New API route that receives bulk yield summaries from grain-tickets and updates matching insurance policies.

1. No auth guard (this is a server-to-server push from grain-tickets, not a user action). Use an ecosystem token check: `request.headers.get('x-ecosystem-token')` must match `process.env.EMBED_TOKEN` — return 401 if missing/wrong.
2. Request body: `{ summaries: [{ registryFieldId, registryCropId, farmName, cropName, totalNetLbs, totalNetBU, yieldPerAcre, acres, ticketCount }], cropYear: number }`
3. For each summary item:
   - Query insurance_policies matching `registry_field_id = registryFieldId AND registry_crop_id = registryCropId AND policy_year = cropYear`
   - If match found: compute yieldPerAcre using insurance policy's `planted_acres` as denominator (NOT the Farm.acres from grain-tickets — per user decision). Formula: `totalNetBU / planted_acres`
   - Update matched policy: `actual = computed yieldPerAcre`, `actual_synced_from_grain = true`, `yield_synced_at = new Date().toISOString()`
   - Recompute `claim_alert` using `computeClaimAlert({ guarantee, actual, coverage_level })` from `@/lib/insurance/calc`
   - If no matching policy: skip (per user decision — "flag as no insurance policy found" means just skip the push, the yield is still computed and stored in grain-tickets)
4. Return `{ ok: true, matched: N, skipped: N, errors: [] }`
5. Use createServerClient from @supabase/ssr with service_role key for writes (bypasses RLS).

**In farm-budget/server.js — Create POST /api/yield-from-grain:**
1. Ecosystem token check: `req.headers['x-ecosystem-token']` must match `process.env.ECOSYSTEM_TOKEN`
2. Request body: `{ summaries: [...], cropYear }`
3. Store summaries in module-level variable `let _grainYields = { data: null, updatedAt: null }`
4. Build lookup map: `"registryFieldId|registryCropId" → { yieldPerAcre, totalNetBU, ticketCount, syncedAt }`
5. Return `{ ok: true, count: summaries.length }`

Also add a `GET /api/yield-from-grain` endpoint so the client can fetch the cached yields:
- Returns `{ yields: _grainYields.data, updatedAt: _grainYields.updatedAt }`

   - Portal URL from `process.env.PORTAL_ORIGIN || 'http://localhost:3010'`

   - Same try/catch — independent of portal push (use Promise.allSettled for parallel)
4. Log push results: `console.log('Yield push: portal=${portalResult}, budget=${budgetResult}')`

Use AbortSignal.timeout(5000) on both fetch calls (consistent with existing cross-app fetch pattern).
