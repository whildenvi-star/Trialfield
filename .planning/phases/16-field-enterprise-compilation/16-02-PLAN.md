**Part A: POST /api/compile/[year]/route.ts — commit handler**

Create `organic-cert/src/app/api/compile/[year]/route.ts` with a POST export:

1. Parse `year` from dynamic route params. Validate it's a number 2020-2100.
2. Accept JSON body: `{ fieldIds: string[] }` — the list of matched fieldIds the user selected for commit. This enables partial commits (only fields the user chose to commit).
3. Call `buildPreview(year)` from compile-engine to get the current CompilePreview.
4. Filter preview rows to only those where `row.fieldId` is in the submitted `fieldIds` AND `row.status !== "unmatched"`.
5. For each qualifying row, run `prisma.fieldEnterprise.upsert()`:
   ```typescript
   await prisma.fieldEnterprise.upsert({
     where: {
       fieldId_cropYear_crop_label: {
         fieldId: row.fieldId,
         cropYear: year,
         crop: row.crop,
         label: null,
       },
     },
     create: {
       fieldId: row.fieldId!,
       cropYear: year,
       crop: row.crop,
       plantedAcres: row.registryAcres ?? row.plantedAcres,
       organicStatus: "ORGANIC",
       lotNumber: generateLotNumber(year, row.crop, row.fieldName ?? ""),
     },
     update: {
       plantedAcres: row.registryAcres ?? row.plantedAcres,
       crop: row.crop,
     },
   });
   ```

   The `generateLotNumber` helper creates a lot number in the format `{year}-{CROP}-{FIELD}` where CROP is uppercase abbreviation and FIELD is uppercase field name — matching the existing pattern in FieldEnterprise records (e.g. "2024-SRWW-KOPP"). Implement this as a local helper function within the route file.

6. Return JSON: `{ committed: number, skipped: number }` with `Cache-Control: no-store`.
7. On error, return 500 with error message.

IMPORTANT: The upsert uses `label: null` in the where clause. This relies on the partial unique index from Phase 15 migration `20260303025533_add_partial_unique_enterprise_label_null`. The index enforces uniqueness for `(fieldId, cropYear, crop) WHERE label IS NULL`.
