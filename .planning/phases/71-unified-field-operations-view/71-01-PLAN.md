  <name>Task 1: Create field-ops-groups.js classifier module</name>
  <files>farm-budget/public/field-ops-groups.js</files>
  <action>
Create a new file `farm-budget/public/field-ops-groups.js` exposing `window.FieldOpsGroups` with two exports:

**GROUP_ORDER** — array defining the display sequence:
`['Tillage', 'Fertility', 'Planting', 'Pre-emerge', 'Post-emerge', 'Fungicide', 'Harvest', 'Other']`

**classifyItem(name, itemType)** — returns one of the GROUP_ORDER strings based on pattern matching.
- `name`: the productName (for inputs) or implementName (for machinery)
- `itemType`: 'input' | 'pass' | 'custom'
- Returns group string (never null — unmatchable items fall into 'Other')

Build the pattern rules from the actual names found in data.json. Use case-insensitive matching on `.toLowerCase()`.

**Tillage** (machinery): chisel, disk, soil finisher, cultivat (matches cultivator), rock picker, stalk chop
**Planting** (machinery + inputs): planter, drill, custom no till planting, seed treatment (products with "seed treatment" in name), inoculant (Ppst, Exceed, Rhizolizer, Rhizolozer), "KWS CC rye", "Red Clover", "Rye seed", "Oats"
**Pre-emerge** (inputs + custom): "Application - Liquid Ppi", "Application - Liquid Pre", "Application - Burndown", "Application - VRT", Prowl, Outlook, Sharpen, Sonic, Zidua, Resicore, Authority, Verdict, Forsyte, pendimethalin, prowl, "hooded redball", "tine weed", spinner (implement or "Application - Spinner")
**Post-emerge** (inputs + custom): "Application - Post", Liberty, Basagran, Buccaneer, Durango, PowerMax, RoundUp, Enlist, Armezon, Atrazine, Status, Laudis, Callisto, Steadfast, Distinct, Mauler, Cobra, Huskie, Flexstar, Raptor, Volunteer, Weed Slayer, Weed Zapper (implement), Hooded Redball (implement)
**Fungicide** (inputs): Miravis, Palisade, Headline, Veltyma, Strobilurin (any fungicide keyword)
**Fertility** (inputs): products matching fertilizer patterns — NPK notation (contains numbers like "28%", "32%", "46-0-0", "18-46-0", "10-34-0", "0-0-60", "21-0-0", "12-0-0", "0-46-0"), anhydrous, urea, ammonia, AMS, DAP, potash, sulfur (S04), "Chicken Litter", "Tulls manure", manure, compost, lime, gypsum, chicken, feathermeal, chilean nitrate, boron, zinc, manganese, calcium, "TeraFed", "Eco Tec", "Acomplish Max", biological soil amendments broadly
**Harvest** (machinery): combine, buggy, grain cart, ear picker, trucking, truck, chop/haul, dump cart
**Other** (catch-all): rotary hoe, hand weeding, mow and condition, tedding, baling, windrow, detassle, "Wi Tonnage Tax", "Water", mixing charge, misc.

Implementation approach:
1. Define an array of `{ group, patterns }` objects in GROUP_ORDER sequence — patterns are lowercase substrings to test with `name.toLowerCase().includes(pattern)`.
2. Walk them in order, return first match. If none, return 'Other'.
3. Custom application inputs ("Application - ...") should be classified by the operation suffix (Post → Post-emerge, Pre/PPI → Pre-emerge, Spinner → Pre-emerge, Burndown → Pre-emerge).
4. Export as an IIFE assigning to `window.FieldOpsGroups = { GROUP_ORDER, classifyItem }`.

The `itemType` parameter is available for tiebreaking (e.g., a "Spinner" implement is Pre-emerge, but a "Spinner" product could be Other) but is secondary to name matching.

No external dependencies — plain vanilla JS.
