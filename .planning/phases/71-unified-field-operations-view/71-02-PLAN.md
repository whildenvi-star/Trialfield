  <action>
Add a `renderFieldOpsPanel()` function to field-editor.js. Insert it after the `renderMachRows()` function block (around line 580) and before the `renderAuxRows()` function.

**renderFieldOpsPanel() logic:**

1. **Build the unified item list** — merge `currentField.inputs` and `currentField.machinery` into a single working array, tagging each item with its source type:
   - From `currentField.inputs`: type = 'input'. If productName matches "Application -" prefix → type = 'custom'
   - From `currentField.machinery`: type = 'pass'
   - Seeds in `currentField.seeds` (if any): type = 'seed' — include them as read-only rows with name = variety, $/ac from the seed cost in the budget; these are display-only in this view (seed editing stays in the Seed tab)

2. **Classify each item** using `window.FieldOpsGroups.classifyItem(name, itemType)` where name is productName or implementName.

3. **Build a groups map** — iterate `FieldOpsGroups.GROUP_ORDER`, collect items per group, skip empty groups.

4. **Compute per-item costs:**
   - Input item cost: look up product in `window.refData.products`, use `Calc.computeApplicationPrice(product) * inp.quantity`
   - Pass item cost: look up implement in `window.refData.implements`; if `m.useHire && impl.customHireRate > 0` use `impl.customHireRate * m.passes`, else `impl.costPerAcre * m.passes`
   - Custom item (Application -): same as input item logic
   - Seed item: use `budget.seedTotal / (currentField.acres || 1)` as an approximation — or simply show "see Seed tab" with 0 cost (simpler, avoids double-computing seed)
   - Field total = costPerAcre * acres (use `currentField.acres || currentField.plantedAcres || 0`)

5. **Render HTML** into `#fo-groups-container`:

For each non-empty group, render:
```html
<div class="fo-group" data-group="{groupName}">
  <div class="fo-group-header fo-group-toggle" data-group="{groupName}">
    <span class="fo-group-chevron">▾</span>
    <span class="fo-group-name">{groupName}</span>
    <span class="fo-group-subtotal">{subtotalPerAc}/ac</span>
  </div>
  <div class="fo-group-body">
    <table class="compact-table fo-table" style="width:100%">
      <thead><tr><th></th><th>Name</th><th>Type</th><th>Rate</th><th>$/ac</th><th>Total $</th><th></th></tr></thead>
      <tbody>
        {rows}
      </tbody>
    </table>
    <button class="btn-sm fo-add-item" data-group="{groupName}" style="margin-top:0.4rem;font-size:0.72rem">+ Add to {groupName}</button>
  </div>
</div>
```

Each row:
```html
<tr data-drag-idx="{globalIdx}" data-item-type="{input|pass|custom|seed}" data-source-idx="{sourceIdx}">
  <td class="drag-handle">⠿</td>
  <td>{name}</td>
  <td><span class="fo-type-badge fo-badge-{type}">{type}</span></td>
  <td class="number" style="font-size:0.75rem">{rateDisplay}</td>
  <td class="number">{$/ac}</td>
  <td class="number">{total$}</td>
  <td><button class="btn-danger fo-remove" data-item-type="{type}" data-source-idx="{sourceIdx}" style="font-size:0.68rem">X</button></td>
</tr>
```
- rateDisplay: for inputs → `{qty} {unit}`, for passes → `{passes} pass` (or `{passes} passes`)
- Seed rows: drag handle is hidden (`<td></td>`), remove button is disabled (`style="display:none"`) — editing is in Seed tab

6. **Grand total** — sum all group subtotals, update `#fo-grand-val` with total $/ac and `#fo-grand-total-field` with total $ (acres × $/ac). Show the `#fo-grand-total` div.

7. **Event wiring** after HTML is set:
   - `.fo-group-toggle` click: toggle `fo-collapsed` class on the `.fo-group` parent; update chevron symbol (▾ ↔ ▸)
   - `#fo-expand-all` / `#fo-collapse-all` buttons: add/remove `fo-collapsed` on all `.fo-group` elements
   - `.fo-remove` click: splice from `currentField.inputs` or `currentField.machinery` by `data-source-idx` and `data-item-type`; call `renderFieldOpsPanel()` and `updatePreview()`
   - `.fo-add-item` click: handled in Plan 03 (leave as a no-op for now — `console.log('add to group:', group)`)

8. **Call renderFieldOpsPanel()** from `populateForm()` — add it after `renderMachRows()` call at line ~321. Also call it at the end of `renderInputRows()` and `renderMachRows()` if the current section is `fieldops-unified` (check which panel is active before re-rendering to avoid double-render cost).
