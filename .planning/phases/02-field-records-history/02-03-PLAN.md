  <name>Task 1: Add manual entry Sheet forms to the history timeline page</name>
  <files>organic-cert/src/app/(app)/fields/[id]/history/page.tsx</files>
  <action>
Add three Sheet (slide-over panel) forms to the history timeline page for manual record entry. Use separate forms per operation type (per research recommendation -- field counts differ: tillage ~3 fields vs harvest 8+ fields).

**Sheet trigger points:**
- The "Add records" button on empty season cards (from Plan 02) opens a type selector, then the appropriate form
- A floating "+ Add Record" button at the top of the timeline (next to the filter bar) opens a type selector dialog, then the form
- The "Edit" button on expanded operation cards opens the appropriate form pre-filled with that record's data

**Type selector:**
- Small dialog or popover with 3 options: Tillage, Application, Harvest
- Each option has its badge color and icon (matching timeline badges)
- Selecting one opens the corresponding Sheet form
- When triggered from a season card, pre-fill the crop year context

**All forms share these patterns (per user decisions):**
- Field is pre-determined (we are on `/fields/[id]/history`, so the field is known)
- Enterprise selector: since operations belong to FieldEnterprise (not Field directly), the form needs to select which enterprise (year) to add the record to. Show a select with available enterprises for this field. If triggered from a season card, pre-select that year's enterprise. If no enterprise exists for the target year, show a message: "No crop year record exists for {year}. Create one first." with a link to `/field-enterprises`.
- Date input: default to today (`format(new Date(), "yyyy-MM-dd")` from date-fns). Persist last-used date in `localStorage` key `lastEntryDate`.
- After successful save: `toast.success("Record saved")` via sonner, clear form (but keep the selected enterprise and date per user decision), show "Add Another" button that keeps the Sheet open
- Validation: all required fields checked before submit. Show inline error messages on empty required fields.

**Tillage form (Sheet):**
- Title: "Add Tillage Operation"
- Fields: Operation type select (from FieldOpType enum: TILLAGE, CULTIVATION, or free-text input for other types), Date (date input), Notes (optional textarea)
- POST to `/api/field-enterprises/{enterpriseId}/operations` with body: `{ type: selectedType, operationDate: date, notes, dataSource: "MANUAL" }`
- Minimal form -- 3 fields as per user decision ("Operation type only plus date and field. No depth or additional details required")

**Application form (Sheet):**
- Title: "Add Input Application"
- Fields: Product selector (searchable -- use cmdk Command component to search materials from `/api/materials`), Application rate (number input), Rate unit selector (lbs/acre, gal/acre, oz/acre, units/acre), Application date (date input), Acres treated (number, default to enterprise acres), Notes (optional textarea)
- Support multiple products per application: "Add Product" button adds another product row (product + rate + unit). Each product row creates a separate `MaterialUsage` record with the same applicationDate.
- POST to `/api/field-enterprises/{enterpriseId}/applications` for each product row with body: `{ materialId, applicationRate, rateUnit, applicationDate, areaApplied, notes }`
- Use cmdk for product search since the materials list can be large. Fetch materials once on Sheet open and cache.
- "Suggest from recent" section: show last 3 products used in applications for this field (query from the loaded history data, not a separate API call)

**Harvest form (Sheet):**
- Title: "Add Harvest Record"
- Fields: Harvest date (date input), Yield per acre (number), Yield unit (select: bu, lbs, tons), Acres harvested (number, default enterprise acres), Equipment selector (searchable cmdk from `/api/equipment`), Net weight (optional, computed or manual), Notes (optional textarea)
- Equipment selector: cmdk Command component searching equipment from `/api/equipment`. Include an "Add New Equipment" button at the bottom of the cmdk list that opens a nested Dialog with: equipment name (required), equipment type (required, from EquipmentType enum select), then POSTs to `/api/equipment` and refreshes the equipment list.
- Lot number: display a read-only computed lot number preview below the form showing what the auto-generated lot number will be (based on enterprise cropYear, crop, field name). Include a "Lot number will be auto-generated" helper text. If a manual override is desired (per user decision), add a small "Override" link that reveals an editable text input for custom lot number.
- POST to `/api/field-enterprises/{enterpriseId}/harvest` with body: `{ harvestDate, yieldPerAcre, yieldUnit, acresHarvested, equipmentId, netWeight, notes }`. The API (updated in Plan 01) auto-creates the CropLot.
- Display the returned CropLot lot number in the success toast: `toast.success("Harvest saved - Lot: {lotNumber}")`

**Edit mode:**
- When "Edit" button is clicked on an expanded timeline card, open the appropriate Sheet form with all fields pre-filled from the existing record data
- Change the Sheet title to "Edit {Type} Operation"
- Change the save button text to "Update"
- On save, use PUT to the same API endpoint with the record ID
- After save, refresh the timeline data by re-fetching from the history API

**State management:**
- `useState` for: activeSheet ("tillage" | "application" | "harvest" | null), editingRecord (the record being edited, or null for new), selectedEnterpriseId
- Sheet open/close controlled by `activeSheet !== null`
- On Sheet close, reset editingRecord to null
- After any save, re-fetch the history data to show the new/updated record in the timeline

**Styling:**
- Sheet slides in from the right (shadcn Sheet with `side="right"`)
- Sheet width: `sm:max-w-lg` for tillage, `sm:max-w-xl` for application and harvest (more fields need more space)
- Use shadcn Label, Input, Textarea, Select, Button components inside the form
- Form sections separated with `Separator` component
- Submit button at bottom: brand green background, full width
