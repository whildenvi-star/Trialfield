Create the field-centric 3-year history timeline page at `src/app/(app)/fields/[id]/history/page.tsx`.

This is the core Phase 2 page. It fetches from `GET /api/fields/[id]/history?offset={offset}` and renders a vertical timeline. Mark as `"use client"`.

**Page structure:**
1. **Header:** Field name (from API response), breadcrumb (Fields > Field Name > History), year selector
2. **Year selector:** Buttons or select to shift the 3-year window. Default offset=0 (current year and 2 prior). Increasing offset shifts the window back (offset=1 shows currentYear-1, -2, -3). Re-fetches on change.
3. **Season sections:** Loop over the `years` array (NOT over enterprises). For each year, find the matching enterprise from the API response. If no enterprise exists for that year, render an empty season card.
4. **Filter bar** (between header and timeline)
5. **Timeline content** (season cards with operations)

**Season card (enterprise exists):**
- Season header: "Growing Season {year}" with crop name
- Summary stats row: crop planted, total applications count, total acres treated (from enterprise.acres), harvest yield total
- Vertical timeline within the season: all operations (fieldOperations + materialUsages + harvestEvents + fertilityEvents) merged into one array, sorted chronologically by their respective date fields

**Empty season card (no enterprise for year):**
- Header: "Growing Season {year}"
- Body: "No operations recorded" message
- "Add records" button (links to manual entry — for now, just render the button with a placeholder action; Plan 03 will wire it to the Sheet form)

**Operation summary card (collapsed state):**
- Left side: date formatted "MMM d" (e.g., "Apr 15")
- Color-coded badge for operation type using this mapping:
  - Applications/materialUsages: `bg-green-100 text-green-800` (FlaskConical icon)
  - Harvest/harvestEvents: `bg-amber-100 text-amber-800` (Wheat icon)
  - Tillage (fieldOperations where type is TILLAGE/CULTIVATION): `bg-blue-100 text-blue-800` (Tractor icon)
  - Planting: `bg-purple-100 text-purple-800` (Sprout icon)
  - Other fieldOperations: `bg-stone-100 text-stone-800`
- Key metric: For applications: product name + rate. For harvest: yield per acre. For tillage: operation type name. For fertility: product + rate.
- Source indicator: `RefreshCw` icon (size 12, text-stone-400) for `dataSource === "SYNCED"`, `Pencil` icon for `dataSource === "MANUAL"` (or when dataSource is absent on materialUsages/fertilityEvents, default to MANUAL display)
- Click to expand

**Operation detail card (expanded state):**
- All fields for that record type:
  - Application: product name, rate, rate unit, application date, field area treated, notes
  - Harvest: yield per acre, yield unit, acres harvested, equipment used, lot number (from cropLots relation), harvest date, notes
  - Tillage: operation type, date, notes
  - Fertility: product, rate, application date, notes
- Approval provenance for SYNCED records: Parse `syncRunId` from the `notes` field (pattern: `syncRunId: {uuid}`). If found, display "Synced from Case IH FieldOps" with the sync date. For Phase 2, do NOT query SyncedOperation table — just show the sync source label. Full "Approved by [name] on [date]" display is tech debt for Phase 3.
- For MANUAL records, show "Manually entered" with no provenance section
- Notes/annotations: display notes field content below the detail fields if present
- "Edit" button on expanded card (placeholder for now — Plan 03 will wire to manual entry form pre-filled with this record's data)

**Filter bar:**
- Operation type multi-select: All, Applications, Harvest, Tillage, Planting, Other (uses shadcn Select or custom checkboxes)
- Date range: start date + end date inputs (type="date")
- Data source: All, Synced, Manual (select)
- Approval status: All, Approved, Pending (select — for SYNCED records only, based on whether the record has approval provenance)
- Filters applied client-side to the merged operation array before rendering
- "Clear filters" button to reset

**Timeline rendering pattern:**
```

  id: string;
  type: "application" | "harvest" | "tillage" | "planting" | "fertility" | "other";
  date: Date;
  dataSource: "MANUAL" | "SYNCED";
  data: any; // The raw record
}
```
Map each enterprise's operations/applications/harvests/fertility into TimelineItem[], sort by date, then render.

- Import icons from `lucide-react`: `RefreshCw`, `Pencil`, `Wheat`, `FlaskConical`, `Tractor`, `Sprout`, `ChevronDown`, `ChevronRight`, `Filter`, `Printer`
