For same-origin embeds, postMessage is redundant (localStorage sync already works) but harmless — belt-and-suspenders.

  <action>
Organic-cert is the only cross-origin embed (cert.whughesfarms.com). It already has an inline script that reads `localStorage.getItem('mru-theme')` and adds `in-iframe` class. But since it's cross-origin, it reads its OWN localStorage (not the portal's), so the theme won't match unless explicitly synced.

Update the inline `<script>` in organic-cert's layout.tsx to add a `message` event listener when running inside an iframe:

```javascript
if(window!==window.top){
  document.documentElement.classList.add('in-iframe');
  window.addEventListener('message', function(e) {
    if(e.data && e.data.type === 'glomalin-theme') {
      var t = e.data.theme;
      if(t === 'light') {
        document.documentElement.classList.add('light');
        if(document.body) document.body.classList.add('light');
      } else {
        document.documentElement.classList.remove('light');
        if(document.body) document.body.classList.remove('light');
      }
    }
  });
}
```

This goes in the existing dangerouslySetInnerHTML inline script block, extending the current iframe detection logic. The listener only activates when in an iframe (not standalone).

For security: We intentionally use `*` for postMessage origin checking here because:
1. organic-cert only acts on `glomalin-theme` typed messages (namespaced)
2. The action is purely cosmetic (theme toggle)
3. The portal origin may vary between dev/prod environments

However, if the current layout.tsx has the portal origin available via env var, use that for origin validation: `if(e.origin !== 'https://app.whughesfarms.com') return;` — but only if NEXT_PUBLIC_PORTAL_ORIGIN is defined.
