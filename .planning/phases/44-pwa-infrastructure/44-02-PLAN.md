    1. Install idb: `cd glomalin-portal && npm install idb`

    2. Create `src/lib/offline/types.ts` with TypeScript interfaces:

       ```typescript
       // Queued operation — a field pass confirmation or addition waiting to sync
       export interface QueuedOperation {
         id: string;                    // crypto.randomUUID()
         type: 'confirm-pass' | 'add-pass';
         fieldOperationId?: string;     // existing ID for confirm-pass
         fieldEnterpriseId?: string;    // for add-pass
         operationType?: string;        // FieldOpType value (TILLAGE, CULTIVATION, etc.)
         operationDate: string;         // ISO date string
         operatorId: string;            // Supabase user ID
         operatorName: string;          // display name for optimistic UI
         description?: string;          // optional notes
         passNumber?: number;           // which pass number
         acresWorked?: number;
         createdAt: string;             // ISO timestamp when queued
         status: 'pending' | 'syncing' | 'failed' | 'synced';
         errorMessage?: string;         // populated on failure
         retryCount: number;
       }

       // Cached crop plan — field + enterprise + inputs + passes snapshot
       export interface CachedCropPlan {
         fieldId: string;               // key — farm-registry field ID
         fieldName: string;
         crop: string;
         variety?: string;
         acres: number;
         enterprise?: string;
         inputs: Array<{
           product: string;
           rate: string;
           unit: string;
         }>;
         passes: Array<{
           id: string;
           type: string;
           passNumber?: number;
           status: 'PLANNED' | 'CONFIRMED';
           operationDate?: string;
           operatorName?: string;
         }>;
         cachedAt: string;              // ISO timestamp of cache write
       }

       // DB schema type for idb
       export interface OfflineDB {
         'operation-queue': {
           key: string;
           value: QueuedOperation;
           indexes: { 'by-status': string };
         };
         'crop-plan-cache': {
           key: string;
           value: CachedCropPlan;
         };
       }
       ```

    3. Create `src/lib/offline/db.ts`:
       - Import `openDB, DBSchema, IDBPDatabase` from 'idb'
       - Import types from './types'
       - Define DB_NAME = 'glomalin-offline', DB_VERSION = 1
       - `getDb()` function: opens/creates the database with upgrade callback that creates both object stores:
         - 'operation-queue' store with keyPath 'id' and index 'by-status' on 'status' field
         - 'crop-plan-cache' store with keyPath 'fieldId'
       - Singleton pattern: cache the db promise so multiple calls reuse the same connection
       - Export `offlineQueue` object with methods:
         - `add(op: Omit<QueuedOperation, 'id' | 'createdAt' | 'status' | 'retryCount'>)` — generates id, sets createdAt/status/retryCount defaults, puts into store
         - `getAll()` — returns all queued operations
         - `getPending()` — uses index 'by-status' to get only status='pending' ops
         - `update(id: string, updates: Partial<QueuedOperation>)` — get + merge + put
         - `delete(id: string)` — delete from store
         - `clear()` — clear all entries
       - Export `cropPlanCache` object with methods:
         - `put(plan: CachedCropPlan)` — upserts by fieldId, sets cachedAt to now
         - `get(fieldId: string)` — get single plan by key
         - `getAll()` — returns all cached plans
         - `clear()` — clear all entries
         - `getLastSyncTime()` — reads all entries, returns most recent cachedAt or null
       - All methods are async and handle the case where IndexedDB is unavailable (SSR guard: check `typeof indexedDB !== 'undefined'` before operations)
