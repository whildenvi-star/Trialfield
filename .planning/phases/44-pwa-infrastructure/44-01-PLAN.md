  <action>
    1. Install packages: `cd glomalin-portal && npm install @serwist/next serwist`

    2. Create `src/sw.ts` — the service worker entry point:
       - Import from `serwist` (defaultCache, Serwist)
       - Create Serwist instance with `precacheEntries: self.__SW_MANIFEST` (populated by @serwist/next at build time)
       - Add runtime caching routes using defaultCache as base, then add:
         - NavigationRoute with NetworkFirst strategy (falls back to cached app shell)
         - Static assets (JS/CSS/fonts/images) with StaleWhileRevalidate strategy
       - Call `serwist.addEventListeners()` to wire up install/activate/fetch handlers
       - Type the service worker scope: `declare const self: ServiceWorkerGlobalScope`

    3. Update `next.config.mjs` to wrap config with @serwist/next:
       - Import `withSerwist` from `@serwist/next`
       - Wrap the existing config: `withSerwist({ swSrc: "src/sw.ts", swDest: "public/sw.js", disable: process.env.NODE_ENV === "development" })(nextConfig)`
       - Keep `output: 'standalone'` in the base config
       - Note: disable in dev mode to avoid service worker caching dev assets

    4. Create `public/manifest.json`:
       ```json
       {
         "name": "Glomalin",
         "short_name": "Glomalin",
         "description": "Farm Operations Portal",
         "start_url": "/",
         "display": "standalone",
         "background_color": "#080604",
         "theme_color": "#080604",
         "icons": [
           { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
           { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" }
         ]
       }
       ```
       Use the dark soil background color (#080604) from the Glomalin design system.

    5. Generate placeholder PNG icons (192x192 and 512x512) in `public/icons/`:
       - Use a simple canvas-based Node.js script (or ImageMagick if available) to create solid #080604 background with "G" letter in #C8860A (accent color) centered
       - These are functional placeholders — user can replace with branded icons later

    6. Optionally create `src/app/manifest.ts` as a Next.js dynamic manifest route (if @serwist/next recommends it over static). If the static `public/manifest.json` approach works, skip the dynamic route and just use the static file.

    7. Update `tsconfig.json` if needed to include `@serwist/next` types or `WebWorker` lib for service worker types. Add `"WebWorker"` to `compilerOptions.lib` if `ServiceWorkerGlobalScope` is not recognized.
