  <action>
Read each page file (and any referenced client components if they contain the layout logic). Audit for mobile breakage against these criteria:

**Audit checklist per page:**
1. Any fixed width values not responsive (e.g., `w-[800px]`, `min-w-[600px]`) — replace with responsive max-w + overflow-hidden
2. Multi-column grid layouts with no mobile fallback — add `grid-cols-1 md:grid-cols-N`
3. `overflow-x-auto` tables with many columns — if 5+ columns, apply Pattern 5 (table-to-card) from RESEARCH.md; if fewer than 5 columns, `overflow-x-auto` is acceptable
4. Button/tab touch targets below 44px — add `min-h-[44px]` to interactive elements
5. Hardcoded `left-[220px]` or `ml-[220px]` without responsive prefix — add `md:` prefix

**Per-page specific guidance from RESEARCH.md:**

- `compliance/page.tsx` — May render `ComplianceShell`. Read the file; if it renders tabs/buttons, check touch target sizing. Add `min-h-[44px]` to any tab or button element that is shorter. If compliance page renders `ComplianceShell` from `src/components/compliance/`, read that component too and fix touch targets there.

- `field-ops/page.tsx` — Field Operations TC Log. Read the file. Likely has a table or list. If table has more than 5 columns: apply card pattern. If single-column list: likely fine, just verify no fixed widths.

- `marketing/page.tsx` — Grain Marketing. Read the file. May have price tables or charts. If table: audit columns. If chart/canvas: ensure `w-full` and `max-w-full` so it does not overflow.

- `weather/page.tsx` — Precipitation. RESEARCH.md notes "uses p-4 md:p-6 max-w-* — likely adequate." Read the file to confirm. If already responsive, no changes needed.

- `field-timeline/page.tsx` — Field Timeline. Read the file. Timeline components may use horizontal scroll or absolute positioning. If so, ensure `overflow-x-hidden` on the container and timeline items stack vertically on mobile (`flex-col sm:flex-row`).

- `field-history/page.tsx` — RESEARCH.md notes this already uses card pattern Links and is likely adequate. Read the file briefly to confirm no horizontal scroll. If adequate: no changes needed to this file.

**For each page that needs changes:**
- Make targeted, minimal Tailwind responsive class additions
- Do NOT redesign pages — only add responsive modifiers to existing classes
- Do NOT change any business logic or data fetching
- Prefer `md:hidden` + `hidden md:block` for card/table alternatives when a full conversion is warranted
- Prefer `grid-cols-1 md:grid-cols-N` for multi-column layouts that just need mobile stacking

After auditing all pages, run `npx tsc --noEmit` to confirm no errors introduced.
