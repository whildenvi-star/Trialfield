<objective>
Fix the two production env gaps that cause portal CLU record creation to fail silently on every field add from farm-registry.

Purpose: The portal webhook at /api/fsa/webhook/field-created requires `?token=<EMBED_TOKEN>` for auth, but propagateField() never appends it — every call returns 403 in any environment where EMBED_TOKEN is set. Additionally, PORTAL_URL is missing from farm-registry/.env so it defaults to localhost:3010. Together these mean AUTO-01 and AUTO-02 are satisfied only in local dev, never in production.

Output: One-line code fix in farm-registry/server.js + two env var additions. All three targets in propagateField() are now symmetric. CLU placeholder records created in production on field add.
</objective>
