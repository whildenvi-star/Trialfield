**Schema changes (4 additions):**

1. Add `nopResolved Boolean @default(false)` to `Material` model — after the `nopStatus` field. This is the flag that distinguishes "user has explicitly reviewed and assigned NOP status" from "defaulted to APPROVED without review". Existing Material records get `false` (correct — they were never explicitly reviewed via compile workflow).

2. Add `dataSource DataSource @default(MANUAL)` to `MaterialUsage` model — after the `notes` field. Existing manual entries get MANUAL (correct). Compiled records will set SYNCED.

3. Add `dataSource DataSource @default(MANUAL)` to `SeedUsage` model — after the `notes` field. Same logic as MaterialUsage.

4. Add `@@unique([farmId, crop, variety])` to `SeedLot` model — after the existing `seedUsages` relation. This enables idempotent upsert for seed compilation instead of brittle findFirst+create.

**input-mapper.ts — `mapInputs(cropYear, farmId)`:**

Core logic:
1. Fetch `getBudgetProducts()` and `getBudgetFieldsWithInputs()` in parallel via `Promise.all`.
2. Build a product lookup map: `productName -> BudgetProduct`.
3. Get organic FieldEnterprise IDs for the cropYear from Prisma: `prisma.fieldEnterprise.findMany({ where: { field: { farmId }, cropYear }, include: { field: { select: { name: true, farmBudgetFieldName: true } } } })`.
4. Build field-to-enterprise map: `(lowercaseBudgetFieldName OR lowercaseFarmBudgetFieldName) -> FieldEnterprise`.
5. Filter budget fields to organic only: `systemCode.startsWith("ORG")`.
6. For each organic field's `inputs[]`:
   a. Resolve field to FieldEnterprise using the map (try `field.name.toLowerCase()` first, then check `farmBudgetFieldName` mappings).
   b. If no FieldEnterprise match, skip (field not compiled in Phase 16 yet).
   c. Resolve `input.productName` to a Material via `prisma.material.findFirst({ where: { farmId, name: input.productName } })`.
   d. If no Material exists, record it as `materialId: null` — the commit step will create the stub.
   e. Build `InputPreviewRow` with `action: "new"` (first compile) or compare against existing `MaterialUsage` records.
7. For determining action (new vs update vs unchanged):
   - Query existing `MaterialUsage` for each enterprise where `dataSource === "SYNCED"`.
   - If no matching SYNCED usage exists for this enterprise+material, `action = "new"`.
   - If a matching SYNCED usage exists and quantity differs, `action = "update"`.
   - If identical, `action = "unchanged"`.
8. Get unit from `BudgetProduct` lookup: `product?.unit ?? "units"`.

The function returns `InputCompileResult` with preview rows, summary counts, and count of unresolved materials.

**Commit logic for inputs (inside the POST route, not mapper):**

When `preview === false` (commit mode):
1. Call `mapInputs()` to get current state.
2. Check FieldEnterprise records exist: if 0 found, return 400 "Compile enterprises first".
3. In a `prisma.$transaction`:
   a. For each `InputPreviewRow` where `materialId === null` (new product): upsert Material using `prisma.material.upsert({ where: { farmId_name: { farmId, name: productName } }, create: { farmId, name: productName, nopStatus: "APPROVED", nopResolved: false, category: "OTHER", notes: "Compiled from farm-budget — assign NOP status" }, update: {} })`. Store returned materialId.
   b. `deleteMany` all SYNCED MaterialUsage records for target enterprises.
   c. `createMany` new MaterialUsage records with: `materialId` (resolved), `fieldEnterpriseId`, `applicationDate` (synthesized from season: Spring = `${year}-04-01T12:00:00Z`, Fall = `${year-1}-10-15T12:00:00Z`), `rate: input.quantity`, `rateUnit: product.unit || "units"`, `acres: field.acres`, `dataSource: "SYNCED"`, `notes: "Compiled from farm-budget season:${input.season}"`.

**CRITICAL: Never overwrite NOP status.** The `Material.upsert.update` block MUST be empty `{}` or only touch non-NOP fields. Once `nopResolved: true`, the user's assignment persists.

**seed-mapper.ts — `mapSeeds(cropYear, farmId)`:**

Core logic:
1. Fetch `getBudgetProducts()` is not needed for seeds. Fetch `getBudgetFieldsWithInputs()` (has `seed` field) and fetch `GET /api/seeds` from budget via a new helper. 2. Build seed lookup: `(variety.toLowerCase()) -> BudgetSeed`. NOTE: Research says match on `(crop, variety)` but the field data only has `variety`, not `crop` in the seed object. The crop comes from the field's crop. So match `field.crop` normalized + `field.seed.variety` against `seeds[]` entries.
   - Normalization needed: budget field crops are like "ORG Blue Corn", "ORG Wheat", "ORG Soybeans" — strip "ORG " and "ORG IRR " prefix, then match against seeds[].crop. E.g., "ORG Wheat" -> "Wheat", "ORG Blue Corn" -> "Blue_Corn" (check actual seeds data: "Blue_Corn" uses underscore). Build a simple normalizer: `strip ORG prefix, strip IRR, trim, replace spaces with underscores for comparison`.
   - Actually, inspecting the data: seeds have `crop: "Blue_Corn"` while fields have `crop: "ORG Blue Corn"`. The normalizer should strip the ORG/IRR prefix AND try both space and underscore variants.
3. For each organic field with a non-null `seed`:
   a. Resolve field to FieldEnterprise (same as input-mapper).
   b. Find matching BudgetSeed by (normalizedCrop, variety) — if not found, create SeedLot stub with null brand/supplier.
   c. Upsert SeedLot via `prisma.seedLot.upsert({ where: { farmId_crop_variety: { farmId, crop: normalizedCrop, variety } }, create: { farmId, crop: normalizedCrop, variety, brand: matchedSeed?.brand ?? null, supplier: null, isOrganic: true, isUntreated: true, commercialAvailDoc: false, notes: "Compiled from farm-budget" }, update: {} })`. Note: the crop stored in SeedLot should be the normalized form (e.g., "Wheat", "Blue Corn"), not the prefixed form.
   d. Build `SeedPreviewRow` with action based on existing SeedUsage records (same new/update/unchanged logic as inputs).
