**nop-compliance.ts — stateless compliance check functions:**

Create `organic-cert/src/lib/compile/nop-compliance.ts` with two exported functions:

1. `checkMaterialCompliance(material, context?)` — evaluates a single Material against NOP rules:

```typescript
export interface NopCheckResult {
  materialId: string;
  materialName: string;
  verdict: "pass" | "fail" | "restricted" | "needs-review";
  reasons: string[];
  dateEstimated?: boolean;  // true when using season-derived dates
}

export function checkMaterialCompliance(
  material: { id: string; name: string; nopStatus: string; nopResolved: boolean; omriListed: boolean; category: string },
  context?: { applicationDate?: Date; harvestDate?: Date; isComposted?: boolean; cropContactsSoil?: boolean }
): NopCheckResult
```

Logic:
- If `!material.nopResolved` → return `{ verdict: "needs-review", reasons: ["NOP status not yet assigned"] }`.
- If `material.nopStatus === "PROHIBITED"` → return `{ verdict: "fail", reasons: ["Material is PROHIBITED under NOP (7 CFR 205)"] }`.
- If `material.nopStatus === "RESTRICTED"`:
  a. Check if `material.category === "FERTILIZER"` AND context has applicationDate + harvestDate → call `checkManureDayRule()` from `@/lib/day-rule-calc`. If `!dayRule.isCompliant`, return `{ verdict: "fail", reasons: [dayRule.warning] }`. If compliant, return `{ verdict: "restricted", reasons: ["Restricted material — verify application conditions are met"] }`.
  b. If no date context or not FERTILIZER category, return `{ verdict: "restricted", reasons: ["Restricted — application conditions must be verified for NOP compliance"] }`.
- If `material.nopStatus === "APPROVED"` or `material.nopStatus === "EXEMPT"` → return `{ verdict: "pass", reasons: [] }`.
- Append `"OMRI Listed"` to reasons array when `material.omriListed === true` (informational, does not change verdict).

2. `checkSeedCompliance(seedLot)` — evaluates a SeedLot:

```typescript
export interface SeedCheckResult {
  seedLotId: string;
  variety: string;
  verdict: "pass" | "review" | "needs-doc";
  reasons: string[];
}

export function checkSeedCompliance(
  seedLot: { id: string; variety: string; isOrganic: boolean; isUntreated: boolean; commercialAvailDoc: boolean }
): SeedCheckResult
```

Logic per NOP §205.204:
- If `seedLot.isOrganic === true` → `{ verdict: "pass", reasons: ["Organic seed"] }`.
- If `seedLot.isOrganic === false && seedLot.isUntreated === true`:
  - If `seedLot.commercialAvailDoc === true` → `{ verdict: "pass", reasons: ["Non-organic untreated — commercial availability documented"] }`.
  - If `seedLot.commercialAvailDoc === false` → `{ verdict: "needs-doc", reasons: ["Non-organic untreated — needs commercial availability documentation"] }`.
- If `seedLot.isOrganic === false && seedLot.isUntreated === false` → `{ verdict: "review", reasons: ["Non-organic treated seed — prohibited unless emergency variance granted"] }`.

IMPORTANT: Both functions are pure — no Prisma imports, no DB access. They receive plain objects and return plain results. This makes them callable from both server routes and client components.

**POST /api/materials/batch-resolve route:**

Create `organic-cert/src/app/api/materials/batch-resolve/route.ts`.

Purpose: Batch-update NOP status for multiple materials in one request — supports the "Save All" button on the unresolved materials panel.

Request body: `{ resolutions: Array<{ id: string; nopStatus: "APPROVED" | "RESTRICTED" | "PROHIBITED" | "EXEMPT"; notes?: string }> }`

Logic:
1. Validate body: `resolutions` must be a non-empty array. Each item needs `id` (string) and `nopStatus` (one of the 4 valid values). Return 400 if invalid.
2. In a `prisma.$transaction`, for each resolution:
   a. `prisma.material.update({ where: { id }, data: { nopStatus: resolution.nopStatus, nopResolved: true, notes: resolution.notes ?? undefined } })`.
   b. Log audit via `logAudit()` (import from `@/lib/audit-logger`) with action "UPDATE", entityType "Material".
3. Return `{ updated: resolutions.length }`.

Use the same error handling pattern as `organic-cert/src/app/api/materials/[id]/route.ts`: try/catch, return 500 with error message on failure.
