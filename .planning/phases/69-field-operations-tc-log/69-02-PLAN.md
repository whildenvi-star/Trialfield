**File 1: `glomalin-portal/src/app/(protected)/app/field-ops/page.tsx`**

Server component. Mirrors field-timeline/page.tsx structure:

1. Fetch all fields from farm-registry via `fetchRegistryService('/api/fields?active=true')` (import from `@/app/api/mobile/_lib/proxy`). Map to `{ id, name, aliases, reportingAcres }`.
2. Graceful error: if registry unavailable, render a centered error card (same style as field-timeline).
3. Wrap `FieldOpsClient` in `<Suspense fallback={null}>` — required because FieldOpsClient uses useSearchParams.
4. Pass `fields` and `initialFieldId` (from `searchParams.field`) as props.
5. Outer div: `className="flex h-screen bg-glomalin-bg text-glomalin-text overflow-hidden"`.

**File 2: `glomalin-portal/src/app/(protected)/app/field-ops/field-ops-client.tsx`**

`'use client'` component. Props: `{ fields: RegistryField[], initialFieldId: string | null }`.

**Layout:** Same split-panel as field-timeline — 288px left sidebar, flex-1 right workspace.

**Left sidebar:**
- Header: "Field Ops TC Log" (font-mono, sm, semibold) + "TC Sign-off Log" (xs, muted).
- Below header: search input (font-mono, xs) that filters the field list by name/alias. Use local `useState` for search text.
- Field list: scrollable, each field is a button styled like field-timeline's FieldList items. Show field name + reportingAcres in muted text. Highlight selected field with accent border-left.

**Right workspace — when no field selected:**
- Centered muted text: "Select a field to view and add TC records".

**Right workspace — when field selected:**
- Header bar: field name (sm, semibold, font-mono) + year selector dropdown (current year default; options: current year and 3 prior years, e.g., 2026, 2025, 2024, 2023). Use `useState` for selectedYear, default `new Date().getFullYear()`.
- "Add TC" button (accent-colored, top-right of header bar). Opens add form inline (not modal) below the header.

**Add TC inline form** (shown when addFormOpen state is true):
- Fields:
  - **Operation Type** — `<select>` with options: Tillage, No-Till, Planting, Herbicide, Fertilizer, Scouting, Harvest, Hauling, Other.
  - **Date** — `<input type="date">` defaulting to today (format YYYY-MM-DD).
  - **TC'd by** — display logged-in user's name. Use `createBrowserClient` from `@supabase/ssr` to get user session on mount, store `{ id, email, fullName }` in state. Show name as read-only text below a label.
  - **Sign off as** — `<select>` populated from GET `/api/field-ops/operators`. Fetch once when form opens (if not already loaded). Options: "Self (default)" + each operator name. If a non-self operator is selected, append that operator's name to the notes field as `"[Signed off by: {operatorName}]"` when submitting. Do NOT attempt to use cert_user_id — the operators route returns `{ id, fullName, role }` only, and the sign-off identity is stored via notes append.
  - **Notes** — `<textarea>` optional, 2 rows.
- Buttons: "Save TC" (accent) + "Cancel" (muted). On Save: POST to `/api/field-ops/tcs`, optimistically add to TC list with a temp entry, replace with real data on response. Show inline error text if POST fails.

**TC list / table:**
- Loading state: "Loading..." muted text.
- Empty state: "No TC records for this field and year."
- Each TC row (use table rows — simpler for desktop):
  - Columns: Date | Operation Type | TC'd By | Notes | Delete
  - Date formatted as `MMM D, YYYY` (use `new Date(op.operationDate).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })` — no date-fns needed).
  - TC'd By: `tcByName` or "Unknown".
  - Delete button: trash icon (use inline SVG — no icon library). Only visible/enabled if the logged-in user created the TC (compare current user supabase `id` with a `createdBySupabaseId` field, OR if role is admin. Since cert does not store supabase IDs, use a simpler heuristic: any non-admin sees the Delete button only on rows where `tcByName` matches their `fullName`. Admins see Delete on all rows. On click: show inline confirmation "Delete this TC? " + "Yes" / "Cancel" inline on the row. On Yes: DELETE `/api/field-ops/tcs/{id}?fieldEnterpriseId={feid}&tcByCertUserId={tcByCertUserId}`.

**Data fetching:**
- On field or year change: fetch `GET /api/field-ops/tcs?fieldId={id}&year={year}`. Store in `useState<TcRecord[]>`. The GET response includes `fieldEnterpriseId` at the top level so the delete call can pass it as a query param.
- Get current user role + fullName: use `createBrowserClient` from `@supabase/ssr` — call `supabase.auth.getUser()` on mount, then query the profiles table for role and full_name. Store `{ id, role, fullName }` in state.

**URL state:** Sync selected field to `?field=` search param using `useRouter` + `useSearchParams` (same as field-timeline pattern).

**Styling:** Font-mono throughout. Glomalin design tokens (bg-glomalin-bg, bg-glomalin-surface, border-glomalin-border, text-glomalin-text, text-glomalin-muted, text-glomalin-accent). Table: `w-full text-xs font-mono`, `th` with border-b, `td` with py-2 px-3. Add TC button: `bg-glomalin-accent text-black px-3 py-1.5 text-xs font-mono rounded hover:opacity-90`.

TypeScript: Explicit types for TcRecord `{ id: string, operationType: string, operationDate: string, notes: string | null, tcByName: string | null, tcByCertUserId: string | null }` and response shape `{ tcs: TcRecord[], fieldEnterpriseId: string, year: number, noEnterprise?: boolean }`. No `any` without eslint-disable comment.
