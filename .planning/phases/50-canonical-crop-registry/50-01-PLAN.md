  <name>Task 1: Define crop schema and seed initial crop records in data.json</name>
  <files>farm-registry/data/data.json</files>
  <action>
Add a `crops` array to farm-registry/data/data.json alongside the existing `growers` and `fields` arrays. Each crop record has:

```json
{
  "id": "crop_001",          // Stable ID (crop_NNN format)
  "name": "Yellow Corn",     // Canonical display name (human-readable, per user decision)
  "category": "Corn",        // Grouping category for charts/reports (Claude's discretion: adding this for farm-budget chart grouping)
  "organic": false,          // Boolean organic flag (per user decision — NOT baked into name)
  "bushelWeight": 56,        // lb/bu (null for crops measured in tons/other)
  "unit": "Bu",              // Default unit: "Bu", "Tons", "Cwt"
  "color": "#1565c0",        // Hex color for consistent cross-app charts
  "aliases": ["Yellow Corn", "YC"],  // Known aliases for backfill matching
  "rmaCropCode": null,       // Optional RMA crop code
  "active": true             // Soft delete flag
}
```

Build the initial crop list by scanning these existing data sources:

**farm-budget cropTypes[].subCrops[]** (data.json ~line 15801+):
- Corn category: Yellow Corn, Seed Corn, ORG Yellow Corn, ORG White Corn, ORG Blue Corn, RR Corn, ORG Seed Corn, ORG Food Corn, White Corn
- Soybeans category: RR Soybeans, Soybeans, High Oil Soybeans, ORG Soybeans, GA ORG SEED BEANS, ORG SEED BEANS, Scoular Soybeans, DF Seed Beans, ORG Natto Beans
- Wheat category: Wheat, ORG Wheat, ORG Seed Wheat
- Rye category: Milling Rye, Hybrid Seed Rye, ORG Rye
- Sweet Corn category: IRR Sweet Corn, Org Sweet Corn, DRY Land Sweet Corn
- Food Beans category: Lima Beans, Snap Beans, ORG Snap Beans, ORG Food Beans
- Peas category: Peas (and any sub-crops)
- Plus any other cropTypes found in data.json

**grain-tickets CropConfig** (Prisma DB — cropName strings like "Hybrid Rye", "Organic Yellow Corn")
**portal FSA_CROP_LIST** (fsa-crop-list.ts — "Hybrid Rye", "Organic Corn", etc.)
**seed-inventory** (product crops)

For each unique crop:
1. Pick the canonical name: full human-readable form (e.g., "Soft Red Winter Wheat" not "SRWW")
2. Set `organic: true` for crops with ORG/Organic prefix — strip the prefix from canonical name. Example: "ORG Yellow Corn" becomes `{ name: "Yellow Corn", organic: true }`
3. Build aliases array from ALL known name variants across apps (case variations, abbreviations, prefixes)
4. Copy bushelWeight from farm-budget cropTypes (testWeight field), color from cropTypes.color
5. Use crop_NNN format for IDs (crop_001, crop_002, etc.)

IMPORTANT: Each meaningfully different crop is a SEPARATE record per user decision. "Yellow Corn" and "White Corn" are separate. "Seed Corn" is separate from "Yellow Corn". But "ORG Yellow Corn" is NOT separate — it's "Yellow Corn" with organic=true.

EXCEPTION to organic rule: When the organic version has fundamentally different pricing/market (e.g., organic natto beans vs regular soybeans), keep as separate crop record with organic=true. Use judgment — the goal is that each record represents a crop that would be tracked separately in grain tickets and budget.

The aliases array is critical — include EVERY name variant found in the data: "ORG Yellow Corn", "Organic Yellow Corn", "Organic Corn", "YC", etc.

FSA land-use categories (Idle, Fallow, CRP, Cover Crop) are NOT crops per user decision — exclude them.

<task type="auto">
  <name>Task 2: Add /api/crops CRUD endpoints to farm-registry server</name>
  <files>farm-registry/server.js</files>
  <action>
Add crop CRUD endpoints to farm-registry/server.js, following the same patterns used for the existing /api/fields endpoints:

**GET /api/crops** — Returns all active crops (filter `active !== false`). Supports optional `?q=` query param for search/autocomplete (case-insensitive match on name or any alias). Returns array of crop objects.

**GET /api/crops/autocomplete** — Lightweight endpoint returning `[{id, name, category, organic, aliases}]` for dropdown population (same pattern as /api/fields/autocomplete). Place BEFORE the `:id` route to prevent Express param shadowing.

**GET /api/crops/:id** — Returns single crop by ID. 404 if not found.

**POST /api/crops** — Creates new crop. Auto-generates `crop_NNN` ID (find max existing NNN + 1). Requires `name` at minimum. Defaults: `organic: false`, `active: true`, `aliases: [name]`, `unit: "Bu"`. Validates name is non-empty and not duplicate.

**PUT /api/crops/:id** — Updates crop. All fields except `id` are updatable. Returns updated crop.

**DELETE /api/crops/:id** — Soft-delete (sets `active: false`). Returns 204.

Use the same JSON file read/write pattern as existing field endpoints (read data.json, modify in memory, write back with backup). Follow existing error handling patterns.

Add CORS headers (already global in the server).
