  <name>Task 1: Create farm-budget and fsa-acres crop ID backfill scripts</name>
  <files>farm-budget/backfill-crop-ids.js, fsa-acres/backfill-crop-ids.js</files>
  <action>
Follow the EXACT same pattern as the field ID backfill scripts from Phase 49 (farm-budget/backfill-field-ids.js, fsa-acres/backfill-field-ids.js). Each script is self-contained with duplicated normalize+alias matching logic.

**Normalize function:** `trim + collapse whitespace + toLowerCase` — same as field backfill.

**farm-budget/backfill-crop-ids.js:**
1. Fetch crop list from `http://localhost:3005/api/crops`
2. Build alias lookup map: `normalized alias string -> [{cropId, cropName}]`
3. Process `data.json` records that need crop IDs:
   - `enterprises[]` — match `enterprise.crop` string against aliases. Add `registryCropId` field.
   - `cropTypes[].subCrops[]` — match `subCrop.name` against aliases. Add `registryCropId` field.
4. Handle organic prefix variations: normalize should strip "ORG ", "Organic ", "Org " prefixes for matching, but also try the full string first (exact match takes priority).
5. Dry-run (default): print matched/unmatched/skipped stats + write `backfill-crop-report.json`
6. `--commit` flag: write changes to data.json (with backup, same as field backfill)
7. Idempotent: skip records that already have `registryCropId`

**fsa-acres/backfill-crop-ids.js:**
1. Fetch crop list from `http://localhost:3005/api/crops`
2. Build alias lookup map
3. Process `data.json` CLU records: match `clu.crop` string against aliases. Add `registryCropId` field.
4. Same organic prefix normalization
5. Same dry-run/commit pattern, same report output
6. Deduplicate unmatched list on unique crop names (many CLUs share the same crop)

Both scripts: print summary stats to console, write full results to `{app-dir}/backfill-crop-report.json`.
