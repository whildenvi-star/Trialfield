  <action>
Create scripts/backup.sh as a bash script that:

1. **Configuration section at top:**
   - BACKUP_ROOT="/var/backups/farm-ops"
   - APP_ROOT="/srv/farm-ops" (canonical deploy path from DEPLOY.md)
   - RETENTION_DAYS=7
   - LOG_FILE="/var/log/farm-ops-backup.log"
   - TIMESTAMP=$(date +%Y-%m-%d)
   - BACKUP_DIR="$BACKUP_ROOT/$TIMESTAMP"

2. **Logging function:**
   - log() that writes timestamped messages to both stdout and LOG_FILE
   - Log start time, each step, any errors, and end time

3. **Create backup directory:**
   - mkdir -p "$BACKUP_DIR"/{json,postgres}

4. **JSON data file backups (5 apps):**
   - Copy each data.json with app-prefixed name:
     - farm-budget/data/data.json -> $BACKUP_DIR/json/farm-budget.json
     - farm-registry/data/data.json -> $BACKUP_DIR/json/farm-registry.json
     - fsa-acres/data/data.json -> $BACKUP_DIR/json/fsa-acres.json
     - meristem-malt/data/data.json -> $BACKUP_DIR/json/meristem-malt.json
     - seed-inventory/data/data.json -> $BACKUP_DIR/json/seed-inventory.json
   - Use cp (not rsync -- simpler, files are small)
   - Log success/failure for each file

5. **PostgreSQL database backups (2 local DBs):**
   - Source credentials from each app's .env file (DATABASE_URL)
   - pg_dump --format=custom for grain-tickets DB -> $BACKUP_DIR/postgres/grain-tickets.dump
   - pg_dump --format=custom for organic-cert DB -> $BACKUP_DIR/postgres/organic-cert.dump
   - Use --format=custom (compressed, supports selective restore)
   - Log success/failure for each dump

6. **Supabase PostgreSQL backup (1 remote DB):**
   - Source SUPABASE_DB_URL from glomalin-portal .env file
   - pg_dump --format=custom -> $BACKUP_DIR/postgres/glomalin-portal.dump
   - This is the MOST CRITICAL backup -- Supabase free tier has NO built-in backups
   - Log with emphasis that this is the only copy of portal data

7. **Retention cleanup:**
   - find $BACKUP_ROOT -maxdepth 1 -type d -mtime +$RETENTION_DAYS -exec rm -rf {} \;
   - Log which directories were cleaned up (if any)

8. **Summary:**
   - Log total backup size (du -sh $BACKUP_DIR)
   - Log completion time and status (success/partial/failed based on error count)

9. **Cron installation instructions in a comment block at top:**
   ```
   # Install: crontab -e
   # 0 2 * * * /srv/farm-ops/scripts/backup.sh
   ```

Make the script executable (chmod +x). Use set -euo pipefail at top. Track error count so partial failures don't abort the entire script (use individual error trapping, not global set -e for the backup steps -- wrap each in an if/then with error counter).

Actually: use set -uo pipefail (NOT set -e) so individual step failures increment error counter rather than aborting. Each backup step should be wrapped in: if command; then log success; else log error; increment error counter; fi.

**restore.sh:**

Create scripts/restore.sh that:

1. **Usage:** restore.sh [YYYY-MM-DD] [--component json|postgres|all]
   - First arg: backup date (required)
   - Second arg: optional component filter (default: all)
   - If no args, print usage and list available backups (ls $BACKUP_ROOT)

2. **Validation:**
   - Check backup directory exists for given date
   - Confirm with user before proceeding: "This will overwrite current data. Continue? [y/N]"

3. **JSON restore:**
   - Copy each prefixed JSON back to its app data directory
   - farm-budget.json -> $APP_ROOT/farm-budget/data/data.json
   - (same pattern for all 5 apps)

4. **PostgreSQL restore (local):**
   - pg_restore --clean --if-exists for grain-tickets and organic-cert
   - Source connection strings from .env files

5. **Supabase restore:**
   - pg_restore --clean --if-exists using SUPABASE_DB_URL from glomalin-portal .env
   - Print WARNING that this overwrites production Supabase data

6. **Post-restore:**
   - Suggest restarting PM2: "Run: pm2 restart all"
   - Log restore completion

Make executable (chmod +x).
