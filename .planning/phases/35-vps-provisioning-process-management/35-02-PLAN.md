This avoids conflict with glomalin-portal which uses port 3000.

**Part B — CORS lockdown (SEC-01):**

Add environment-driven CORS middleware to all 6 Express apps. Each app must read `PORTAL_ORIGIN` from environment and restrict cross-origin requests.

Pattern to implement in each server.js (add near the top, after express() initialization):

```javascript
const corsOptions = {
  origin: process.env.PORTAL_ORIGIN || 'http://localhost:3000',
  credentials: true
};
```

For apps that already import and use `cors` package (farm-registry, seed-inventory):
- Replace `app.use(cors())` with `app.use(cors(corsOptions))`
- The cors package is already in their package.json

For apps that do NOT currently use cors (grain-tickets, farm-budget, fsa-acres, meristem-malt):
- Add `const cors = require('cors');` at the top (after express import)
- Add `app.use(cors(corsOptions));` before route definitions
- Check each app's package.json for the cors dependency. If cors is not listed, add a comment in the .env.example noting `npm install cors` is needed, and add it to the package.json dependencies.

IMPORTANT: For development mode, the fallback `http://localhost:3000` allows the portal to call Express APIs locally. In production, `PORTAL_ORIGIN` will be set to `https://portal.farm.example.com` (or the real domain).

Keep any existing CORS-related proxy routes intact (farm-registry has an FSA proxy route — do not remove it).

For grain-tickets specifically: it uses the Anthropic SDK for the Glomalin chat agent. The CORS middleware does NOT interfere with outbound API calls to Anthropic — CORS only restricts inbound browser requests, so this is fine.
  </action>
