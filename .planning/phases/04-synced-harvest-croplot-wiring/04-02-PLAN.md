  <files>organic-cert/src/lib/report-assembler.ts</files>
  <action>
The harvest log section in `assembleReportData` (around lines 264-287) reads `harvest.cropLots[0]?.lotNumber` for each HarvestEvent. This only works when the CropLot's `harvestEventId` points to THIS harvest. For synced harvests that updated an existing CropLot (accumulated into it), `harvest.cropLots` is empty because `harvestEventId` still points to the first harvest.

**Fix the "FLATTEN HARVESTS" section** to use a two-step lot number resolution:

1. Before the harvest loop, build a `Map<string, string>` that maps `fieldEnterpriseId` to `lotNumber`. Populate it from the enterprise's CropLots (available via the already-included `harvestEvents.cropLots` relation). Iterate through all enterprises and their harvest events to find CropLots:

```typescript
// Build enterprise-to-lot lookup for fallback
const enterpriseLotMap = new Map<string, string>();
for (const field of farm.fields) {
  for (const enterprise of field.enterprises) {
    for (const harvest of enterprise.harvestEvents) {
      for (const lot of harvest.cropLots) {
        // First lot found per enterprise wins (there should be at most one for synced data)
        if (!enterpriseLotMap.has(enterprise.id)) {
          enterpriseLotMap.set(enterprise.id, lot.lotNumber);
        }
      }
    }
  }
}
```

2. In the harvest loop, change the lot number resolution from:
```typescript
const lotNumber = harvest.cropLots[0]?.lotNumber ?? null;
```
to:
```typescript
const lotNumber = harvest.cropLots[0]?.lotNumber ?? enterpriseLotMap.get(enterprise.id) ?? null;
```

This ensures every HarvestEvent for an enterprise shows the enterprise's lot number, regardless of which specific harvest created the CropLot.

**Mass balance note:** No changes needed. The mass balance query (lines 295-335) reads directly from `prisma.cropLot.findMany(...)` — it does not go through HarvestEvent. Once CropLots exist (from Plan 01), mass balance works automatically. Verify this is still the case after reading the code.

**Do NOT change:**
