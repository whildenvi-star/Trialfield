**sync-status-provider.tsx:**
This is a `'use client'` component that:
1. Creates the Supabase browser client to supply the `getToken` callback to `useSyncStatus`
2. Calls `useSyncStatus(getToken)` to get sync state
3. Has a `mounted` state guard (useEffect sets it true after mount) — renders null until mounted to prevent SSR hydration mismatch on navigator.onLine
4. Manages the auto-dismiss "Synced X items" toast state:
   - On mount, add a `window.addEventListener('sync:completed', handler)` — remove on unmount
   - Handler: receive `event.detail.count`, set `syncMessage` state to `"Synced ${count} item${count !== 1 ? 's' : ''}"`, set `showSyncMessage` to true
   - Auto-dismiss: after 3000ms (use `useRef<ReturnType<typeof setTimeout>>` for the timer ID, clear on cleanup)
   - The toast renders as a small fixed or top-pinned div when `showSyncMessage` is true — use the same inline auto-dismiss pattern from `offline-summary-cards.tsx` (check that file for the exact pattern)
5. Renders:
   ```tsx
   <>
     {mounted && (
       <SyncStatusBanner
         isOnline={isOnline}
         pendingCount={pendingCount}
         syncState={syncState}
         errorMessage={errorMessage}
         onRetry={drainQueue}
       />
     )}
     {showSyncMessage && (
       <div className="...auto-dismiss toast styles...">
         {syncMessage}
       </div>
     )}
   </>
   ```
