Complete Phase 2 offline sync UI:
- SyncStatusBanner: slim top strip with Offline/Syncing/Error states, queue count inline
- QueueDetailSheet: read-only drawer listing queued items on banner tap
- Auto-dismiss "Synced X items" toast after successful drain
- ConflictDrawer: two-button resolution for data conflicts
- All components mounted in protected layout — visible from every page on mobile

  <how-to-verify>
Open portal.whughesfarms.com on your phone (or use Chrome DevTools mobile emulation at 375px):

**Test 1 — Online state (no banner):**
1. Open the portal while connected to the internet
2. Navigate to any page (Dashboard, Fields, etc.)
3. Expected: NO banner visible at the top of the screen — silence is correct for online state

**Test 2 — Offline banner:**
1. In Chrome DevTools: Network tab → throttling → Offline (or turn off phone WiFi/data)
2. Reload the page
3. Expected: Slim red/danger-colored strip appears below the header: "Offline" (or "Offline • N items queued" if items exist)
4. Expected: Banner is a slim strip — not a modal, not a full card

**Test 3 — Queue detail sheet:**
1. While offline, submit a field observation (this queues an item)
2. Return to any page — banner should show "Offline • 1 item queued"
3. Tap the banner
4. Expected: Sheet slides open showing the queued item with type and timestamp; no delete/cancel buttons; only a close button

**Test 4 — Sync drain and toast (reconnect):**
1. With a queued item, turn connectivity back on
2. Expected: Banner briefly shows "Syncing… 1 item" with spinner
3. Expected: Banner disappears and a brief "Synced 1 item" message appears then auto-dismisses within ~3 seconds
4. Expected: No banner visible after successful sync

**Test 5 — Visual feel:**
1. Navigate between pages while the banner is visible
2. Expected: Banner persists across page navigation (it's in the layout)
3. Expected: Banner feels like a system status bar — slim, non-intrusive, informational
