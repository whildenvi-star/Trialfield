  <action>
Create `src/hooks/useSyncStatus.ts` as a new file (check it does not already exist first).

This hook is the SINGLE source of truth for sync state in the app. It must NOT be instantiated in multiple components — it is mounted once in a client wrapper at layout level and state is passed down as props.

Implement the hook exactly as specified in RESEARCH.md Pattern 1, with these additional requirements:

1. **pendingCount must sum BOTH queues.** The existing codebase has `offlineQueue` (for QueuedOperation items from operation-queue store) AND `observationQueue` (for PendingObservation items from observation-queue store). The `refreshCount` function must call both `offlineQueue.getPending()` and `observationQueue.getPending()` and sum their lengths:
```typescript
const refreshCount = useCallback(async () => {
  const [ops, obs] = await Promise.all([
    offlineQueue.getPending(),
    observationQueue.getPending()
  ])
  setPendingCount(ops.length + obs.length)
}, [])
```

2. **Auto-drain on reconnect:** The `handleOnline` listener must call `drainQueue()` immediately when connectivity returns (already in RESEARCH.md Pattern 1).

3. **SSR safety:** Initialize `isOnline` to `true` (optimistic). Actual value is set in `useEffect` after mount via `navigator.onLine`. Never access `navigator` at module scope.

4. **SyncState type:** Export as `'idle' | 'syncing' | 'error'`

5. **SyncStatus interface:** Export the interface so the banner component can import it.

6. **getToken parameter:** The hook takes `getToken: () => Promise<string | null>` as its argument. This will be provided by the SyncStatusProvider client component in Plan 02.

7. **Conflict records:** After `processQueue` completes, if `result.conflicts.length > 0`, dispatch a `CustomEvent('sync:conflicts', { detail: result.conflicts })` on `window` so the conflict drawer (Plan 03) can react. Do NOT import or reference the conflict drawer directly — keep this hook independent.

8. **Inline notification:** After a successful drain (`result.synced > 0` and `result.failed.length === 0`), dispatch a `CustomEvent('sync:completed', { detail: { count: result.synced } })` on `window`. The SyncStatusProvider (Plan 02) will listen to this to show the auto-dismiss "Synced X items" message.

Import `processQueue` from `'@/lib/offline/sync-engine'`, `offlineQueue` and `observationQueue` from `'@/lib/offline/db'` (verify these export names by reading the file first).
