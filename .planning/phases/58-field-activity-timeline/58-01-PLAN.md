<objective>
Build the timeline aggregation API that fetches activity data from all 4 sources (farm-budget planned passes, organic-cert confirmed operations, organic-cert SyncedOperations for FieldOps machine data, and grain-tickets deliveries) for a given registry field ID, merges them into a unified chronological list, and returns them with source tags and degradation warnings.

Purpose: This is the data backbone for the field activity timeline view — without the aggregation API, the UI has nothing to render.
Output: TypeScript types, per-source fetch functions, and a GET /api/timeline/:fieldId endpoint.
</objective>

  <action>
    Create `glomalin-portal/src/lib/timeline/types.ts`:

    Define these types:
    - `TimelineSource = 'budget' | 'cert' | 'fieldops' | 'grain'`
    - `TimelineEntry` — a unified activity record with fields:
      - `id: string` — unique identifier
      - `source: TimelineSource`
      - `date: string | null` — ISO date string (null for PLANNED passes with no date)
      - `sortDate: string` — guaranteed date for sorting (PLANNED with no date uses "9999-12-31" to sort last)
      - `activityType: string` — e.g., "Tillage", "Planting", "Harvest", "Delivery", "SPRAYING"
      - `summary: string` — one-line collapsed display text (e.g., "[Budget] Spring N - Urea 120 lb/ac")
      - `detail: Record<string, unknown>` — source-specific expanded detail object
      - `status: 'planned' | 'confirmed' | 'completed' | null` — pass status where applicable
      - `pairedWith: string | null` — ID of the paired entry (budget planned linked to cert confirmed)
      - `sourceLink: string | null` — URL/path to view in source app (e.g., embed path for grain-tickets)
    - `TimelineResponse` — API response shape:
      - `fieldId: string`
      - `fieldName: string`
      - `entries: TimelineEntry[]`
      - `warnings: string[]` — names unavailable sources
      - `year: number`

    Create `glomalin-portal/src/lib/timeline/fetch-sources.ts`:

    Import proxy helpers from `@/app/api/mobile/_lib/proxy` and `resolveFieldEnterpriseId` from `@/app/api/mobile/_lib/cert-bridge`.

    **fetchBudgetActivities(registryFieldId: string): Promise<TimelineEntry[]>**
    - Call fetchBudgetService('/api/fields') to get all fields, find the field with matching registryFieldId.
    - If no match, return [].
    - Extract planned passes from field.machinery array. Each entry becomes a TimelineEntry with:
      - source: 'budget', status: 'planned', activityType: from implementName
      - summary: `[Budget] ${implementName}` with rate/product info from field.inputs if correlated
      - detail: { implementName, products (from field.inputs), acres: field.plantedAcres }
      - sourceLink: null (budget is an embed, no deep link)
    - Also extract field.inputs as separate "Input Application" planned entries with date null.

    **fetchCertActivities(registryFieldId: string): Promise<TimelineEntry[]>**
    - Resolve fieldEnterpriseId via resolveFieldEnterpriseId(registryFieldId).
    - Fetch field enterprise with operations: fetchCertService(`/api/field-enterprises/${fieldEnterpriseId}`).
    - Map fieldOperations to TimelineEntry with:
      - source: 'cert', status: 'confirmed', date: operationDate
      - activityType: op.type (FieldOpType enum value)
      - summary: `[Organic Cert] ${type}` + operator name if present
      - detail: { operator, acresWorked, equipment, passStatus, description, costPerAcre }
      - pairedWith: op.budgetImplementId ?? null (links back to budget planned pass)
      - sourceLink: '/app/org-cert' (embed link)
    - If cert is unreachable, throw (caller catches via allSettled).

    **fetchFieldOpsActivities(registryFieldId: string): Promise<TimelineEntry[]>**
    - Resolve fieldEnterpriseId via resolveFieldEnterpriseId(registryFieldId).
    - Fetch cert fields to find the organic-cert Field matching the enterprise, then get its CaseIHFieldMapping.
    - Fetch fetchCertService('/api/admin/staged-ops') filtered by mappedFieldId if possible, or fetch all and filter client-side.
    - NOTE: If no staged-ops endpoint with filter exists, fetch all SyncedOperations via fetchCertService and filter by mappedFieldId matching the resolved cert field ID.
    - Map SyncedOperations to TimelineEntry with:
      - source: 'fieldops', status: 'completed', date: operationDate
      - activityType: rawOpType (e.g., "FERTILIZER", "HERBICIDE")
      - summary: `[FieldOps] ${rawOpType}` + acresWorked
      - detail: { caseIHFieldName, products (from JSON), acresWorked, rawPayload snippet }
      - sourceLink: null
    - If no FieldOps data exists (no CaseIH sync set up), return [].
    - On failure, throw.

    **fetchGrainActivities(registryFieldId: string, cropYear: number): Promise<TimelineEntry[]>**
    - Call fetchGrainService('/api/farms') to get all farms, find the farm with registryId === registryFieldId.
    - If no match, return [].
    - Call fetchGrainService(`/api/tickets?cropYear=${cropYear}`) and filter tickets by farm name.
    - Map tickets to TimelineEntry with:
      - source: 'grain', date: ticket.date, status: 'completed'
      - activityType: 'Delivery'
      - summary: `[Grain] ${crop} - ${Math.round(netWeight/testWeight)} BU to ${destination or buyer}`
      - detail: { ticketNo, netWeight, moisture, crop, buyer, hbtBinNo, destination }
      - sourceLink: '/app/grain-tickets' (embed link)

    **mergeTimeline(sources: PromiseSettledResult<TimelineEntry[]>[], sourceNames: TimelineSource[]): { entries: TimelineEntry[], warnings: string[] }**
    - Iterate sourceNames and corresponding results.
    - For fulfilled: collect entries. For rejected: add source name to warnings.
    - Sort all collected entries by sortDate ascending, then by source priority (cert before budget for same date).
    - Pair budget + cert entries: for each cert entry with pairedWith, find the matching budget entry by ID and set the budget entry's pairedWith to the cert entry's ID.
    - Return { entries, warnings }.

    Create `glomalin-portal/src/app/api/timeline/[fieldId]/route.ts`:

    - Import createClient from '@/lib/supabase/server' for auth check.
    - Import mergeTimeline and all 4 fetch functions from '@/lib/timeline/fetch-sources'.
    - Import CURRENT_CROP_YEAR from '@/lib/config'.
    - Import TimelineResponse from '@/lib/timeline/types'.

    **GET handler:**
    1. Auth check: `const supabase = await createClient()`, `const { data: { user } } = await supabase.auth.getUser()`. Return 401 if no user.
    2. Extract fieldId from params: `const { fieldId } = await params`.
    3. Read optional `year` query param, default to CURRENT_CROP_YEAR.
    4. Fetch field name from farm-registry: fetchRegistryService('/api/fields') → find by id === fieldId. If not found, return 404 with `{ error: 'Field not found in registry' }`.
    5. Call all 4 sources with Promise.allSettled:
       ```
       const results = await Promise.allSettled([
         fetchBudgetActivities(fieldId),
         fetchCertActivities(fieldId),
         fetchFieldOpsActivities(fieldId),
         fetchGrainActivities(fieldId, year),
       ])
       ```
    6. Call mergeTimeline(results, ['budget', 'cert', 'fieldops', 'grain']).
    7. Build response: `{ fieldId, fieldName, entries, warnings, year }` satisfying TimelineResponse.
    8. Return NextResponse.json(response).
    9. Wrap entire handler in try/catch. On error, return 500 with `{ error: 'Failed to fetch timeline' }`.

    The progressive loading pattern (streaming entries as each source completes) is a UI concern handled in Plan 02 — this API returns the full merged result in a single response.
