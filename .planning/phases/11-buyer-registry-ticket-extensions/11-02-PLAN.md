Make 4 changes to server.js:

**1. Extend `dbTicketToJson` to include buyerId, destination, and cropYear:**
Add these three fields to the returned object:
```javascript
buyerId: dbTicket.buyerId || null,
destination: dbTicket.destination || null,
cropYear: dbTicket.cropYear
```
Also add `buyerName` by accepting an optional second parameter — a buyers lookup map (or null). If the map is provided and `dbTicket.buyerId` is set, include `buyerName: buyersMap[dbTicket.buyerId] || null`. This avoids N+1 queries. If no map provided, set `buyerName: null`.

**2. Add `?buyerId=N` filter support to `GET /api/tickets`:**
Before calling `prisma.ticket.findMany()`, build a `where` object. If `req.query.buyerId` is present, parse it as int and add `where.buyerId = bid`. Pass `{ where, orderBy: { date: 'desc' } }` to findMany.
Also load all buyers once via `prisma.buyer.findMany()` and build a `buyersMap` (`{ [id]: name }`) to pass to `dbTicketToJson` for enriching each ticket with `buyerName`.

**CRITICAL: Cache-Control override.** When the request has query params (e.g., `?buyerId=5`), set `res.set('Cache-Control', 'no-store')` BEFORE sending the response. This prevents the 10-second cache middleware from serving stale filtered results when the user changes the buyer filter. See research pitfall #3.

**3. Accept `buyerId` and `cropYear` in `POST /api/tickets`:**
In the existing POST handler, after destructuring req.body, extract:
```javascript
const buyerId = req.body.buyerId ? parseInt(req.body.buyerId, 10) || null : null;
const explicitCropYear = req.body.cropYear ? parseInt(req.body.cropYear, 10) : null;
```
In the `prisma.ticket.create({ data: { ... } })` call, add:
- `buyerId: buyerId` (nullable FK)
- `destination: null` (clear free-text when buyerId is set; new tickets always use buyer FK)
- For cropYear: use `explicitCropYear || getCropYear(date)` — explicit form value takes precedence, falls back to date-derived year.

**4. Accept `buyerId` and `cropYear` in `PUT /api/tickets/:id`:**
In the existing PUT handler, add `buyerId` and `cropYear` to the handled fields:
- If `req.body.buyerId !== undefined`: `updateData.buyerId = req.body.buyerId ? parseInt(req.body.buyerId, 10) || null : null`
- If `req.body.cropYear !== undefined`: `updateData.cropYear = parseInt(req.body.cropYear, 10) || getCropYear(req.body.date || existing.date)`
