**6. cropYear derivation logic — update getCropYear() in server.js:**

The user decision states: "June-Dec = that year, Jan-May = prior year (late delivery from prior harvest)."

Update the existing `getCropYear(dateStr)` helper:
```javascript
function getCropYear(dateStr) {
  if (!dateStr) return new Date().getFullYear();
  const parts = (dateStr + '').split('-');
  const year = parseInt(parts[0], 10);
  const month = parseInt(parts[1], 10);
  if (isNaN(year)) return new Date().getFullYear();
  // Harvest-season logic: Jun-Dec = this year, Jan-May = prior year
  if (month >= 1 && month <= 5) return year - 1;
  return year;
}
```

This means a ticket dated 2026-02-15 gets cropYear=2025 (late delivery from prior harvest). A ticket dated 2025-08-15 gets cropYear=2025 (normal harvest season).
