**Schema change — add GrainBin model to prisma/schema.prisma:**

Add after the Buyer model:

```
model GrainBin {
  id        Int      @id @default(autoincrement())
  name      String   @unique  // "Bin #1", "Bin #2", etc.
  capacity  Float    @default(0)  // Bushel capacity (e.g., 3700)
  notes     String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

Also add `grainBinId Int?` FK to the Ticket model (alongside existing `buyerId Int?`):
- `grainBinId Int?` — FK to GrainBin, null unless destination is a bin
- `grainBin GrainBin? @relation(fields: [grainBinId], references: [id])`
- Add `@@index([grainBinId])` to Ticket
- Add `tickets Ticket[]` relation to GrainBin model

Run `npx prisma migrate dev --name add-grain-bin` to create the migration.

**Server routes — add to grain-tickets/server.js:**

1. **GrainBin CRUD** (follows exact same pattern as existing Crops/Farms routes):
   - `GET /api/grain-bins` — `prisma.grainBin.findMany({ orderBy: { name: 'asc' } })`
   - `POST /api/grain-bins` — create with `{ name, capacity, notes }`. Validate name required. Handle P2002 for duplicate name.
   - `PUT /api/grain-bins/:id` — update by id. Handle P2025 (not found), P2002 (duplicate).
   - `DELETE /api/grain-bins/:id` — delete by id. Before deleting, check for tickets referencing this bin (`prisma.ticket.count({ where: { grainBinId: id } })`). If count > 0, return 409 with error "Cannot delete — N tickets reference this bin. Reassign them first." Handle P2025.

2. **Buyer proxy from farm-budget** (cross-app fetch):
   - `GET /api/buyers` — fetches buyers from `http://localhost:3001/api/buyers` using native `fetch()` with AbortController 5-second timeout. On success, return the array. On failure (network error, timeout, non-200), return `[]` with a `_source: 'unavailable'` marker so the client knows farm-budget is down. Log the error but do not crash.
   - Important: This is a read-only proxy. No POST/PUT/DELETE for buyers in grain-tickets.

3. **Merged destinations endpoint:**
   - `GET /api/destinations` — fetches both buyers (from farm-budget via the proxy logic above) and grain bins (from local DB). Returns a merged array:
     ```json
     [
       { "id": 5, "type": "buyer", "name": "Meristem Malt", "shortCode": "MRM", "buyerType": "maltster" },
       { "id": 1, "type": "bin", "name": "Bin #1", "capacity": 3700 }
     ]
     ```
   - Each item has `type: "buyer"` or `type: "bin"` for the client to distinguish. Sort alphabetically by name with bins grouped first (so "[BIN] Bin #1" style display ordering is handled client-side).

4. **BuyerColumnMap routes** (per-buyer settlement column mapping for Phase 12):
   - `GET /api/buyers/:id/column-maps` — `prisma.buyerColumnMap.findMany({ where: { buyerId: parseInt(id) } })`
   - `PUT /api/buyers/:id/column-maps` — upsert by compound unique `buyerId_fieldName`. Accept `{ fieldName, csvColumn }` in body. Use `prisma.buyerColumnMap.upsert({ where: { buyerId_fieldName: { buyerId, fieldName } }, update: { csvColumn }, create: { buyerId, fieldName, csvColumn } })`. Note: the compound unique field name is `buyerId_fieldName` (Prisma auto-generates from `@@unique([buyerId, fieldName])`).
   - `DELETE /api/buyers/:id/column-maps/:mapId` — delete a single column mapping by id.

5. **Update dbTicketToJson** to include `buyerId`, `grainBinId`, `destination`, and `cropYear` in the JSON output. These fields are already on the Ticket model but not currently emitted. Add:
   ```javascript
   buyerId: dbTicket.buyerId || null,
   grainBinId: dbTicket.grainBinId || null,
   destination: dbTicket.destination || null,
   cropYear: dbTicket.cropYear
   ```

6. **Cache-Control fix:** For `GET /api/tickets` and `GET /api/destinations`, override the middleware's `Cache-Control: public, max-age=10` with `res.set('Cache-Control', 'no-store')` at the route level. This prevents stale filter results when the user switches destination/buyer filters (see Research pitfall #3).
