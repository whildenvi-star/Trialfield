**page.tsx complete rewrite — preserve all existing sections (crop info, inputs) and enhance passes section:**

**State management:**
- `passes` state: mutable array of passes from API (with status, operationDate, operatorName, isUnplanned, fieldOperationId, fieldEnterpriseId)
- `operators` state: loaded on mount from /api/mobile/operators
- `currentUser` state: the logged-in user's certUserId and fullName (from Supabase session + profiles match in operators list)
- `undoTimer` state: active undo timeout reference
- `pendingConfirm` state: passId being confirmed (for undo window)
- `showAddSheet` state: boolean for add-pass bottom sheet
- `editingPass` state: pass object being edited (for edit bottom sheet)

**Progress bar** at top of passes section:
- "3 of 7 passes complete" with a filled bar segment
- Count confirmed passes (status === 'CONFIRMED') vs total passes
- Use accent color (#C8860A) for the filled portion

**Pass list — sorted by agronomic operation sequence:**
Define sort order: TILLAGE=1, PLANTING=2, SPRAYING=3 (covers Herbicide/Fertilizer), CULTIVATION=4, MOWING=5, HARVEST=6, OTHER=7 (covers Scouting). Sort passes by this order, then by passNumber within same type.

**Each pass row:**
- Left side: large tap target checkbox area (min 48x48px per mobile guidelines)
  - Unchecked (PLANNED): gray circle icon, tap triggers confirm flow
  - Checked (CONFIRMED): green filled check circle
  - Unplanned: blue badge "Unplanned" next to the type name
- Center: operation type name + "Pass #N" subtitle + confirmation date and operator name (if confirmed)
- Tap on confirmed pass -> open edit bottom sheet

**Confirm flow (per CONTEXT.md decisions):**
1. Operator taps unchecked pass -> instant optimistic UI update (gray -> green check, status -> CONFIRMED, date -> today, operator -> current user name)
2. Show undo toast at bottom of screen for 5 seconds: "Pass confirmed — Undo" with undo button
3. After 5 seconds (if not undone): fire confirmPass() API call
4. If undone: revert optimistic UI, clear timer, no API call
5. If API call fails: revert UI, show error toast "Failed to confirm pass"
6. Only one pending confirmation at a time — if operator taps another pass while toast showing, commit the first immediately (clear timer, fire API), then start new confirmation

**Undo toast component (inline, not a library):**
- Fixed position at bottom of screen, above FAB
- Dark background (#2a2218), accent text, "Undo" button
- 5-second auto-dismiss with CSS transition fade-out
- Uses setTimeout + useRef for timer management

**Add unplanned pass — Floating Action Button (FAB):**
- Fixed position bottom-right, 56x56px circle, accent color (#C8860A), plus icon
- Position: bottom-right with safe area padding (bottom: env(safe-area-inset-bottom, 16px) + 16px)
- Tap opens bottom sheet overlay

**Add pass bottom sheet:**
- Slides up from bottom, dark semi-transparent backdrop
- Form fields:
  - Field: pre-filled from current view (read-only display, not editable since they're already on the field page)
  - Operation type: dropdown/select with options: Tillage, Planting, Herbicide, Fertilizer, Harvest, Scouting, Other
  - Date: date input, defaults to today
  - Operator: shows logged-in user name, with "Change" link to show operator picker
  - Notes: optional text input
- "Add Pass" submit button
- On submit: POST to /api/mobile/passes/add, optimistically add to passes array with blue "unplanned" badge, close sheet
- On error: show toast, remove optimistic entry

**Edit pass bottom sheet:**
- Opens when tapping a confirmed pass
- Shows: operation type (read-only), current date (editable date input), current operator (with change option)
- "Save" button -> PUT to /api/mobile/passes/[passId]
- On success: update local passes array
- On error: show error toast, revert

**Operator selector (shared between add and edit):**
- Dropdown/select showing all operators from the operators list
- Each option: "Full Name (role)" format
- Defaults to logged-in user
- Resets to logged-in user after each confirmation (per CONTEXT.md: "Always resets to logged-in user after each confirmation")

**Styling notes:**
- Use existing dark soil aesthetic tokens from the project (bg #080604, surface #0e0c0b, border #2a2218, accent #C8860A, text #e8d8c0, muted #6a5a4a)
- But actually use Tailwind classes that exist in the project (bg-card, text-foreground, text-muted-foreground, border) — these map to the soil tokens via tailwind.config.ts
- Green confirmed: bg-green-100 text-green-800 (existing pattern from Phase 45)
- Blue unplanned: bg-blue-100 text-blue-800
- All interactive elements: min 48x48px touch targets
- No external libraries for toast/sheet — inline implementations (consistent with Phase 45 pattern of no external UI libs)
