  <name>Task 1: Install idb and create IndexedDB queue helpers + sync hook</name>
  <files>
    src/lib/observation-db.ts
    src/hooks/useObservationQueue.ts
  </files>
  <action>
1. Install idb: `npm install idb`

2. Create src/lib/observation-db.ts — IndexedDB helpers using idb ^8:
   - Define PendingObservation interface: { localId?: number, note: string, fieldId?: string, photoBlob?: Blob, synced: boolean, createdAt: number }
   - openObservationDB(): opens database 'observation-queue' version 1, creates 'pending' object store with autoIncrement keyPath 'localId' and 'synced' index
   - Wrap openDB in try/catch — if IndexedDB unavailable (Safari Private), return null and let callers fall back to direct upload
   - queueObservation(payload): adds record with synced:false, createdAt:Date.now()
   - getPendingObservations(): returns all records where synced === false via the synced index
   - markSynced(localId): reads record, puts with synced:true
   - purgeOldSynced(maxAgeDays=7): deletes synced:true records older than maxAgeDays
   - Export PendingObservation type and all functions

3. Create src/hooks/useObservationQueue.ts — React hook:
   - useObservationQueue() returns { submitObservation, pendingCount, isSyncing }
   - submitObservation(note, fieldId?, photoBlob?):
     a. Always write to IndexedDB first via queueObservation (if IDB available)
     b. Immediately attempt uploadOne() to POST /api/observations
     c. If upload succeeds: markSynced the record, show sonner toast "Observation submitted"
     d. If upload fails (network error): leave in queue, show toast "Saved offline — will sync when connected"
     e. If IDB unavailable (Safari Private fallback): attempt direct upload only, show error toast on failure
   - uploadOne(obs: PendingObservation): builds FormData if photoBlob present, otherwise JSON body. POSTs to /api/observations. Returns boolean success.
   - Sync-on-reconnect: useEffect registers window.addEventListener('online', syncPending). Also calls syncPending on mount if navigator.onLine is true.
   - syncPending(): iterates getPendingObservations(), uploads each, marks synced on success. After loop, if any synced > 0: show sonner toast "{count} observation(s) synced". Then calls purgeOldSynced().
   - pendingCount: state number updated after every submit and sync
   - isSyncing: boolean state true during syncPending execution
   - Cleanup: remove event listener on unmount
  </action>
