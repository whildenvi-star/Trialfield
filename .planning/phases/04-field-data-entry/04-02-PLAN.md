  **Step 3: Create `src/lib/offline/observation-queue.ts`**

  Export these functions (follow the `offlineQueue` pattern from `db.ts`):

  ```typescript
  import { getDb } from './db'
  import type { PendingObservation } from './types'

  const STORE = 'observation-queue' as const

  function isAvailable(): boolean {
    return typeof indexedDB !== 'undefined'
  }

  export const observationQueue = {
    async add(obs: Omit<PendingObservation, 'localId' | 'synced' | 'createdAt'>): Promise<number> {
      if (!isAvailable()) throw new Error('IndexedDB unavailable')
      const db = await getDb()
      const record: Omit<PendingObservation, 'localId'> = {
        ...obs,
        synced: false,
        createdAt: Date.now(),
      }
      return db.add(STORE, record as PendingObservation) as Promise<number>
    },

    async getPending(): Promise<PendingObservation[]> {
      if (!isAvailable()) return []
      const db = await getDb()
      return db.getAllFromIndex(STORE, 'by-synced', IDBKeyRange.only(0))
      // Note: booleans in IDB indexes -- false maps to 0. If this doesn't work,
      // store synced as 0/1 number instead of boolean, or use getAll() and filter.
    },

    async markSynced(localId: number): Promise<void> {
      if (!isAvailable()) return
      const db = await getDb()
      const tx = db.transaction(STORE, 'readwrite')
      const record = await tx.store.get(localId)
      if (record) {
        record.synced = true
        await tx.store.put(record)
      }
      await tx.done
    },

    async pendingCount(): Promise<number> {
      const pending = await this.getPending()
      return pending.length
    },

    async purgeOld(maxAgeDays = 7): Promise<void> {
      if (!isAvailable()) return
      const db = await getDb()
      const cutoff = Date.now() - maxAgeDays * 24 * 60 * 60 * 1000
      const tx = db.transaction(STORE, 'readwrite')
      let cursor = await tx.store.openCursor()
      while (cursor) {
        if (cursor.value.synced && cursor.value.createdAt < cutoff) {
          await cursor.delete()
        }
        cursor = await cursor.continue()
      }
      await tx.done
    },
  }
  ```

  IMPORTANT: IndexedDB boolean indexing can be tricky. The `synced` field stored as `false` may index as `0` or not at all depending on the browser. A safer approach is to store `synced` as `0 | 1` number in the interface instead of boolean, OR just use `getAll()` and filter in JS. Choose the approach that works -- test by checking `getPending()` returns correct results.

  Internal helper `uploadObservation(obs: PendingObservation)`:
  - If `obs.photoBlob` exists: build FormData with `note` and `photo` (blob as File), POST to `/api/observations` without Content-Type header
  - If no photo: POST JSON `{ note: obs.note }` to `/api/observations` with `Content-Type: application/json`
  - Throw on non-2xx response

  Hook `useObservationQueue()` returns:
  - `submitObservation(note: string, photoBlob?: Blob): Promise<void>` -- queue-first: writes to IDB, then immediately tries upload. On upload success, marks synced. On failure, leaves in queue.
  - `pendingCount: number` -- number of unsynced observations
  - `isSyncing: boolean` -- true while sync drain is running
  - `lastSyncMessage: string | null` -- e.g. "2 observations synced" for toast display

  Implementation:
  - `submitObservation`: Call `observationQueue.add({ note, photoBlob })` to get localId. Then try `uploadObservation(obs)`. On success: `observationQueue.markSynced(localId)`. On catch: leave in queue (will sync on reconnect). Update `pendingCount` after.
  - `syncPending` function: Get all pending via `observationQueue.getPending()`. Set `isSyncing: true`. Loop through, try `uploadObservation` for each, `markSynced` on success. Count successes. Set `isSyncing: false`. Update `pendingCount`. If any synced, set `lastSyncMessage` to `"${count} observation(s) synced"`.
  - `useEffect` for online listener:
    ```typescript
    useEffect(() => {
      const handler = () => { syncPending() }
      window.addEventListener('online', handler)
      // Also sync on mount if online (catch reconnects while app was backgrounded)
      if (navigator.onLine) syncPending()
      return () => window.removeEventListener('online', handler)
    }, [])
    ```
  - `useEffect` to initialize pendingCount on mount: `observationQueue.pendingCount().then(setPendingCount)`
  - Also run `observationQueue.purgeOld(7)` on mount (fire and forget) to clean old synced records.

  Safari Private Mode guard: Wrap all IDB calls in try/catch. If IDB is unavailable (Safari private mode), fall back to direct upload only (no queuing). Set a `idbAvailable: boolean` ref on mount.
