<task type="auto">
  <name>Task 1: FieldObservation Prisma model and API routes</name>
  <files>
    prisma/schema.prisma
    src/app/api/observations/route.ts
    src/app/api/observations/photo/[filename]/route.ts
  </files>
  <action>
1. Add FieldObservation model to prisma/schema.prisma:
   - id: String @id @default(cuid())
   - farmId: String (relation to Farm)
   - fieldId: String? (optional relation to Field)
   - submittedById: String (relation to User)
   - note: String
   - photoPath: String? (relative filename under uploads/observations/)
   - createdAt: DateTime @default(now())
   - Add @@index([farmId, createdAt]) and @@index([fieldId])
   - Add the inverse relation fields on Farm, Field, and User models (fieldObservations FieldObservation[])

2. Run `npx prisma db push` to sync schema.

3. Create POST /api/observations (src/app/api/observations/route.ts):
   - Call getAuthContext() first; return 401 if null
   - Detect content-type: multipart/form-data vs application/json
   - For multipart: use request.formData() to extract note, fieldId, photo File
   - If photo present: mkdir uploads/observations/ with recursive:true, generate filename as `obs-${Date.now()}-${crypto.randomUUID()}.jpg`, write buffer via fs.writeFile
   - For JSON: parse body for note and fieldId
   - Validate note is non-empty string (use zod: z.string().min(1).max(2000))
   - Create FieldObservation via prisma.fieldObservation.create with farmId from auth context
   - Return 201 with created observation JSON

4. Create GET /api/observations (same route.ts):
   - Call getAuthContext(); return 401 if null
   - Query prisma.fieldObservation.findMany where farmId matches auth context
   - Order by createdAt desc, limit 50
   - RBAC: CREW sees only their own (where submittedById = ctx.id); ADMIN and OFFICE see all for the farm
   - Return 200 with observations array

5. Create GET /api/observations/photo/[filename]/route.ts:
   - Call getAuthContext(); return 401 if null
   - Sanitize filename with path.basename() to prevent traversal
   - Read file from uploads/observations/{safe filename}
   - Return Response with Content-Type: image/jpeg, Cache-Control: max-age=86400
   - Return 404 if file not found
  </action>
