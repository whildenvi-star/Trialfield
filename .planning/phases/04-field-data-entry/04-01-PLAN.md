  **Step 1: Create the Supabase table.**

  Run SQL against the project's Supabase instance. The project uses `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` env vars. Check if `npx supabase` CLI is available; if not, create a migration SQL file at `supabase/migrations/` or document the SQL for the user to run.

  Table schema:
  ```sql
  CREATE TABLE IF NOT EXISTS field_observations (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    submitted_by uuid NOT NULL REFERENCES auth.users(id),
    note text NOT NULL,
    photo_path text,
    created_at timestamptz NOT NULL DEFAULT now()
  );

  ALTER TABLE field_observations ENABLE ROW LEVEL SECURITY;

  CREATE POLICY "Users can insert own observations"
    ON field_observations FOR INSERT
    WITH CHECK (auth.uid() = submitted_by);

  CREATE POLICY "Users can read own observations"
    ON field_observations FOR SELECT
    USING (auth.uid() = submitted_by);

  CREATE INDEX IF NOT EXISTS idx_field_obs_user_date
    ON field_observations(submitted_by, created_at DESC);
  ```

  **Step 2: Create `src/app/api/observations/route.ts`**

  POST handler:
  - Authenticate: `createClient()` from `@/lib/supabase/server`, then `supabase.auth.getUser()`. Return 401 if no user.
  - Check `content-type` header: if includes `multipart/form-data`, parse with `request.formData()`; otherwise `request.json()`.
  - Extract `note` (required, non-empty string -- return 400 if missing/empty).
  - If photo File present in FormData:
    - `const uploadsDir = path.join(process.cwd(), 'uploads', 'observations')`
    - `await fs.mkdir(uploadsDir, { recursive: true })`
    - Generate filename: `obs-${Date.now()}-${crypto.randomUUID()}.jpg`
    - `const buffer = Buffer.from(await file.arrayBuffer())`
    - `await fs.writeFile(path.join(uploadsDir, filename), buffer)`
    - Store filename (not full path) as `photo_path`
  - Insert: `supabase.from('field_observations').insert({ submitted_by: user.id, note, photo_path }).select().single()`
  - Return 201 with the created row.

  GET handler:
  - Same auth pattern.
  - `supabase.from('field_observations').select('*').eq('submitted_by', user.id).order('created_at', { ascending: false })`
  - Return JSON with `{ observations: data }`.

  Use `import { promises as fs } from 'fs'` and `import path from 'path'` and `import crypto from 'crypto'` for Node built-ins.

  State:

  - `feedback: { type: 'success' | 'error', message: string } | null`

  Photo capture input:
  ```tsx
  <input
    ref={photoRef}
    type="file"
    accept="image/*"
    capture="environment"
    className="sr-only"
    onChange={handlePhotoChange}
  />
  ```
  Trigger via a styled button that calls `photoRef.current?.click()`.

  `handlePhotoChange`:
  - Get file from `e.target.files?.[0]`
  - Resize using `createImageBitmap(file)` then canvas:
    ```typescript
    async function resizeImage(file: File, maxPx = 1200): Promise<Blob> {
      const img = await createImageBitmap(file)
      const scale = Math.min(maxPx / img.width, maxPx / img.height, 1)
      const canvas = document.createElement('canvas')
      canvas.width = Math.round(img.width * scale)
      canvas.height = Math.round(img.height * scale)
      const ctx = canvas.getContext('2d')!
      ctx.drawImage(img, 0, 0, canvas.width, canvas.height)
      return new Promise(resolve =>
        canvas.toBlob(blob => resolve(blob!), 'image/jpeg', 0.8)
      )
    }
    ```
  - Store resized blob in state, create preview URL via `URL.createObjectURL(blob)`
  - Clean up previous preview URL if replacing photo

  Submit handler `handleSubmit`:
  - Prevent default, set `isSubmitting: true`
  - If `photoBlob` exists: build `FormData` with `note` field and `photo` blob (as File). `fetch('/api/observations', { method: 'POST', body: formData })` -- do NOT set Content-Type header.
  - If no photo: `fetch('/api/observations', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ note }) })`
  - On 201: set success feedback, reset form state (note='', photoBlob=null, revoke preview URL)
  - On error: set error feedback with message from response
  - Finally: `isSubmitting: false`

  Layout (mobile-first with Tailwind + glomalin tokens):
  - `<form onSubmit={handleSubmit} className="space-y-4">`
  - Textarea: `className="w-full rounded-lg border border-glomalin-border bg-glomalin-surface text-glomalin-text p-3 min-h-[120px] text-base"` with `rows={4}`, placeholder "Describe what you observed in the field..."
  - Attach Photo button: `className="w-full py-3 rounded-lg border border-glomalin-border text-glomalin-text font-mono text-sm"` -- shows "Change Photo" when preview exists
  - Photo preview: `<img src={previewUrl} className="w-full rounded-lg mt-2" />` when previewUrl is set
  - Submit button: `className="w-full py-3 rounded-lg bg-glomalin-accent text-white font-bold text-base disabled:opacity-50"` -- disabled when `!note.trim() || isSubmitting`
  - Feedback: success as green text, error as red text below form

  Cleanup: `useEffect` with cleanup function that revokes `previewUrl` on unmount.
