Create DEPLOY.md at the repo root. This is the single deployment reference for the entire platform. Structure it as numbered sections in dependency order:

**Section 1: Prerequisites**
- Ubuntu 22.04+ VPS with root/sudo access (recommend 2GB+ RAM for 8 apps)
- A registered domain name
- DNS provider access (for A records)

**Section 2: DNS Configuration**
- Create A records pointing to VPS IP for:
  - yourdomain.com
  - portal.yourdomain.com
  - budget.yourdomain.com
  - fsa.yourdomain.com
  - malt.yourdomain.com
  - cert.yourdomain.com
  - registry.yourdomain.com
  - seed.yourdomain.com
  - tickets.yourdomain.com
- Note: Can use wildcard *.yourdomain.com A record instead of 9 individual records
- DNS must propagate before Caddy can issue TLS certificates

**Section 3: System Setup**
- Update system packages
- Install Node.js 20 LTS (via NodeSource)
- Install PM2 globally (`npm install -g pm2`)
- Install PostgreSQL 15+ (for grain-tickets, organic-cert databases)

**Section 4: Install Caddy**
- Add Caddy official apt repo (per caddyserver.com/docs/install#debian-ubuntu-raspbian)
- `sudo apt install caddy`
- Note: Caddy runs as a systemd service and handles auto-HTTPS

**Section 5: Clone & Install**
- `git clone <repo-url> /srv/farm-ops`
- `cd /srv/farm-ops`
- For each app directory: `cd <app> && npm install && cd ..`
- For Next.js apps: `cd glomalin-portal && npm run build && cd ../organic-cert && npm run build && cd ..`
- Note about organic-cert having its own .git (may need separate clone or submodule handling)

**Section 6: Environment Configuration**
- For each app, copy `.env.example` to `.env` and fill in real values
- List the critical env vars that MUST be set:
  - `PORTAL_ORIGIN=https://portal.yourdomain.com` (all Express apps)
  - `NEXT_PUBLIC_SITE_URL=https://portal.yourdomain.com` (glomalin-portal)
  - `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` (glomalin-portal)
  - `DATABASE_URL` (grain-tickets, organic-cert)
- Reference .env.example files for full list

**Section 7: PostgreSQL Setup**
- Create databases: `createdb grain_tickets` and `createdb organic_cert`
- Create app user with password
- Run Prisma migrations: `cd grain-tickets && npx prisma db push && cd ../organic-cert && npx prisma db push`

**Section 8: Start Apps with PM2**
- `cd /srv/farm-ops`
- `pm2 start ecosystem.config.js`
- `pm2 save`
- `pm2 startup` (generates systemd config so PM2 auto-starts on reboot)
- Verify: `pm2 status` should show all 8 apps "online"

**Section 9: Configure Caddy**
- Copy Caddyfile: `sudo cp /srv/farm-ops/Caddyfile /etc/caddy/Caddyfile`
- Set domain: edit `/etc/caddy/Caddyfile` or set DOMAIN env var in Caddy systemd override
- For systemd env var approach: `sudo systemctl edit caddy` and add `Environment=DOMAIN=yourdomain.com` under `[Service]`
- Reload: `sudo systemctl reload caddy`
- Caddy will automatically obtain Let's Encrypt certificates
- Verify: `curl -I https://portal.yourdomain.com` should return 200

**Section 10: Updating (Deploy Workflow)**
- ssh into VPS
- `cd /srv/farm-ops && git pull origin main`
- `cd <changed-app> && npm install`
- For Next.js: `npm run build`
- `pm2 restart ecosystem.config.js && pm2 save`

**Section 11: Port Map Reference**
- Table of app name, port, subdomain, technology (matches ecosystem.config.js comment header)

**Section 12: Troubleshooting**
- `pm2 logs <app-name>` to check app logs
- `sudo journalctl -u caddy` to check Caddy logs
- `caddy validate --config /etc/caddy/Caddyfile` to check config syntax
- If cert fails: verify DNS propagation with `dig portal.yourdomain.com`
- If app 502: check `pm2 status` — app may have crashed

Use fenced code blocks for all commands. Keep it practical — no theory, just steps. Target audience: a developer who knows Linux basics but hasn't set up this specific stack before.
