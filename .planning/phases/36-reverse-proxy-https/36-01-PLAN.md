<task type="auto">
  <name>Task 1: Create Caddyfile with subdomain routing for all 8 apps</name>
  <files>Caddyfile</files>
  <action>
Create a Caddyfile at the repo root with 8 site blocks — one per app. Use placeholder domain `{$DOMAIN:farm.example.com}` as an environment variable so the actual domain can be set at deploy time without editing the file.

Subdomain-to-port mapping (matches ecosystem.config.js):
- portal.{$DOMAIN} -> localhost:3000 (glomalin-portal, Next.js)
- budget.{$DOMAIN} -> localhost:3001 (farm-budget, Express)
- fsa.{$DOMAIN} -> localhost:3002 (fsa-acres, Express)
- malt.{$DOMAIN} -> localhost:3003 (meristem-malt, Express)
- cert.{$DOMAIN} -> localhost:3004 (organic-cert, Next.js)
- registry.{$DOMAIN} -> localhost:3005 (farm-registry, Express)
- seed.{$DOMAIN} -> localhost:3006 (seed-inventory, Express)
- tickets.{$DOMAIN} -> localhost:3007 (grain-tickets, Express)

Each site block should:
1. Use `reverse_proxy localhost:{port}` directive
2. Include `encode gzip` for compression
3. For the two Next.js apps (portal, cert), add `header_up X-Forwarded-Proto {scheme}` so Next.js knows it's behind HTTPS (needed for secure cookies, Supabase auth redirects)

Also add a bare `{$DOMAIN}` block that redirects to `portal.{$DOMAIN}` so the root domain goes to the portal.

Add a comment header documenting: the placeholder domain env var, how to set it (`DOMAIN=yourdomain.com caddy run`), and the port map matching ecosystem.config.js.

Caddy handles auto-HTTPS natively — no explicit TLS config needed. Caddy will automatically obtain Let's Encrypt certificates for all subdomains when DNS A records point to the VPS IP.
