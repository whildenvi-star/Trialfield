Mark Done handler (per RESEARCH.md Pattern 3 + Pitfall 3):
```ts
async function handleMarkDone(plan: CachedCropPlan, passId: string) {
  // 1. Write to offline queue first
  await offlineQueue.add({
    type: 'confirm-pass',
    fieldId: plan.fieldName,  // use fieldName as fieldId — check actual field if db has an id
    passId,
    passType: pass.type,
    operationDate: new Date().toISOString().split('T')[0],
    operatorId: '',   // unknown in client component without user prop — use empty string; server-side sync fills in
    operatorName: '',
    description: 'Marked complete from dashboard',
  })
  // 2. Optimistic update — per RESEARCH.md Pitfall 3: MUST update state here
  setLocalPlans(prev =>
    prev.map(p =>
      p === plan
        ? { ...p, passes: p.passes.map(pass => pass.id === passId ? { ...pass, status: 'CONFIRMED' as const } : pass) }
        : p
    )
  )
}
```

NOTE: If `offlineQueue.add` signature requires operatorId, check `src/lib/offline/types.ts` for which fields are required vs optional. Use empty string for operatorId/operatorName if required — the sync engine on the server fills in the actual user from the auth session during replay.
