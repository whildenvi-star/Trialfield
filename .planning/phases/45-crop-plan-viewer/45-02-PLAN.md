<objective>
Build the mobile-first Crop Plan Viewer UI: a field list page with enterprise grouping and search, a field detail page with crop info, inputs, and pass checklist, and the offline caching layer that keeps both pages functional without network.

Purpose: Field operators need to check crop plans on their phone in the field. This phase makes that data browsable and searchable, with offline support for areas with no signal. Per user decisions: enterprise-grouped cards with simple text search, single scrollable detail page, pull-to-refresh, 48px tap targets, and stale-data color indicators.

Output: Two Next.js pages, a sync/cache utility, and a reusable offline banner component.
</objective>

1. Create `glomalin-portal/src/lib/offline/crop-plan-sync.ts`:
   - Export `syncCropPlans(token: string): Promise<{ fields: CropPlanListItem[]; syncTimestamp: string }>` that:
     a. Fetches `/api/mobile/crop-plans` with `Authorization: Bearer ${token}` header
     b. On success: iterate response.fields, call `cropPlanCache.put(field)` for each (the put method uses fieldId as key and sets cachedAt automatically)
     c. Return the fetched data
     d. On network error: throw so caller can fall back to cache
   - Export `syncCropPlanDetail(token: string, fieldId: string): Promise<CachedCropPlan>` that:
     a. Fetches `/api/mobile/crop-plans/${fieldId}` with Bearer token
     b. On success: call `cropPlanCache.put(response)` to cache it
     c. Return the data
     d. On network error: throw
   - Export `getCachedCropPlans(): Promise<CachedCropPlan[]>` that:
     a. Calls `cropPlanCache.getAll()` and returns the array
   - Export `getCachedCropPlan(fieldId: string): Promise<CachedCropPlan | undefined>` that:
     a. Calls `cropPlanCache.get(fieldId)`
   - Export `getLastSyncTime(): Promise<string | null>` that:
     a. Calls `cropPlanCache.getLastSyncTime()` and returns ISO string or null
   - Define and export `CropPlanListItem` interface matching the API list response shape:
     `{ fieldId: string, fieldName: string, crop: string, variety: string, acres: number, enterprise: string, enterpriseId: string }`
   - All functions must have SSR guards (typeof window === 'undefined' check) returning empty arrays/null

2. Create `glomalin-portal/src/components/pwa/offline-banner.tsx`:
   - Client component ('use client')
   - Uses `navigator.onLine` and `online`/`offline` event listeners (with mounted guard to prevent SSR hydration mismatch)
   - When offline: render a thin banner at the top with text "Offline - showing cached data" on a muted background (e.g., bg-amber-100 text-amber-800 or similar)
   - When online: render nothing (null)
   - Export as default for easy import
   - Keep it simple — no props needed for now
