<objective>
Create the portal API routes that aggregate crop plan data from farm-budget and organic-cert into a mobile-friendly shape for the Crop Plan Viewer. This is the data layer that both online and offline UI will consume.

Purpose: Field list and detail pages need a single API call rather than multiple cross-service fetches from the client. The portal acts as a BFF (backend for frontend), joining budget fields, enterprise groupings, inputs, seed data, and pass status from organic-cert into the CachedCropPlan-compatible shape.

Output: Two API routes under /api/mobile/crop-plans/ plus an updated proxy helper.
</objective>

3. Fetch in parallel using Promise.all:
   a. `fetchBudgetService('/api/fields/' + fieldId)` -> field object (includes inputs[], seed, machinery[])
   b. `fetchBudgetService('/api/enterprises')` -> enterprises[] for name lookup
4. From the field object, extract:
   - `fieldName`: field.name
   - `crop`: field.crop
   - `variety`: field.seed?.variety ?? null
   - `population`: field.seed?.population ?? null (seed population/rate)
   - `seedTreatment`: field.seed?.treatment ?? null
   - `acres`: field.plantedAcres ?? field.acres ?? 0
   - `enterprise`: lookup enterprise name from field.enterpriseId
   - `inputs`: map field.inputs to `{ product: i.productName, rate: String(i.quantity ?? ''), unit: i.unit ?? 'per acre' }` — filter out items with no productName
   - `passes`: map field.machinery to `{ id: m.id ?? crypto.randomUUID(), type: m.implementName, passNumber: idx + 1, status: 'PLANNED' as const, operationDate: null, operatorName: null }` — each machinery entry is a planned pass
5. Try to enrich pass status from organic-cert:
   - The field.id from farm-budget may not directly correspond to organic-cert field-enterprise IDs. Look for a `registryFieldName` property on the field that links to organic-cert.
   - If unable to match, leave all passes as PLANNED (graceful degradation — Phase 46 will wire up proper confirmation)
   - This is intentional: the MVP shows planned passes from the budget, and Phase 46 adds confirmation
6. Return `NextResponse.json({ fieldId, fieldName, crop, variety, population, seedTreatment, acres, enterprise, inputs, passes, syncTimestamp: new Date().toISOString() })`
7. On farm-budget failure: return 502. On field not found: return 404.

The response shape matches the CachedCropPlan interface from lib/offline/types.ts — the UI layer can cache it directly.
