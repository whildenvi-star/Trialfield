**multer diskStorage setup in server.js:**

Add a second multer instance (alongside the existing `upload` memoryStorage used for `/api/scan`) for settlement file uploads:

```javascript
const settlementStorage = multer.diskStorage({
  destination: function (req, file, cb) {
    const dir = path.join(__dirname, 'uploads', 'settlements');
    fs.mkdirSync(dir, { recursive: true });
    cb(null, dir);
  },
  filename: function (req, file, cb) {
    const ext = path.extname(file.originalname).toLowerCase();
    cb(null, Date.now() + '-' + Math.random().toString(36).slice(2) + ext);
  }
});
const uploadSettlement = multer({
  storage: settlementStorage,
  limits: { fileSize: 10 * 1024 * 1024 },
  fileFilter: function (req, file, cb) {
    const allowed = ['.csv', '.xlsx', '.xls'];
    const ext = path.extname(file.originalname).toLowerCase();
    if (allowed.includes(ext)) cb(null, true);
    else cb(new Error('Only CSV and Excel files are accepted'));
  }
});
```

Add `const fs = require('fs');` and `const XLSX = require('xlsx');` at the top of server.js (xlsx is already in package.json).

**Settlement routes to add in server.js:**

1. `POST /api/settlements/parse` — accepts `file` (multipart), `buyerId` (form field), `cropYear` (form field). Uses `uploadSettlement.single('file')`. Parses the saved file with `XLSX.read(fs.readFileSync(req.file.path), { type: 'buffer', cellDates: true })`. Extracts headers from first row (filter empty strings). Generates 5-row preview (skip header row, filter blank rows, slice 0-5, align to header count). Creates a `Settlement` record with `buyerId`, `cropYear`, `sourceFile: req.file.originalname`, `filePath: req.file.path`. Loads saved `BuyerColumnMap` for this buyer and returns `{ settlementId, headers, previewRows, savedMapping }`. Return 400 if no file, buyerId, or cropYear. Return 422 if file is empty.

2. `POST /api/settlements/:id/commit` — accepts JSON `{ mapping }` where mapping is `{ fieldName: csvColumnName }` (e.g., `{ ticketNo: 'Ticket No', netWeight: 'Net Weight' }`). Loads the Settlement by id, reads the saved file from `settlement.filePath`, parses with XLSX. Maps each data row through the mapping to build SettlementLine objects. For each row:
   - `ticketNo`: `String(value).trim() || null`
   - `date`: if value is Date, use directly; if string, `new Date(value)` with NaN check; else null. Apply noon UTC anchoring (`T12:00:00.000Z`) for date-only strings.
   - `netWeight`, `moisture`, `netBushels`: `parseFloat(value) || null`
   - `price`, `deductions`, `netPayment`: `value != null ? String(parseFloat(value)) : null` (Decimal fields require strings)
   Filter out rows where both ticketNo and netWeight are null (empty rows). Use `prisma.settlementLine.createMany({ data: lines })`. Save/update BuyerColumnMap via `prisma.buyerColumnMap.upsert` on each mapping entry (using `buyerId_fieldName` compound key). Return `{ ok: true, linesCreated: N }`.

3. `GET /api/settlements` — returns all settlements ordered by `importedAt desc`, including `buyer` relation (for buyer name) and `_count: { lines: true }` for line count. Add `Cache-Control: no-store`.

4. `GET /api/settlements/:id` — returns a single settlement with its buyer relation and all lines.

5. `DELETE /api/settlements/:id` — deletes settlement by id (SettlementLines cascade via `onDelete: Cascade` in schema). Also deletes the uploaded file from disk using `fs.unlinkSync(settlement.filePath)` if filePath exists. Return `{ ok: true }`.

**Important implementation notes:**
- Use `header: 1` and `defVal: ''` in `XLSX.utils.sheet_to_json()` to get raw array rows.
- Filter headers with `.filter(h => h.length > 0)` to drop empty columns.
- Do NOT normalize ticket numbers — store raw from file. Normalization happens in Phase 13.
- Decimal fields (price, deductions, netPayment) MUST be passed as strings to Prisma, not floats.
- Wrap `fs.unlinkSync` in try/catch in the delete handler (file may not exist).

**settlements.js — new file:**

Create `grain-tickets/public/settlements.js` as a vanilla JS module following the same pattern as tickets.js and farms.js (functions called from tab activation, no ES modules). Key functions:

1. `initSettlements()` — called once on page load. Attaches event listeners for the sub-nav toggle (import vs history), file upload button, commit button.

2. `loadSettlementBuyers()` — populate the buyer `<select>` from `refData.destinations` filtering for `type === 'buyer'`. Each option value is the buyer id (numeric). Show buyer name + shortCode.

3. `handleFileUpload()` — on "Upload & Preview" click:
   - Validate buyer selected, crop year entered, file selected
   - Build `FormData` with `file`, `buyerId`, `cropYear`
   - `fetch('/api/settlements/parse', { method: 'POST', body: formData })`
   - On success: store `settlementId` in module state, call `renderColumnMapping()` and `renderPreview()`
   - On error: show error message in status div

4. `renderColumnMapping(headers, savedMapping)` — show the column mapping panel. Render 8 mapping rows (one per SettlementLine field): ticketNo (required), date, netWeight, moisture, netBushels, price, deductions, netPayment. Each row has a label and a `<select>` with:
   - `-- skip --` (value "")
   - One `<option>` per header from the parsed file
   - Pre-select options matching `savedMapping` values
   Show the "Commit Import" button.

5. `renderPreview(headers, previewRows)` — render a `<table>` showing the first 5 data rows with their original column headers. This gives the user confidence the file parsed correctly before they commit.

6. `handleCommit()` — on "Commit Import" click:
   - Collect mapping from all `<select>` elements (skip empty values)
   - Validate at least ticketNo is mapped (show error if not)
   - `fetch('/api/settlements/' + settlementId + '/commit', { method: 'POST', headers: {'Content-Type':'application/json'}, body: JSON.stringify({ mapping }) })`
   - On success: show success toast with line count, reset the import form, switch to history view, reload settlement list
   - On error: show error message

7. `loadSettlements()` — fetch `GET /api/settlements`, render a table with columns: Buyer, Crop Year, Source File, Imported At (formatted), Lines, Actions (delete button). Called when history sub-tab is activated.

8. `handleDeleteSettlement(id)` — confirm dialog, then `fetch('/api/settlements/' + id, { method: 'DELETE' })`. On success, reload settlement list.
