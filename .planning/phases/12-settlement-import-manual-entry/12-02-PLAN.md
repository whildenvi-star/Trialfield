Add the following routes to server.js for manual settlement entry and line management. These supplement the parse/commit routes from Plan 12-01.

**1. POST /api/settlements — create a settlement header for manual entry**

Accepts JSON `{ buyerId, cropYear, notes }`. Creates a `Settlement` record with no `sourceFile` and no `filePath` (both null — distinguishes manual from file-import settlements). Returns the created settlement with `{ id, buyerId, cropYear, notes, importedAt }`. Validate that `buyerId` and `cropYear` are present (return 400 if missing). Parse `buyerId` and `cropYear` as integers.

**2. POST /api/settlements/:id/lines — add a single settlement line**

Accepts JSON with all 8 SettlementLine fields plus optional `notes`:
```json
{
  "ticketNo": "H066666",
  "date": "2025-10-15",
  "netWeight": 55480,
  "moisture": 12.9,
  "netBushels": 985.3,
  "price": 5.45,
  "deductions": 12.50,
  "netPayment": 5357.39,
  "notes": "optional note"
}
```

Implementation:
- Verify the settlement exists (404 if not)
- Parse fields:
  - `ticketNo`: `(ticketNo || '').trim() || null`
  - `date`: if truthy, `new Date(date + 'T12:00:00.000Z')` with NaN check; else null (noon UTC anchoring per Phase 10 convention)
  - `netWeight`, `moisture`, `netBushels`: `parseFloat(value) || null`
  - `price`, `deductions`, `netPayment`: `value != null && value !== '' ? String(parseFloat(value)) : null` (Decimal fields must be strings)
  - `notes`: `(notes || '').trim() || null`
- Create via `prisma.settlementLine.create({ data: { settlementId, ...parsedFields } })`
- Return 201 with the created line

**3. PUT /api/settlements/:settlementId/lines/:lineId — update a settlement line**

Accepts JSON with any subset of the 8 fields + notes. Only update fields that are present in the body. Use the same parse logic as the create route. Verify the line exists and belongs to the given settlement (return 404 if not). Update via `prisma.settlementLine.update()`. Return the updated line.

**4. DELETE /api/settlements/:settlementId/lines/:lineId — delete a single settlement line**

Verify the line exists and belongs to the settlement (404 if not). Delete via `prisma.settlementLine.delete()`. Return `{ ok: true }`.

**5. GET /api/settlements/:id/lines — list all lines for a settlement**

Return all `SettlementLine` records for the given settlement, ordered by `id asc` (insertion order). Include settlement buyer info via the settlement relation for display context.

**Important implementation notes:**
- All Decimal fields (price, deductions, netPayment) must be passed as strings to Prisma, not floats.
- Date anchoring uses noon UTC (`T12:00:00.000Z`) per Phase 10 convention to prevent timezone shift.
- The line routes are nested under `/api/settlements/:id/lines` — the `settlementId` parameter ensures lines are scoped to their settlement.
- For the PUT route, build `updateData` object dynamically — only include fields present in `req.body` to allow partial updates.
