   - Mark as `"use client"` (needs state, event handlers, fetch).
   - This page has three visual states based on connection status:

   **State A: Not Connected (no field mappings exist)**
   - Show a Card explaining what FieldOps connection does.
   - Show connection status: "Not Connected" badge (red).
   - "Test Connection" button: calls `GET /api/admin/fieldops/fields`.
     - If successful (fields returned): show green success message with field count, link to "Set up field matching" at `/admin/fieldops/matching`.
     - If returns empty fields (linkedAccountWarning): show a prominent yellow/amber alert: "Your Case IH account returned no fields. This may be a Linked Account limitation -- contact CNH support to verify API access for your account." (API-05 requirement).
     - If error (401/500): show red error with details. For 401, suggest checking FIELDOPS_CLIENT_ID, FIELDOPS_CLIENT_SECRET, and FIELDOPS_SUBSCRIPTION_KEY environment variables.
   - If using mock mode: show an info banner "Running in mock mode (no API credentials configured)".

   **State B: Connected, Fields Matched (mappings exist)**
   - Show connection status: "Connected" badge (green).
   - Show sync state: last sync timestamp (formatted with date-fns), last sync status badge (success=green, partial=yellow, error=red, no_data=amber), operations imported count.
   - If `linkedAccountWarning` is true: show persistent amber warning (API-05).
   - "Sync Now" button: calls `POST /api/admin/sync`.
     - While syncing: disable button, show spinner, display "Syncing..." text.
     - On completion: show result summary card with: fieldsProcessed, operationsStaged, operationsSkipped, warnings list. Use a per-field breakdown format showing field name + operations count.
     - On error: show red error card with message.
   - Links to sub-pages:
     - "Manage Field Matching" -> `/admin/fieldops/matching`
     - "Review Staged Operations" -> `/admin/fieldops/review` (show pending count badge if > 0)
     - "Operation Type Mappings" -> show inline table of current mappings with ability to add new ones (POST to `/api/admin/fieldops/op-types`)

   **State C: Sync Complete, Pending Review**
   - Same as State B but with a prominent call-to-action: "X operations pending review" linking to `/admin/fieldops/review`.

3. **Fetch initial data on mount**:
   - GET `/api/admin/fieldops/fields` to check connection status.
   - GET `/api/admin/fieldops/mappings` to check if field matching is done.
   - Load sync state from the fields response (or a separate sync-state endpoint).
   - GET `/api/admin/staged-ops?status=PENDING` to get pending count for the review badge.

4. **Operation Type Mapping inline section**:
   - Show a table of existing rawOpType -> nopCategory mappings.
   - If unmapped types exist (from GET /api/admin/fieldops/op-types), show them with a dropdown to select NOP category (TILLAGE, PLANTING, APPLICATION, HARVEST, CULTIVATION, SPRAYING) and a "Save" button.
   - Per user decision: first time a new Case IH operation type appears, admin maps it. Mapping auto-applies to subsequent syncs.

Use existing UI components: Card, Button, Badge, Table from `@/components/ui/`. Use `toast` from sonner for success/error notifications. Follow the same styling patterns as the existing admin page and dashboard.

**Part A: Field Matching Page (`/admin/fieldops/matching/page.tsx`)**

1. Mark as `"use client"`.

2. **Layout**: Two-column layout on desktop (1 column on mobile):
   - Left column: Case IH fields list (from `GET /api/admin/fieldops/fields`)
   - Right column: Selected Case IH field details + organic-cert field search

3. **Case IH Fields List (left)**:
   - Show all Case IH fields with: name, farm name (from GFFB hierarchy), area if available.
   - Each field shows its mapping status:
     - Green check + organic-cert field name if already matched
     - Yellow "Unmatched" badge if not yet matched
   - Filterable: text search at top of list to filter Case IH fields by name.
   - Clickable: selecting a Case IH field opens the matching panel on the right.

4. **Matching Panel (right)**:
   - Shows selected Case IH field details at top (name, farm, area).
   - Uses `cmdk` (CommandDialog or inline Command component) for searching organic-cert fields.
     - Import from `cmdk` package (already installed).
     - Fetch organic-cert fields from the existing `/api/fields` endpoint (or embed in the mappings GET response).
     - Show field name, crop year, organic status in search results.
     - Auto-suggest: pre-populate suggested match using normalized field name comparison (call findSuggestedMatch logic client-side, or have the API return suggestions).
   - On selecting an organic-cert field: call `POST /api/admin/fieldops/mappings` with `{ caseIHFieldId, caseIHFieldName, caseIHFarmName, organicCertFieldId }`.
   - Show toast on success ("Field matched: [Case IH name] -> [organic-cert name]").
   - "Create New Field" option at bottom of search results: if no match exists, link to the field creation flow (or show a note "Create this field in Fields first, then return to match it"). Per user decision: unmatched fields prompt admin to create a new field record.

5. **Bulk progress indicator**: Show "X of Y fields matched" progress bar at top of page.

6. **Re-mapping**: If a field is already matched, clicking it shows current mapping with "Change Mapping" button that reopens the cmdk search.

**Part B: Staged Operations Review Page (`/admin/fieldops/review/page.tsx`)**

1. Mark as `"use client"`.

2. **Fetch staged ops**: `GET /api/admin/staged-ops?status=PENDING` on mount.

3. **Layout**: Table with columns:
   - Case IH Field Name
   - Operation Date (formatted with date-fns)
   - Operation Type (rawOpType + mappedNopCategory if available)
   - Products (summary -- first product name or "N products")
   - Acres
   - Mapped Field (organic-cert field name, or "Unmapped" warning)
   - Actions (Approve / Reject buttons)

4. **Filtering**:
   - Filter by sync run (dropdown of syncRunIds with dates)
   - Filter by field name (text search)
   - Filter by operation type
   - Status tabs: PENDING (default), APPROVED, REJECTED

5. **Approve action**:
   - Call `POST /api/admin/staged-ops/[id]` with `{ action: 'approve' }`.
   - On 409 (manual record conflict): show dialog explaining "A manual record already exists for this date and type. Manual data takes priority. Would you like to reject this synced record instead?" with Reject button.
   - On success: show toast, remove from PENDING list (or move to APPROVED tab), update count.

6. **Reject action**:
   - Show small dialog/popover asking for optional rejection reason.
   - Call `POST /api/admin/staged-ops/[id]` with `{ action: 'reject', rejectedReason }`.
   - On success: show toast, remove from PENDING list.

7. **Bulk approve**: "Approve All Visible" button that approves all currently displayed PENDING ops. Calls approve for each sequentially (not in parallel to avoid overwhelming the server). Show progress ("Approving 5 of 23...").

8. **Empty state**: If no PENDING ops, show "All caught up! No operations pending review." with link back to FieldOps hub.

9. **Unmapped field warning**: If a staged op has no mappedFieldId, show a warning icon and tooltip "This field is not yet matched to an organic-cert field. Match it in Field Matching before approving." Disable the Approve button for unmapped ops.

Use existing UI components: Table, Button, Badge, Dialog, Input, Command from `@/components/ui/`. Use `toast` from sonner. Follow existing page patterns (see field-enterprises/page.tsx for table layout reference).
