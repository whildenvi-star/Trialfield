Update the `generateLotNumber` function in `organic-cert/src/lib/lot-generator.ts` to accept an optional `label` parameter and append a label-derived suffix when present. This prevents `CropLot.lotNumber` unique constraint violations when the same crop appears twice on a field in the same year.

1. **Change the function signature:**
   ```typescript
   export function generateLotNumber(
     year: number,
     crop: string,
     fieldName: string,
     label?: string | null
   ): string {
   ```

2. **Add label suffix logic** after computing the base lot number:
   ```typescript
   const base = `${year}-${abbreviateCrop(crop)}-${abbreviateField(fieldName)}`;
   if (!label) return base;
   // Convert label to a safe suffix: "North 40" -> "N40", "Corn 2" -> "C2"
   const suffix = label.replace(/[^a-zA-Z0-9]/g, "").substring(0, 4).toUpperCase();
   return `${base}-${suffix}`;
   ```

3. **Do NOT change** `abbreviateCrop` or `abbreviateField` — they remain as-is. Only the `generateLotNumber` function gains the optional fourth parameter.

This is backward-compatible: all existing call sites pass 3 arguments, and the new `label` parameter defaults to `undefined` which produces the same base lot number as before.

Implement acre validation and utilization across three API routes. All acre math is computed on read, never stored.

### A. POST /api/field-enterprises/route.ts

After the enterprise `prisma.fieldEnterprise.create()` call:

1. **Pass label to lot generator:** Update the `generateLotNumber` call site to forward `body.label` (or the created enterprise's `label`) as the fourth argument. This ensures split-field lot numbers get the label suffix.

2. **Add acre reconciliation query** (parallel with field lookup for performance):
   ```typescript
   const [siblings, field] = await Promise.all([
     prisma.fieldEnterprise.findMany({
       where: { fieldId: body.fieldId, cropYear: body.cropYear },
       select: { plantedAcres: true },
     }),
     prisma.field.findUnique({
       where: { id: body.fieldId },
       select: { totalAcres: true },
     }),
   ]);
   ```

3. **Compute and include in response:**
   ```typescript
   const totalPlanted = siblings.reduce((s, e) => s + e.plantedAcres, 0);
   const acreWarning =
     field && totalPlanted > field.totalAcres
       ? `Planted acres (${totalPlanted.toFixed(1)}) exceed field total (${field.totalAcres.toFixed(1)} ac)`
       : null;

   return NextResponse.json(
     {
       ...enterprise,
       acreWarning,
       acreReconciliation: {
         totalPlanted,
         fieldTotal: field?.totalAcres ?? null,
         fallowAcres: field ? Math.max(0, field.totalAcres - totalPlanted) : null,
         isOverAllocated: field ? totalPlanted > field.totalAcres : false,
       },
     },
     { status: 201 }
   );
   ```

4. **Do NOT block saves** when over-allocated. The warning is informational per user decision: "yellow warning but allow save."

### B. PUT /api/field-enterprises/[id]/route.ts

Apply the same acre reconciliation pattern after `prisma.fieldEnterprise.update()`:

1. Query siblings and field (same parallel pattern as POST).
2. Compute `acreWarning` and `acreReconciliation` object.
3. Include both in the JSON response alongside the updated enterprise.
4. If the `label` field is being updated, also regenerate the lot number using the updated label.

### C. GET /api/fields/route.ts

Add `acreUtilization` to each field in the response when it has multiple enterprises for the current crop year:

1. After the existing field-enterprise aggregation logic, for each field compute:
   ```typescript
   const currentYear = new Date().getFullYear();
   const currentYearEnterprises = field.enterprises.filter(
     (e) => e.cropYear === currentYear
   );
   const totalPlantedAcres = currentYearEnterprises.reduce(
     (s, e) => s + e.plantedAcres, 0
   );
   const hasMultipleEnterprises = currentYearEnterprises.length > 1;
   ```

2. Add to the field response object:
   ```typescript
   acreUtilization: hasMultipleEnterprises
     ? {
         planted: totalPlantedAcres,
         total: field.totalAcres,
         fallow: Math.max(0, field.totalAcres - totalPlantedAcres),
         isOverAllocated: totalPlantedAcres > field.totalAcres,
       }
     : null,
   ```

3. Single-enterprise fields return `acreUtilization: null` — they display total acres as today per locked decision.

### Design decisions per user context:
- Over-allocation: yellow warning, allow save (per locked decision)
- Under-allocation: "X ac unallocated" as informational (per locked decision) — this maps to `fallowAcres` in the `acreReconciliation` response
- Acre utilization only on multi-enterprise fields (per locked decision)
- Fallow acres are NEVER stored — always computed as `field.totalAcres - sum(plantedAcres)` (per research anti-pattern guidance)
